## ① 機能概要

利用規約（Terms of Service, ToS）は、CivilizationOS を使う前にユーザーが**サービスの性質・データの扱い・禁止行為**を理解し、同意したことを記録するための法務・運用機能である。条文本体は `docs/legal/terms-of-service.md` をドラフト正本とし、プライバシー・市場注意書は兄弟文書として併読する。

ログイン入口では「利用規約に同意します」チェックで**マジックリンク送信を止める** UI ゲートがある。初回オンボーディング完了時には API 上 `agree_terms` / `agree_privacy` を R2 ユーザープロファイルに保存する設計がある。表示ルート `/terms` では Kernel 経由で規約全文を閲覧できる。

本稿および `docs/legal/` は**法的助言ではない**。公開前は `human-review-checklist.md` に沿った**人間（または専門家）レビュー**が必須である。

- 固体ログイン `LoginPage.tsx` の同意チェックは**クライアント側のみ**で、`/api/auth/magiclink` には `agree_terms` を送っていない。

## ⑥ MiniKernel / C-USB 上の位置づけ

```
World → FeatureNode: auth → Kernel → Component
```

| 項目 | 値 |
|------|-----|
| FeatureNode | `auth` |
| 閲覧 Kernel | `login` / `view` — ルート `/terms`、legacy `A900_terms` |
| 同意 Kernel | `register` / `create` — ルート `/terms/onboarding`、legacy `V05_terms` |
| 固体データ | `frontend/src/kernel/componentData/A900_terms.json`（本文 Markdown 埋め込み） |
| ログイン固体 | `LoginPage.tsx`（Phase 0 正本 UI。Kernel `A0_login.json` は参照・E2E スナップショット用） |
| ITO | IN: ユーザー同意操作 → Transform: API 検証・R2 保存 → OUT: `onboardingComplete` / JWT 利用可能状態 |

規約本文コンポーネントは C-USB 文明原子ではなく、**auth FeatureNode 配下の View コンポーネント**として Kernel UUID ルーティングに従う。条項の「正」は repo 内 Markdown（`docs/legal/`）であり、Kernel JSON 内 Markdown は**表示キャッシュ**に近い位置づけ（現状は内容が乖離している）。

**ドラフト JSON スキーマ（`terms-media-resolver-map` v1）**

規約本文・Kernel JSON には **`section_id` のみ**を埋め込み、YouTube URL は別マッピングで解決する。

```json
{
  "$schema": "civilization-os/legal/terms-media-resolver-map/v1",
  "terms_version": "LEGAL-TOS-DRAFT-001",
  "effective_from": "2026-04-09T00:00:00+09:00",
  "entries": [
    {
      "section_id": "tos-art-01",
      "legal_ref": "第1条",
      "easy_read_ref": "E-01",
      "media_kind": "explain",
      "youtube_url": "https://www.youtube.com/watch?v=EXAMPLE01",
      "title": "第1条を入れた意図（例）"
    },
    {
      "section_id": "tos-rev-001-to-002",
      "legal_ref": null,
      "easy_read_ref": null,
      "media_kind": "revision_why",
      "youtube_url": "https://www.youtube.com/watch?v=EXAMPLE02",
      "title": "0.1-draft から 1.0.0 へ改定した理由（例）",
      "effective_from": "2026-06-01T00:00:00+09:00",
      "from_version": "LEGAL-TOS-DRAFT-001",
      "to_version": "LEGAL-TOS-1.0.0"
    },
    {
      "section_id": "tos-rev-001-to-002",
      "legal_ref": null,
      "easy_read_ref": null,
      "media_kind": "revision_what",
      "youtube_url": "https://www.youtube.com/watch?v=EXAMPLE03",
      "title": "1.0.0 で変わる点の解説（例）",
      "effective_from": "2026-06-01T00:00:00+09:00",
      "from_version": "LEGAL-TOS-DRAFT-001",
      "to_version": "LEGAL-TOS-1.0.0"
    }
  ]
}
```

| フィールド | 説明 |
|------------|------|
| `terms_version` | 適用する規約版 ID（マッピングファイル単位または entry 単位） |
| `section_id` | 条または改定イベントの安定 ID（本文に埋め込む参照キー） |
| `media_kind` | `explain`（条意図） \| `revision_why`（改定理由） \| `revision_what`（改定内容） |
| `youtube_url` | 解決先 URL（本文には出さない） |
| `effective_from` | 当該 URL が有効になる日時（改定動画は版切替日と一致） |

**解決フロー（案）**: UI が `section_id` + `media_kind` + 表示中 `terms_version` を resolver に渡す → `entries[]` から `youtube_url` を取得 → 条文本体は不変。
