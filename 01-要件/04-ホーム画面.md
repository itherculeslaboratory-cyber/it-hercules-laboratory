## ① 機能概要

ホーム画面は、認証済みユーザーがログイン後に最初に着地する **World レイヤーの司令塔**（ルート `/`・`/home`）である。今日の状態（プラチナ・貢献度・未読通知）と **次の一手**（固体観測・各 FeatureNode 入口）だけを置き、詳細操作は観測ハブ・マイページ・各機能画面へ委譲する。

表側の情報設計は REQ-024 に従い、旧 A 系画面名ではなく **World → FeatureNode → Kernel** の文明 OS 概念で入口を示す。初回オンボーディング直後は Twin 案内ツアーを起動し、「文明 OS のどこにいるか」を短時間で理解できる導線とする。

実装は `HomePage.tsx` および `GET /api/home/dashboard` により **implemented** だが、REQ-024 の IA 再整理（6–7 トップ分類との完全整合・メニュー重複解消）は **active** で UX 完走は partial とみなす。

## ③ スコープ内 / 外

### スコープ内（本機能の責務）

- 認証後の **World 着地** と **主要 FeatureNode への入口地図**
- ユーザー単位の **要約表示**（経済指標・未読件数）
- **非 PII 集計**による文明ミニマップ（`buildHomeDashboard`）
- 司法インボックス **プレビュー**（正本は `/judicial/inbox`）
- 環境 IoT **due 予定の軽量通知**（最大 3 件表示）
- オンボーディング直後の **Twin 案内ツアー** 起動
- 空状態・ローディング・API 失敗時の **フォールバック表示**（近似値・再試行導線）

### スコープ外（別機能・別 Kernel に委譲）

- 固体観測・環境 IoT・デジタル観測の **本体フロー**（観測 FeatureNode）
- マーケット取引・決済・出品操作（マーケット / 経済）
- 設定・プロフィール編集の **詳細 UI**（マイページ / `/me/settings`）
- 司法案件の **審理・投票本体**（司法 FeatureNode）
- Builder / Driver / OS 切替（Builder・`/dev/world-routing` 等）
- 全機能一覧・メニュー IA の **正本定義**（`/help/all-features`・`menuConfig.ts` — REQ-024 Slice 1 と共同）
- 血統 OS 専用ダッシュ（黒基調・写真サムネ禁止は `preferences.md` §B/C — ホームは文明 OS コア明るめテーマ）

## ④ 機能要件

### ルーティング・認可

| ID | 要件 |
|----|------|
| H-001 | ルート `/` および `/home` は認証済みユーザーのみ `HomePage` を表示する。未認証時は `null` 返却（Auth ガード側でログインへ） |
| H-002 | ハンバーガーメニュー・全機能一覧と **同じ木** を辿れること（導線の二重管理を避け、REQ-024 で一本化予定） |

### ヒーロー（Above the fold）

| ID | 要件 |
|----|------|
| H-010 | キッカー「司令塔 · Hub」、見出し「ホーム」、リード文（最大 3 行・1 行 1 情報）を表示 |
| H-011 | プライマリ CTA「固体観測を始める」（`data-testid="home-cta-solid-observation"`）→ `SOLID_OBSERVATION_ROUTE` |
| H-012 | セカンダリ CTA「全機能を見る」→ `/help/all-features` |
| H-013 | インラインリンク：観測ハブ・デジタル観測 |
| H-014 | アカウント要約パネル：`@handle`、プラチナ（`home-platinum`）、貢献度（`home-contribution`）、未読通知件数 |

### 主要スコープ（Action Cards）

| ID | 要件 |
|----|------|
| H-020 | 5 カード固定：観測 / 個人 / 通知 / マーケット / 全機能。各カードに eyebrow・1 行説明・primary/secondary リンク |
| H-021 | 各カードは **入口の地図** に留め、詳細操作は先方 FeatureNode へ委譲（画面内完結させない） |
| H-022 | `data-testid="home-action-cards"` で E2E 可能 |

### 文明の現在地（Dashboard strip）

| ID | 要件 |
|----|------|
| H-030 | `GET /api/home/dashboard`（JWT 必須）から minimap を取得。正本は `buildHomeDashboard` |
| H-031 | 指標 3 つ：観測ペース（件/週・直近7日血統追記）、Twin 信頼度（OK 比率、無データ時「データ待ち」）、テンプレ成長（ValueCheck ユーザテンプレ行数） |
| H-032 | API 失敗時：`fallbackHomeMinimap` でユーザー値由来の近似表示し、警告コールアウト（`home-minimap-error`）を出す。raw エラー文字列は出さない |
| H-033 | ローディング中は「更新中」バッジ。指標は `home-minimap-metrics` 内 testid 付き |

### 通知

| ID | 要件 |
|----|------|
| H-040 | 司法インボックス `GET /api/judicial/inbox/me?limit=5` をプレビュー。正本 UI は `/judicial/inbox` |
| H-041 | 未読件数をヘッダーとヒーローに反映。リスト最大 5 件、案件リンク `/judicial/case/:caseId` |
| H-042 | 空状態（`home-inbox-empty`）・ローディング（`home-inbox-loading`）・エラー（`home-inbox-error` + インボックス直接導線） |
| H-043 | 環境 due 予定：`fetchDueEnvSchedules(jwt, 30)` 成功時、最大 3 件を `home-env-due-notifications` に表示 |
| H-044 | 将来の観測・経済通知追加時も **本セクションに統合**（別バナー乱立禁止） |
| H-045 | **次回観測 upcoming/overdue** — `today_lines` に最大 3 行（個体名 · 日数 · 観測入力 deep link） | **ver1 IN** · [`05-観測.md`](./05-観測.md) OBS-RX-UX-11 · OBS-FUP-11 · §4.17.5 |

## ⑥ MiniKernel / C-USB 上の位置づけ

```
World（文明全体の現在地）
  └─ FeatureNode: auth / home（司令塔）
       └─ Kernel: dashboard（集約表示）
            └─ Component: HomeHero, ActionCardGrid, MinimapStrip, InboxPreview, QuickLinks
```

| レイヤー | 本機能での役割 |
|----------|----------------|
| **World** | ホームは REQ-024 §3 の `World` 概念の **Primary surface**（`/home`, `/civilization/engine`, `/dev/world-routing` と並列） |
| **FeatureNode** | 入口地図として observation / mypage / market / board / search 等へ **リンクのみ**（処理核は各 FN） |
| **Kernel** | `dashboard` 型 — IN: JWT + RAG/lineage 集計、Transform: `buildHomeDashboard`、OUT: minimap DTO |
| **Component** | `HomePage` は View 層の **合成 Component**。fetch は `useHomeDashboard`・`useJudicialInboxPreview` に分離（三層分離） |
| **C-USB** | ホーム自体は C-USB ポートを公開しない。表示データは既存 API の OUT スキーマ（OpenAPI `HomeDashboardOk`）に従う |

画面単位設計は禁止（WorldSystem）。ホームは **複数 Kernel 出力の read-only ハブ** であり、文明原子の編集・フォークは行わない。
