## 1. 機能概要

マチアプ（ユーザー価値観数値化）は、ユーザーが **好み・価値観** を離散信号として記録し、**検索 rerank** に反映する機能群である。**採用済み経路**は **ValueCheck**（多次元マトリクス ×/−/◯ + 総合 yes/maybe/no）と **`valuePreset` 静的プリセット**（`Phase_value_alignment.md`）。**Kernel pilot** の `MatchApp`（`/match`）は画像 YES/NO でタグ重みを学習する **理想 UI** の PoC だが、**SAMPLE データのみ**で本番観測画像未接続。oral 理想は fb_00046（緑/赤ボタン・テンプレ収束・再測定タグ）および IHL **append-only tag events**（`指示/it-hercules-laboratory/99-アーカイブ/2026.06-06-legacy` §8.6）と思想が近い。

## 2. ユーザーができること

### 採用済み（ValueCheck）

- **ValueCheck セッション記録**: テンプレ次元（size/horn/color/price/availability 等）を ×/−/◯/スキップで入力し、総合 fit を yes/maybe/no で送信。
- **カスタムテンプレ**: 既定テンプレを fork し、独自次元ラベル・hint を定義（JSONL 追記のみ）。
- **検索ブースト**: ログイン状態で `valueCheckBoost=1` または `valueCheckSessions` クエリにより、**本人の直近セッション**を entity 検索スコアに反映。
- **valuePreset**: クエリパラメータで静的価値観プリセットを適用（ValueCheck とは別ソース）。

### 未採用 / PoC（MatchApp）

- **評価画面**: サンプル画像（実際はラベル付きカード）に対し YES(+1)/NO(-1) 評価。
- **ランキング・タグ・分析**: 収束度・topTags・weakTags・yesRate 表示（`engine.ts` 数式）。
- **Undo**: 直前評価の取り消し。

## 4. 機能要件

### ValueCheck（採用）

| ID | 要件 |
|----|------|
| FR-MCH-01 | **セッションレコード**: `session_id`, `user_id`, `template_id`, `dimensions`, `overall_fit`, `created_at`（`ValueCheckSessionRecord`）。 |
| FR-MCH-02 | **セル値**: `"x"` / `"minus"` / `"circle"` / `null`（スキップ）。総合 `"no"` / `"maybe"` / `"yes"` 必須。 |
| FR-MCH-03 | **既定テンプレ**: `valuecheck_default_v1` — 5 次元（size, horn, color, price, availability）。 |
| FR-MCH-04 | **API**: `GET/POST /api/value-check/*` — テンプレ取得・セッション追記・mine 一覧（認証要）。 |
| FR-MCH-05 | **INSERT ONLY**: `value_check_sessions.jsonl` / templates JSONL は追記のみ。 |
| FR-MCH-06 | **検索順序**: RAG 類似度（+ Twin）→ **`valuePreset`** → **`valueCheckSearchBoost`**（混同禁止）。 |
| FR-MCH-07 | **集約**: 直近 N セッション（1〜15）を `mergeValueCheckBoostContexts`。avoid と prefer 衝突時 **avoid 優先**。 |
| FR-MCH-08 | **ブースト定数**: `STEP_PREFER` / `MAX_PREFER_BONUS` / `STEP_AVOID` / `MAX_AVOID_PENALTY`（ソース正: `valueCheckSearchBoost.ts`）。 |
| FR-MCH-09 | **Zod 正本**: ユーザテンプレ次元検証は `valueCheckTemplateSchema.ts`（JSON Schema は参照）。 |
