## ① 機能概要

設定機能は、認証済みユーザーが **AI 接続・固体観測 LLM・UI 露出・Push 通知** を自己管理する入口である。単一の「設定要件書」としては未統合だが、実装は `/me/settings`（`MeSettingsPage.tsx`）に集約しつつ、プロフィール（マイページ）・通知一覧・各 FeatureNode 内 LLM と **責務分離** している。REQ-024 により dev/admin 導線の **初期非表示** と Capability 表示が追補された。パスワード・OAuth アカウント設定は **非サポート**（認証はマジックリンク正本）。

## ③ スコープ内 / 外

### スコープ内

- `/me/settings` ルートと `MeSettingsPage.tsx`
- **表示言語（`locale`）の変更** — `POST /api/auth/locale` または preferences PATCH（`21-翻訳-言語.md` `FR-I18N-SET-01`〜`04`）

- Web Push 登録（`POST /api/auth/push/subscribe`）

### スコープ外

- 利用規約再同意フロー（ログイン/オンボーディング側）

- 全 FeatureNode の設定を 1 画面に **完全統合**（将来 IA 再設計で検討）
- IHL リポジトリの設定（別認証・別 env 想定）

## ④.5 取引前設定（ユーザー確定 · 2026-06-07 · U-MKT-DSP-10）

> **設計哲学**: **個人情報は掲示板・公開チャットに打ち込まない**。取引 **前** に設定し、争い時は構造化参照のみ（[`11-裁判.md`](./11-裁判.md) §6.8 · FR-DSP-19）。

### 判断・理由

| 判断 | 理由 | 却下した代替案 |
|------|------|----------------|
| **局留め・配送先・振込口座は設定で事前登録** | PII を **公開争い・ボードログ** から分離。配送は局留め **推奨**（氏名が必要） | プライベートボードに住所・口座を自由記述 |
| **参照範囲は取引相手のみ** | マッチング成立後の **当事者 P2P** に必要最小限を渡す | 全公開プロフィールに住所掲載 |
| **マッチング前の設定完了を促す**（必須化は詳細設計） | 取引開始時にボードで PII を書く **誘因を減らす** | 初回取引まで未設定可のまま放置 |

### 設定項目

| 項目 | 内容 | 必須/推奨 | 参照範囲 |
|------|------|-----------|----------|
| **局留め設定** | 郵便局留め用 **氏名（フルネーム）** · 局留め先の指定 | **推奨**（配送時） | **取引相手のみ** |
| **配送先** | 住所等 — 宅配・局留め以外の配送に使う場合 | **推奨**（取引前） | **取引相手のみ** · 掲示板非公開 |
| **銀行振込口座** | P2P 振込用口座情報 | **推奨**（取引前） | **取引相手のみ** · ボード／チャットに **打ち込まない** |

**UX ガイダンス**: プライベートボード・公開エクスチェンジの入力欄では、PII 直書きを避け **設定済み情報を参照** する導線を示す（詳細設計 · UI 設計で具体化）。

## ④ 機能要件（番号付き）

1. **FR-SET-01** 認証済みユーザーのみ `/me/settings` にアクセス可能とする（REQ-002）。
2. **FR-SET-02** 論文系 LLM 設定は `updatePaperMatchLlmPreferences` 経由でサーバに PATCH し、API Key は **保存時のみ送信**・一覧では `hasPaperMatchLlmApiKey` フラグのみ表示する（REQ-006）。
3. **FR-SET-03** 固体観測 LLM は **独立フィールド**（`solidObservationLlm*`）とし、論文 LLM と混在させない。
4. **FR-SET-04** dev ツール表示は既定 **off**。on にすると DEV / Builder / world-routing 等の導線をメニューに露出する（REQ-024）。
5. **FR-SET-05** admin 導線表示は **admin 相当ユーザー** のみ変更可能とする（`isAdminLikeUser`）。
6. **FR-SET-06** Capability サマリ（roles / economy_master_propose / judicial_resolve）は **読み取り専用** で表示する。
7. **FR-SET-07** Web Push 登録は VAPID 公開鍵未取得時に **代替導線**（メール/画面内通知）を案内する。
8. **FR-SET-08** プロフィール編集は本画面の責務外とし、マイページへリンクで誘導する（IA 分離）。
9. **FR-SET-09** 設定変更は R2/ユーザープロファイル JSON に **追記または PATCH 許可フィールドのみ** 更新する（秘密値の平文ログ禁止）。
10. **FR-SET-10** UI 文言に「未実装」「WIP」をユーザー向けに表示しない（`.cursor/rules/no-user-facing-unimplemented.mdc`）。
11. **FR-SET-11** **表示言語（`locale`）** を設定画面から変更できる。選択肢はオンボーディング（§03）と同一集合とする（`21-翻訳-言語.md`）。
12. **FR-SET-12** 言語保存成功後、**製品 UI 全文** が新 `locale` に切り替わる（再ログイン不要が理想 · 詳細設計で確定）。
13. **FR-SET-13** `locale` は **国籍・国（`country`）と独立** — 国変更で言語が自動変更されない。
14. **FR-SET-14** 言語変更はカルマ停止・争い状態に **ブロックされない**（`21-翻訳-言語.md`）。
15. **FR-SET-15** legacy `POST /api/auth/locale` は IHL では **preferences PATCH に統合** する（詳細設計）。
16. **FR-SET-16** **局留め設定**（氏名フルネーム含む）を設定画面で登録・更新できる。**配送には推奨**（§④.5）。
17. **FR-SET-17** **配送先**を設定画面で登録する。参照は **マッチング成立後の取引相手のみ** — 掲示板・公開ボードには露出しない。
18. **FR-SET-18** **銀行振込口座情報**を設定画面で登録する。参照は **取引相手のみ**。プライベートボード・チャットへの **口座直書きを禁止** するガイダンスを表示する。
19. **FR-SET-19** マーケット取引開始前に、上記設定の **完了を促す**（必須化の可否は詳細設計）。未設定時は短い理由付きで設定へ誘導する。
