## ① 機能概要

データ取得元管理は、SwitchBot 等の **環境 IoT** と **物理スロット（Placement）** を結び、測定値を R2 に蓄積する仕組みである。Placement/Binding/Occupancy は **Tier A INSERT ONLY**。**環境テレメトリ（Tier B）** は [`ADR-H-20`](../02-設計/_横断/adr/ADR-H-20-データクラスと書込方針-v1.md) に従い `truth/env/telemetry/v1/.../series.parquet` への **冪等マージ** を許可する（全体が INSERT ONLY 一律ではない · scoped per ADR-H-20）。ローカル collector（Ed25519 署名）は Docker 上で SwitchBot API をポーリングし、秘密値は **R2 に載せず** `collector/.env` と volume のみに置く（REQ-027）。Placement・DeviceBinding・Occupancy・TelemetryIngest の 4 概念（ADR）により、「どの棚・どのデバイス・どの個体」がいつ存在したかを追跡し、固体観測 commit と **単一 POST** で接続する。IHL Phase 0 の append-only・manifest 思想と **in-scope** で整合設計する。

> **v1 UX 原則（2026-06-21 · OBS-RX-*）**: 日常の **個体↔device↔期間** 記録は **観測入力の「環境・設置」チャンク + commit 1 回**で完結する。棚 API（binding/occupancy 単独）は **観測なしの設備変更**・**補正**用。**FR-ENV-06**（環境二重 POST 禁止）維持 — 固体/commit 以外に env 状態を書かない。

| 経路 | 説明 |
|------|------|
| **主** | ユーザー PC 上 Docker / `collector/` 定期 poll → Ed25519 ingest（secret は IHL サーバに載せない） |

- **DeviceBinding**: `deviceId` ↔ `placementId`（時間区間）
- **Occupancy**: `subjectRef`（個体 / annotation 参照）↔ `placementId`（時間区間）
- 交差区間で個体↔device を復元 — 詳細 [`ADR-H-32`](../02-設計/_横断/adr/ADR-H-32-生体-デバイス-期間-紐づけ-v1-DRAFT.md)

**IHL ver1 gap**: DeviceBinding API 未実装（`openBinding: null`）— Occupancy + Placement は API 済。

## ③ スコープ内 / 外

### スコープ内

- ADR 4 概念: Placement / DeviceBinding / Occupancy / TelemetryIngest
- R2 キー `world/env/...`（INSERT ONLY）
- SwitchBot poller + ローカル collector（Ed25519）
- 固体 `SolidEnvironmentSnapshot` との参照接続（ENV-5）
- §7 optional-map（`Placement.meta` に部屋名等 — **幾何マップは任意**）
- env-samples-index 経由の温湿度履歴

### スコープ外

- SwitchBot **秘密キーを R2 やフロントに保存**（fb_00057 議論 — 禁止）
- R2 行の UPDATE/DELETE による Binding 終了表現
- 第三者テナント横断 Placement 共有
- フロアマップ **幾何キャリブ**（§7.4 別 ADR 候補）
- IHL 内の SwitchBot 実装（契約 **移植** は可、実装 duplicate は不可）

## ④ 機能要件（番号付き）

1. **FR-ENV-01** Placement は `placementId` 安定 ID と人間可読ラベル・任意 `meta` を持ち、R2 に **新規 INSERT** のみで作成する（REQ-019）。
2. **FR-ENV-02** DeviceBinding は `startedAt` 必須・`endedAt` は **終了イベント追記** で閉じ、同一 **`(placement_id, device_id, role)`** の未終了 binding 重複は **409** とする（[`05-観測.md`](./05-観測.md) OBS-RX-RD-08 · [ADR-H-33](../02-設計/_横断/adr/ADR-H-33-観測追記-デバイス紐づけ-v1-DRAFT.md) §3.3）。
3. **FR-ENV-03** Occupancy は観測対象の区間を表し、固体 sessionId 等を `subjectRef` で参照可能とする。**v1 正本** `subject_ref = @individual/{individual_id}`（[`05-観測.md`](./05-観測.md) OBS-RX-RD-07）。**v1 運用**（ADR-H-30 §10）では **個体↔温度計↔期間**の区間正本 — **主経路は観測 commit からの自動派生**（ADR-H-33 · OBS-FUP-04）。
4. **FR-ENV-04** TelemetryIngest は `placementId` 未解決行を許容し、事後紐づけは `telemetry.placement.resolved` **新イベント** とする。
5. **FR-ENV-05** QR トークンは **再発行 = 新 INSERT**、期限切れは 404 相当（ADR ENV-3）。
6. **FR-ENV-06** 固体 commit は **環境の二重 POST 禁止** — 棚は Binding/Occupancy、固体は `POST /api/solid-observation/commit` のみ（ENV-5）。
7. **FR-ENV-07** ローカル collector は `X-Civilization-Collector-*` ヘッダで Ed25519 署名検証し、`POST /api/env/collector/ingest` で ingest する（REQ-027）。
8. **FR-ENV-08** R2 env-samples には **測定値 + placement/annotation/QR 文脈** に限定し、SwitchBot token/secret を含めない。
9. **FR-ENV-09** `GET /api/env/history` は期間フィルタ（既定直近 7 日）・最大 400 件/種別とする（ENV-6）。
10. **FR-ENV-10** 将来 IHL collector は **同一 ingest 契約**（canonical JSON 署名・no-overwrite）を再利用可能と設計する（`指示/it-hercules-laboratory/99-アーカイブ/2026.06-06-legacy` 共通 component 契約）。
11. **FR-ENV-11** **デバイス CSV 取り込み（ver1 IN）** — 登録済み `device_id` に対し Export CSV/JSON を **append/diff のみ**で Tier B へマージする。`source` は `switchbot_import`（SwitchBot Hub Export preset）または `csv_import`（汎用 `column_map`）。自然キー `(device_id, bucket_start_unix)` · 1 分粒度 CSV は 5 分バケットへ集約（バケット内最終行 wins）· 応答は `written` / `skipped_duplicate` / `skipped_invalid` / `range`。契約: [ADR-H-35](../02-設計/_横断/adr/ADR-H-35-汎用デバイスCSV取り込み-v1-DRAFT.md) · SwitchBot 運用: [ADR-H-31](../02-設計/_横断/adr/ADR-H-31-SwitchBot-Import-API-v1-DRAFT.md)。

## ⑤ 非機能要件

1. **NFR-ENV-01** 全状態遷移は ProjectRules **R2 INSERT ONLY** に従う（ADR 憲法節）。
2. **NFR-ENV-02** collector キューはローカル `queue.jsonl` append-only。ネットワーク断時も再送可能。
3. **NFR-ENV-03** poller / collector は Docker volume で秘密値を隔離（`collector/.env.example` 参照）。
4. **NFR-ENV-04** 公開 QR GET は Bearer 不要だが、トークンエントロピーと TTL で保護する。
5. **NFR-ENV-05** UI は空棚・409 競合・テレメトリ空を **理由付き** で表示（U-* DoD）。
