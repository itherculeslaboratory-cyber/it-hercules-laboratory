## ①機能概要（平易に3-5文）

UI選択画面改善は、ユーザーが文明 OS の「見え方・導線・OS テンプレート」を自分に合わせて選び、差し替え・編集・フォーク・共有し、投票と使用率で優れた構成が自然に残る仕組みを目指す機能群である。現状は `LayerSelector`（4 レイヤー切替バー）と `WorldRoutingManager`（World Routing Template 管理 UI）が **dev 専用** `/dev/world-routing` にのみ存在し、一般ユーザー向けの正本要件は未整備である。ユーザー oral（fb_00045）は「運営デフォルト 1 つ＋個別差し替え＋C-USB 互換での OS 切替」を想定している。

## ④機能要件（番号付き）

### FR-17-01 World テンプレート選択（将来・AC 未確定）

- 運営が提供する **デフォルト 1 テンプレ**を全ユーザーの初期値とする（fb_00045）。
- ユーザーは使用中に **推奨プリセット**または**自作テンプレ**へ差し替え可能。
- 差し替えは C-USB 必須機能を満たすテンプレに限定する。

### FR-17-02 必須機能チェック

- `WorldRoutingManager` の `REQUIRED_FEATURES` リストを正本化し、テンプレ公開時に欠落があれば警告する。
- 必須例: ログイン、観測、マーケット、掲示板、検索、設定、Builder テンプレ作成 等。

### FR-17-03 テンプレートレベル

- `default`（わさお初期）/ `recommended`（中級プリセット）/ `custom`（ユーザー無限追加）の 3 レベルを維持。
- 各テンプレに author・social（like / platinum）メタを付与（現 UI はスタブ数値）。

### FR-17-04 レイヤー選択バー

- `LayerProvider` + `LayerSelectorBar` で L0–L3 を切替。
- 上位レイヤー（L3）ほど完成品組み合わせ・ effort 最小。L0 はコード直編集想定。
- 選択状態の永続化（ユーザー設定 R2）は未実装 — 要件として要定義。

### FR-17-05 投票・自然淘汰との接続

- 優れたテンプレは使用率・プラチナ投票が閾値に達したら**採用候補**とする（fb_00045、Governance §10）。
- レイヤー3（機能・OS 構成）として投票可能（PlatinumCoinRules §2）。

### FR-17-06 一般ユーザー入口（ギャップ・要新規）

- `/dev/world-routing` 以外に、認証後 3 クリック以内で到達できる **UI 選択**画面を設ける（REQ-024 active）。
- `docs/civilization-ui-ux-intent-map.md` の World 行と整合。

### FR-17-07 dev 管理 UI の現状維持

- `/dev/routing` は hybrid 化済み（`DevPage_routing` + `routing_table` スロット）。
- `/dev/world-routing` は WorldRoutingManager のスタンドアロン表示を維持（移行期）。

## ⑥MiniKernel/C-USB上の位置づけ

```
World（テンプレ選択） → FeatureNode（各機能） → Kernel → Component
```

| 要素 | 位置づけ |
|------|----------|
| **World** | 文明全体の現在地。UI 意図マップでは `/home`, `/civilization/engine`, `/dev/world-routing` が World 入口。 |
| **ROUTING_TABLE** | `frontend/src/kernel/registry.ts`。path × featureNode × kernelName のグラフ。 |
| **LayerSelector** | 編集深度の UX 軸。Kernel UUID ルーティングとは独立した Context。 |
| **C-USB** | テンプレ差し替え時も Component の compatibility / security を満たす subgraph のみ許可。 |

**現状分類**: 実装状態 **stub**（dev 管理 UI のみ）。採用 REQ は REQ-024（active・OS 選択 AC なし）。
