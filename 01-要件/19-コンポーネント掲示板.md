## ④機能要件（番号付き）

### FR-19-01 1 正本 1 スレ

- 走査対象: `docs/`・`design/`・`civilization/`・`ui-reference/` 等（`config/file-board-linkage.json`）。
- 新規 `.md` 正本追加時、`--apply` でスレテンプレ生成＋ CSV 行追加。既存スレは上書きしない。

### FR-19-02 レジストリ CSV

- `rag/file_board_registry.csv`: `repo_path`, `board_kind`, `board_thread_rel`, `post_id`。
- 件数は `collectCanonicalMarkdownRelPaths` と一致（2026-05-01: **975** 行）。CI: `file-board-registry-sync.test.ts`。

### FR-19-03 掲示板区分

- **development** (`boards/.../dev/`): 仕様・憲法・設計。
- **manual** (`boards/.../manual/`): `ui-reference/`, `05-運用/manual.md`。
- `design/V`・`B`・`X`・`G` 正本は legacy-vbgx 誘導をテンプレに自動挿入。

### FR-19-04 RAG 取り込み

- `file_board`: レジストリ行をチャンク化（`ragService.load()`）。
- `file_board_thread`: 優先パス（`file-board-rag-priority.json`）のスレ全文（最大約 14k 字）。
- 全スレ本文の一括埋め込みは既定オフ（CI 負荷）。

### FR-19-05 legacy C-Sync 接続（civ-os のみ · IHL 不採用）

- legacy 文明更新 = spec / post / commit / R2 の 4 媒体整合（`CivilizationSyncEngine.md` — **参照のみ**）。
- **IHL 正本**: GitHub PR/BOARD + R2 append-only。C-Sync 手順は **salvage しない**。

### FR-19-06 運用キュー

- 骨格: `P2-NEXT-FILE-BOARD-LINKAGE-BOOTSTRAP` 完了。
- 記入: `P2-NEXT-FILE-BOARD-STUB-FILL` 完了（`[x]`）。新規正本は `--apply` で追従。

### FR-19-07 製品「コンポーネント掲示板」UI（将来・未確定）

- コンポーネント UUID / fork 単位で議論スレを開く UI は oral のみ。
- `/improve/forks`・graph editor との役割分担を ADR で決める必要あり。

## ⑥MiniKernel/C-USB上の位置づけ

```
Governance（改善判断） → GitHub（PR/BOARD）+ R2（legacy civ-os: C-Sync 4媒体 — 参照のみ） → Component（実装）
```

| 要素 | 位置づけ |
|------|----------|
| **Component** | C-USB 文明原子。掲示板は Component そのものではなく**変更意図の記録媒体**。 |
| **FeatureNode** | `improve` — `/improve/forks` がフォーク・系譜 UI。 |
| **Kernel** | file-board は P 層 Process（`p15_file_linkage`）の運用インフラ。 |
| **legacy C-Sync** | civ-os フォーク・仕様変更の INSERT ONLY R2 + post — **IHL 不採用** |

**名称整理**: REQ-018 の「掲示板」= 開発用 file-board linkage。製品名「コンポーネント掲示板」≠ 同一物（ギャップ）。
