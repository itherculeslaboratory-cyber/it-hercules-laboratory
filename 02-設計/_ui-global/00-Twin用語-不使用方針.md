# 00 「Twin」用語 不使用方針（ADR メモ）

> **ステータス**: **草案 · 人間レビュー反映 2026-06-07**
> **判断 ID**: H-Twin-01（UI 用語）
> **正本**: [`00-AI-HANDOFF-BRIEF.md`](../00-AI-HANDOFF-BRIEF.md) §1.2 / §12 · `ui-reference/preferences.md` §A
> **適用範囲**: IHL の **ユーザー到達 UI**（本フォルダ `02-設計/_ui-global/` のモックアップ全部 + IHL repo `apps/`）

---

## 1. 結論（先に）

**IHL のユーザー向け UI に「Twin」を出さない。** ホームモック（`ihl-01-nav-home.png`）に出ていた「Twin」カードは **civ-os legacy の混入** であり、誤りとして除去した。代替は中立な **「今日の要約」**（システム説明・最大 3 行）。

---

## 2. 「Twin」とは何か（なぜ出ていたか）

| 項目 | 内容 |
|------|------|
| 出自 | **civilization-os（legacy）の persona / character layer**（`civilization/PersonaLayer.md`・User Twin × System Twin の対話運用） |
| 役割（legacy） | RAG を舞台にした Twin 対話・行動喚起の短文表示。civ-os 固有のキャラクター演出。 |
| IHL での扱い | **不採用**。`00-AI-HANDOFF-BRIEF.md` §1.2「やりたくないこと」に **「Twin / Builder / 経済 / 120 画面 API を同リポジトリに混在させる」** と明記。§12 salvage 表でも Twin は **「持っていかない」** 側。 |
| モックに出た理由 | 初版ホーム生成時、civ-os の司令塔 UI 慣習を引きずり Twin カードを **誤って混入**した（要件・ブリーフの IHL 方針と不整合）。 |

> 注: `ui-reference/preferences.md` §A の「**Twin・説明テキストは最大 3 行**」は **legacy civ-os 由来の表現規範**。IHL では「Twin」の語を使わず、**説明テキスト（今日の要約等）の 3 行ルール**としてのみ適用する。

---

## 3. ADR — IHL UI で「Twin」を使わない

| # | 決定 |
|---|------|
| D1 | IHL の **ユーザー到達 UI に「Twin」という語・キャラクター・専用カードを置かない**。 |
| D2 | ホーム等で行動を促す短文が必要なときは、**中立なシステム文言**（例: **「今日の要約」**「ようこそ」「システム説明」）を使う。**最大 3 行・1 行 1 情報・具体**（preferences §A）。 |
| D3 | persona/対話 UX が将来必要になった場合も、**civ-os Twin の再導入ではなく** IHL repo の ADR で別途設計する（本方針の例外は ADR 化必須）。 |
| D4 | 既存モック・UI 説明 md から **「Twin」表記を除去**し、本方針へリンクする。 |
