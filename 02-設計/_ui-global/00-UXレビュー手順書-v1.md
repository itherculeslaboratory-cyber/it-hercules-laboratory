# 00 UX レビュー手順書 — IHL ウォークスルー

> **ステータス**: **草案 · 人間レビュー待ち**
> **作成日**: 2026-06-09
> **目的**: ブラウザ UX ウォークスルーで **新規登録から全機能遷移**を体験し、**設計意図が伝わっているか**・**モック品質**・**要件・遷移との整合**を人間（ユーザー）が目視で検証する。
> **実装禁止**: 本書は設計レビュー用。`frontend/` `backend/` への実装はゲート 4 点人間確定後（`.cursor/rules/design-before-implementation-gate.mdc`）。

## A. 目的・読み方

### A.1 誰が使うか


| 役割                | 作業                                                                |
| ----------------- | ----------------------------------------------------------------- |
| **人間レビュアー（ユーザー）** | 本手順書に沿ってウォークスルーを操作し、チェックボックスを埋め、§F テンプレに記録する                      |
| **設計 AI / 実装 AI** | レビュー結果を受け取り、遷移 doc・mock・`walkthrough.js` を更新する（本書は **レビュー実施者向け**） |


### A.2 「伝わっている」とは何か

設計意図が **モック + クリック導線**だけで、初見ユーザー（あなた）に理解できる状態。次を満たすこと：


| 観点                      | 根拠                                                      | レビューで見るもの                         |
| ----------------------- | ------------------------------------------------------- | --------------------------------- |
| **主タスク 1 つ**            | `ui-reference/preferences.md` §A · `00-画面一覧-全体像.md` §2  | 1 画面 1 主ボタン · 何をすべきか 5 秒で分かる      |
| **3 クリック以内**            | `preferences.md` §A · `00-遷移マップ.md` §2                  | ホームから主タスク完了までの最短クリック ≤3           |
| **3〜5 チャンク**            | `preferences.md` §A                                     | 情報過多・迷路化していないか                    |
| **空 / loading / error** | `definition-of-done-high-finish.md` U-EMPTY · 画面一覧「状態」列 | 設計上の状態が mock または遷移 doc に言及されているか  |
| **理由表示**                | DoD U-A11Y-MIN                                          | 検索・指摘・取引など主要操作に「なぜ」の短文があるか        |
| **色は意味のみ**              | `preferences.md` §A/B · 画面一覧 §1                         | 装飾的多色化なし · ダークトークン `#0D0D0D` 系    |
| **現在地**                 | `00-遷移マップ.md` §5                                        | パンくず / ヘッダ / ウォークスルー breadcrumb   |
| **観測・好み写真**             | `preferences.md` §C                                     | 色補正なし · 撮影条件テキスト併記                |
| **raw エラー禁止**           | `preferences.md` §A                                     | ユーザー向けにスタックトレース風表示がない（mock 上の文言も） |


### A.3 ウォークスルーの起動


| 方法                 | 操作                                                                                                          |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| **A — 最短**         | `[ux-walkthrough/start-walkthrough.bat](./ux-walkthrough/start-walkthrough.bat)` をダブルクリック → 表示 URL をブラウザで開く |
| **B — npm**        | `cd 指示/it-hercules-laboratory/02-設計/_ui-global/ux-walkthrough` → `npm start` → `http://localhost:3000`                  |
| **C — PowerShell** | `.\start-walkthrough.ps1` またはリポジトリルート `.\scripts\start-ihl-walkthrough.ps1`                                 |


初回は `setup-mockups-link.mjs` が `mockups/` junction を自動作成する。画像が 404 のときは `[ux-walkthrough/README.md](./ux-walkthrough/README.md)` を参照。

**レビュー設定（推奨）**: ホットスポット表示 **ON** · ラベル表示 **ON** · 左サイドバーで画面ジャンプ可。

### A.4 本書の読み方

1. **§B 正本マップ**で「何と照合するか」を把握する。
2. **§C** を Phase 0 → 3 の順に実行（チェックボックス `[ ]` を埋める）。
3. **§D** で機能単位の「あるべき姿」を横断確認。
4. **§E** でモック PNG を目視品質チェック。
5. 問題は **§F** に記録し、機能単位 Go/No-Go を判定。
6. **§G** の既知ギャップは **期待しない**（未配線は欠陥ではなく既知事項）。
