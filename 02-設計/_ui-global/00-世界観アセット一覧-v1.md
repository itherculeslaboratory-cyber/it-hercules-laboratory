## 3. 仕様ルール（全アセット共通）

1. **色補正禁止** — 観測・標本系と同様、意味色のみ（[`ui-reference/preferences.md`](../../../ui-reference/preferences.md)）  
2. **背景透過** — UI 上は `#0D0D0D` 系ダークに載る前提。ブランド・経済アイコンは **ユーザー提供 PNG 正本**（2026-06-26）  
3. **解像度** — PNG は `@2x` を別 id で登録可（例: `econ-pt-coin@2x`）  
4. **参照** — UI 設計 md は **相対パス**で本表の `id` をリンクする  
5. **差し替え** — モックと同様、上書き前に `assets/archive/YYYYMMDD/` へコピー（Batch 7 で手順化）

### 3.1 ライト UI 載せ — 黒背景 PNG ブロッカー（2026-06-26）

現行の `brand-logo-*` · `pt-coin` · `indulgence-token` は **黒（または濃色）矩形背景付き PNG** として配置されている。`app-shell` ヘッダ・`/economy/shop` など **ライトサーフェス**では **黒ボックスのアーティファクト**・バランス不良が発生する（2026-06-26 ユーザー判定: 暫定 polish 不採用）。

**polish 着手前にいずれかを確定すること**:

| 方針 | 内容 | 影響画面 |
|------|------|----------|
| **A. 透過エクスポート** | 同一 id で **アルファ透過 PNG** を再配置（色補正は禁止のまま） | ヘッダ · PT ショップ · 経済表示全般 |
| **B. ダークサーフェス spec** | ロゴ/経済アイコン載せ領域を **専用ダークパネル**（`#0D0D0D` 系）に限定する UI 設計を先に書く | `BrandLogo` · `EconIcon` 配置 · shop カード |

**延期 ID**: `P2-NEXT-DEFER-IHL-BRAND-SHOP-UI` — 正本 [`docs/implementation-deferrals.md`](../../docs/implementation-deferrals.md) · ver 計画 §3.1.2
