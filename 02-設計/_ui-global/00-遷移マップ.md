## 1. トップレベル導線（ホーム = ハブ）

```mermaid
flowchart TD
    HOME["🏠 ホーム（司令塔）"]

    HOME -->|1| OBS["観測 /observation"]
    HOME -->|1| MKT["マーケット /market"]
    HOME -->|1| BBS["掲示板ハブ /board（板選び）"]
    HOME -->|1| PREF["好み /match"]
    HOME -->|1| CONTRIB["貢献度 /contribution"]
    HOME -->|1| SET["設定 /settings"]

    OBS -->|2| OBSDETAIL["個体詳細 + 類似 /observation/:id"]
    OBSDETAIL -->|3| OBSCITE["引用 → マーケット出品 / 論文"]

    MKT -->|2| MKTDETAIL["出品詳細 + 申込 /market/:id"]
    MKTDETAIL -->|3| BOARDP["プライベートボード（Stage 1）"]
    BOARDP --> TRANSFER["GMO 振込案内 /market/.../transfer"]

    BBS -->|愚痴/改善/論文/その他| THREAD["スレッド /board/:category（板ごと・現在板を明示）"]
    THREAD -.板切替 1click.-> THREAD
    THREAD -->|…→指摘| DISPUTE["争い 二人部屋（公開観覧）"]
    MKTDETAIL -.出品指摘.-> DISPUTE

    PREF -->|2| PREFSUM["収束サマリ → 検索に反映"]

    CONTRIB --> SHOP["PT ショップ /economy/shop"]
    SET --> SHOP
    SET --> BUILDER["UIビルダー /builder（層に応じ）"]
```

> 太線 = 主導線（クリック数）。点線 = 文脈リンク（出品 → 指摘で争いへ）。

## 2. クリック深度表（最大 3）

| 到達したい主タスク | 経路 | クリック |
|--------------------|------|:--------:|
| 観測を探す | ホーム → 観測 | **1** |
| 個体を引用する | ホーム → 観測 → 個体詳細 → 引用 | **3** |
| 出品を見る | ホーム → マーケット | **1** |
| 取引を申し込む | ホーム → マーケット → 出品詳細（申込） | **2** |
| 振込案内を出す | ホーム → マーケット → 出品 → （取引成立後）振込案内 | **3** |
| 好みを記録する | ホーム → 好み（タップ） | **1** |
| 投稿する | ホーム → 掲示板ハブ → 板 → スレッド（投稿） | **3** |
| 指摘 → 二人部屋 | ホーム → 掲示板ハブ → 板スレッド → 指摘 | **3** |
| 板を切り替える | スレッド左ナビで別板を選ぶ | **1** |
| 貢献度を見る | ホーム → 貢献度 | **1** |
| 免罪符を買う | ホーム → 貢献度/設定 → PT ショップ（購入） | **2** |

> いずれも **3 クリック以内**。4 クリック以上が必要なら導線を見直す（迷路化サイン）。

## 3. 取引タイムライン（マーケット → 振込）

> 正本: [`../01-要件/06-マーケット.md`](../01-要件/06-マーケット.md) §11.0 / §11.0.1 · [`23-GMO銀行振込判定.md`](../01-要件/23-GMO銀行振込判定.md) §3。

```mermaid
flowchart LR
    S0["Stage 0\nマッチング前\n（公開・出品/入札）"]
    S1["Stage 1\nマッチング後\nプライベートボード（2 週間）"]
    S2["Stage 2\n配送完了\n受取確認 → 評価ウィンドウ"]
    S3["Stage 3\n取引成立\n評価確定 or 1ヶ月自動「良い」"]
    FEE["GMO 振込案内\n8% 貢献費（売り手）"]

    S0 --> S1 --> S2 --> S3 --> FEE
    S1 -.双方合意キャンセル.-> X["記録なし"]
```

> **注意**: 成約／`settled` は **取引成立ではない**。8% 振込案内は **Stage 3（取引成立）後** にのみ表示。

## 4. 争いの分岐（掲示板・マーケット共通の「指摘」）

> 正本: [`../01-要件/11-裁判.md`](../01-要件/11-裁判.md) §3 / §6 · §14（哲学）。

```mermaid
flowchart TD
    POST["投稿 / 出品 / Engagement"]
    POST -->|…→ 指摘（タグ+理由 必須）| ROOM["二人部屋（公開観覧）"]
    ROOM -->|合意| FIX["表示切替（旧版は残す・新時刻が勝ち）"]
    ROOM -->|1ヶ月 合意なし| CLOSE["強制クローズ（双方 未解決累計 +1）"]
    ROOM -.配送完了後 Y03/Y04/Y06.-> VOTE["任意: プラチナ投票（7日・1票=1PT）"]
```

> **通報ボタンは置かない**（指摘 + 対話モデル）。開発者は裁判官ではない（§1.1）。
