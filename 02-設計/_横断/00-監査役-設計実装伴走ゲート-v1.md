## 0. なぜこのゲートが必要か（正直な根本原因）

| 事故 | 何が起きたか | 旧監査の穴 |
|------|-------------|-----------|
| **Batch 8 `[x]`** | pytest smoke 緑で B8-Q-23 完走 · DoD #19 に「GitHub 連携 ✓」 | **テスト存在のみ**確認 · 実装が mock_store フォールバックでも PASS |
| **ADR-H-18 stays** | #06/#07/#16/#17 が `stays ✓` のまま OSS 方針と矛盾 | 設計 doc **存在**は見たが **ADR-H-21 への追従**を検証しなかった |
| **DELEGATED-IMPL-GO** | ログ行追加のみ · C1–C4 未実行 | **名誉システム** — エージェントが DoD 表を読まず `✓` 記入 |
| **監査役 Batch 7** | 4 点設計ファイルの有無 · mock↔FR トレース | **実装が設計主張と一致するか**は対象外だった |

**本ゲートの役割**: キュー `[x]` または DoD `✓` を **設計主張と実装のパリティ PASS** なしでは **BLOCK** する。監査役 = **機械 + チェックリスト**（honor system 廃止）。

## 2. 必須チェック（機能 #NN ごと · C1–C4）

| # | 名称 | 合格基準 | 検証方法 |
|---|------|----------|----------|
| **C1** | 設計主張 | DoD / 詳細設計の **具体主張**（例: 「GitHub SSOT」「salvage-adapt」）が `指示/` に grep ヒット | `design-impl-claims.json` の `design_claims` + `design_grep_paths` |
| **C2** | 実装一致 | 設計が GitHub と言えば `get_mock_store()` が当該 route に **無い** · ADR-H-21 salvage なら `test_stays_verification` **のみ**は不可 | `ihl-design-impl-parity-check.mjs` · 手動: 設計行 + コード行引用 |
| **C3** | テスト主張 | 該当 test が **設計主張を assert**（200 OK のみ不可） | テスト本文 grep · `test_must_assert` 列 |
| **C4** | 両層 README | component 機能は `指示/…/component分解/` **と** `components|libs/…/README.md` | パス存在 + 内容が API 契約と矛盾しない |

### 2.1 エージェント義務（`[x]` / `✓` 前）

> **Skill**: [`.cursor/skills/ihl-design-impl-audit/SKILL.md`](../../../../.cursor/skills/ihl-design-impl-audit/SKILL.md)

```
1. node 指示/it-hercules-laboratory/scripts/ihl-design-impl-parity-check.mjs --feature NN
2. PASS なら設計行・コード行・テスト行を 1 セットで証跡に引用
3. FAIL なら [x] 禁止 · 05-運用/queues/AUDIT-FAIL-<id>.md を起票
4. DELEGATED-IMPL-GO ログに c1 c2 c3 c4 列を追記
```

### 2.2 FAIL = BLOCK

- エージェントは **当該機能の `[x]` を付けない**
- `指示/it-hercules-laboratory/05-運用/queues/AUDIT-FAIL-<feature-id>.md` にギャップを記録
- 修復は **POST-OSS-NN** キュー経由（実装コードは別セッション · 本ゲートは governance のみでも可）
