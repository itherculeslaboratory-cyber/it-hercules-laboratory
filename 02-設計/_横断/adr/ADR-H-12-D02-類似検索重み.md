## 決定

**案 A（v0.2）を採用** — `score = 0.5×embedding + 0.2×color + 0.2×size + 0.1×lineage`。

---

## この数字は何か（普通の日本語）

類似画像検索は「ある個体の写真に **似ているもの**を上位に並べる」機能。「似ている」を **4 つの観点で点数化**し、**重み（どれを重視するか）を足し合わせて 1 つのスコア**にする。重みの合計は **1.0**（= 100%）。各観点のスコアは 0〜1 に正規化してから掛ける。

| 観点 | 重み | 何を見ているか | なぜこの重みか |
|------|:----:|----------------|----------------|
| **embedding（見た目の特徴）** | **0.5（50%）** | AI（DINOv2）が画像から抽出した特徴ベクトルの近さ。形・模様・質感など **総合的な見た目** | 「似ている」の主役。**半分**を占めるのが妥当 |
| **color（色）** | **0.2（20%）** | 体色・班紋などの **色の近さ** | 同種・同系統で色が近いものを拾う補助。**色補正しない**写真前提（[`preferences.md`](../../../../ui-reference/preferences.md) §C） |
| **size（大きさ）** | **0.2（20%）** | 体長・角長など **計測値の近さ** | 個体差（成熟度・サイズ）を反映。研究では大きさが重要 |
| **lineage（血統）** | **0.1（10%）** | 系統・親子・交配の **血統的な近さ**（[ADR-H-11](./ADR-H-11-血統-Cross-設計.md) の Snapshot から） | あれば加点する **隠し味**。血統情報が無い個体でも検索が壊れないよう **控えめ 10%** |

**一言で**: 「**見た目を半分、色と大きさを 2 割ずつ、血統を 1 割**で混ぜて、総合的に似ている順に並べる」。

### 計算イメージ

```text
ある個体 X に似た候補 Y のスコア:
  score(Y) = 0.5 × (XとYの見た目の近さ)
           + 0.2 × (色の近さ)
           + 0.2 × (大きさの近さ)
           + 0.1 × (血統の近さ)
→ score が高い順に並べて上位 K 件を返す
```

### 検索の流れ（重みを使う場所）

```text
metadata filter（種・年・QC 等で絞る）
   → 候補 subset（数十〜数百件）
      → embedding で cosine 類似
         → color / size / lineage で rerank（← ここで上の重み）
            → top-K 返却
```

（アイディア §2・[`05`](./05-観測.md) OBS-IMG・HANDOFF §4 検索順と整合。Phase 1 は subset numpy cosine で十分・FAISS 不要 — HANDOFF §7.3）

---

## なぜ A で、B / C ではないか

| 比較 | 理由 |
|------|------|
| **A vs B**（color 0.2 vs 0.3） | B は lineage を持たない。**血統が価値の中心**（アイディア §1）ゆえ lineage を **明示的に 0.1 入れる A** を採る。color を 0.2 に下げて size と対称にし **説明しやすさ**を優先 |
| **A vs C**（C は emb0.7+size0.3） | C は color も lineage も無く、**色・血統を捨てている**。本システムの価値（色補正なし観測・血統証明）と不整合 |
| **合計 = 1.0** | 重みの正規化が明快（％で説明できる）。A は 0.5+0.2+0.2+0.1=1.0 で過不足なし |

---

## 運用上の注意

- **lineage が無い個体**: 血統 Snapshot が無ければ lineage 項は 0（検索は壊れない）。実効的に embedding/color/size の 3 項で並ぶ。
- **重みは調整可能パラメータ**: 本 ADR は **v0.2 を初期値**として固定。将来 `configs/` の重みファイル（pointer 方式・[ADR-H-04](./ADR-H-04-設計規約-v1.2.md) D-01）で **append-only に版管理**し、再 rerank できる（Truth は画像・特徴、重みは派生計算の係数）。
- **embedding / color / size / lineage の各スコアは派生成果物**（[ADR-H-04](./ADR-H-04-設計規約-v1.2.md) §8・Feature Snapshot / Embedding は真実ではない）。重みを変えても元データは不変。
