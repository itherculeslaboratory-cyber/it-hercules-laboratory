## 1. 版管理データ契約

```yaml
terms_version:
  id: string          # semver 例 2026.06.01
  effective_at: ISO8601
  terms_md_ref: string   # docs/legal/terms-of-service.md@commit
  privacy_md_ref: string
  market_notice_ref: string
  plain_explain_ref: string  # 本音解説（法務文と分離）
```

---

## 2. 版更新フロー（運用 · 人間ゲート）

```text
法務が md 更新 ──▶ terms_version 新規 INSERT（R2）
                        │
            既存ユーザー ──┼── GET /api/terms/reaccept-required
                        │         │
                        │    required=true ──▶ 再同意モーダル（O3）
                        │         │
                        └── PATCH agree_terms + agree_privacy（version 付き）
```

| ステップ | 担当 | AI 可 |
|----------|------|:-----:|
| 条文起草 | 法務 | △ たたき台のみ |
| 版 ID 発行 | 運用 | ✓ |
| 再同意 UI | 設計 | ✓ |
| 最終法務 Go | 人間 | **🔒** |

---

## 3. API

| メソッド | ルート | 内容 |
|---------|--------|------|
| `GET` | `/api/terms/current` | 現行版メタ + 本文 URL |
| `GET` | `/api/terms/reaccept-required` | ログイン済み · 要再同意 boolean |
| `POST` | `/api/auth/agree` | `terms_version_id` + `privacy_version_id` |
