## 2. データ契約（版メタ · 同意イベント · resolver マップ）

### 2.1 規約版メタ（`terms_version`）

| フィールド | 型 | 規約 |
|------------|-----|------|
| `id` | str | 版 ID（例 `draft-2026-06` → 発効後 `LEGAL-TOS-1.0.0`） |
| `is_draft` | bool | 草案フラグ（binding 前は `true` · 現行実装 `true`） |
| `legal_gate` | str | `HUMAN-02-LEGAL`（binding 化に必要な人間ゲート） |
| `effective_at` | ISO8601 | 発効日時（binding 版のみ · 現行は未設定） |
| `terms_md_ref` | str | 条文正本の参照（`docs/legal/terms-of-service.md@commit`） |

- **現行実装**: `terms_content` は `version="draft-2026-06"` · `is_draft=True` · `legal_gate="HUMAN-02-LEGAL"` · 草案 `sections[]` を返す（§3.1）。
- **effective_at / terms_md_ref** は binding 発効時に人間ゲート完了後に付与（将来）。

### 2.2 技術同意イベント（`legal_agree_v1` · append-only · Truth）

`POST /api/v1/legal/agree` は 1 件を **append-only** で追記:

| フィールド | 値 | 規約 |
|------------|-----|------|
| `schema` | `legal_agree_v1` | schema-pack 整合 |
| `agree_id` | `agree_{token_hex}` | `os.urandom(6).hex()`（一意 ID） |
| `actor_id` | str（`u_demo` 既定 · 本番は hash 化前提） | PII 最小化（Security.md） |
| `terms_version` | str | 同意した版 ID |
| `is_draft_terms` | bool（`true`） | 草案への同意である旨（binding ではない） |
| `legal_gate` | `HUMAN-02-LEGAL` | binding 確定が人間ゲートである記録 |

- 保存先: `default_event_root() / "legal" / "v1" / {actor_id} / {agree_id}.json`。
- **append-only**（#00 土台規約 · UPDATE/DELETE 禁止）。同一 `agree_id` パス既存は 409 `ALREADY_AGREED`。
- **重要**: 本イベントは **技術的な同意操作の記録**であり、**binding 条文への法的同意の効力は HUMAN-02-LEGAL 完了後の版にのみ生じる**（`is_draft_terms=true` がそれを明示）。

### 2.3 動画 resolver マップ（`terms-media-resolver-map` v1 · 設計契約）

規約本文・UI には **`section_id` のみ**を埋め込み、YouTube URL は別マッピングで解決する（要件 §12.3 · NFR-VIDEO-01 / NFR-RESOLVER-01）:

| フィールド | 説明 |
|------------|------|
| `terms_version` | 適用版 ID |
| `section_id` | 条/改定イベントの安定 ID（本文埋め込みキー · 例 `tos-art-01`） |
| `media_kind` | `explain`（条意図）/ `revision_why` / `revision_what` |
| `youtube_url` | 解決先 URL（本文には出さない） |
| `effective_from` | 当該 URL の有効日時（改定は版切替日と一致） |

- **設計のみ**（IHL 実装は未配線 · RTM `status=gap`/`deferred`）。版変更時はマッピングのみ更新し条文 Markdown は不変（FR-RESOLVER-02）。

## 3. API / route 契約（IHL 実エンドポイント）

> 破壊的変更は CR + バージョン。現行 prefix なし（`@app.get`）。

### 3.1 `GET /api/v1/terms`（FR-10 閲覧 · FR-01 草案正本 · FR-LEGAL-02 版）

| 項目 | 契約 |
|------|------|
| 認証 | 不要（未認証で全文閲覧可 · FR-10 / FR-UX-03） |
| 200 応答 | `{ "version": str, "is_draft": bool, "legal_gate": "HUMAN-02-LEGAL", "sections": [{id,title,body}] }` |
| 草案表示 | `is_draft=true` のとき UI は「草案・binding ではない」旨を表示（NFR-02 未実装表記禁止と両立 — 事実ベース） |

### 3.2 `POST /api/v1/legal/agree`（FR-13/14 同意永続化 · FR-21）

| 項目 | 契約 |
|------|------|
| body | `{ "terms_version": str, "actor_id": str = "u_demo" }`（`LegalAgreeBody`） |
| 副作用 | `legal/v1/{actor_id}/{agree_id}.json` を append-only 追記（§2.2） |
| 成功 201 | `{ "status": "agreed", "agree_id": str, "terms_version": str }` |
| 重複 409 | `detail = "ALREADY_AGREED"`（同一 agree_id パス既存 · 理論上の衝突） |

### 3.3 エラーコード規約（共通）

| 種別 | HTTP | detail | 要件 |
|------|------|--------|------|
| 同意重複 | 409 | `ALREADY_AGREED` | FR-14 |
| （将来）規約取得失敗 | 404/503 | 運用アラート（法務ゲート） | 遷移設計 §4 |

### 3.4 二層表示契約（法的版 / やさしい版 · FR-LEGAL / FR-EASY）

| 項目 | 契約 |
|------|------|
| 第一表示 | **法的版（binding 想定）**を既定タブとする（FR-LEGAL-01） |
| 免責固定文 | 「契約上適用されるのは法的版のみ。やさしい版・解説動画は説明用で binding ではない」（FR-EASY-03 · P0） |
| 対応関係 | やさしい版の各番号 → 法的条番号を明示（FR-EASY-02 · 例「これは第5条です」） |
| binding 範囲 | binding は **法的版条文のみ**。やさしい版・動画は非 binding（条項マトリクス §22） |

- **現行実装**: 単一 `sections[]`（草案）。二層タブ・やさしい版本文は **設計確定 · 実装は法務 binding 後**（RTM `status=review`/`deferred`）。

### 3.5 動画 resolver 契約（FR-VIDEO / FR-RESOLVER）

- UI が `section_id` + `media_kind` + 表示中 `terms_version` を resolver に渡し `youtube_url` を取得（条文本体は不変）。
- 解決 API 案: `GET /api/terms/media-map?version={terms_version}`（FR-RESOLVER-04 · 既存 `video_routing` の ad-hoc 拡張は避ける）。
- **IHL 実装は未配線**（設計契約のみ · RTM `status=gap`）。

## 4. 状態機械（規約閲覧 · 同意 · 版更新）

### 4.1 同意イベントのライフサイクル

```
(no agree) --POST /legal/agree(version,actor)--> (agreed: legal/v1/{actor}/{agree_id}.json)
(agreed)   --同一 agree_id パス既存----------> 409 ALREADY_AGREED（append-only · 上書き禁止）
(agreed)   --版更新(version' 発効)-----------> 既存同意は不変 · 新版に対し新 agree イベント
```

- **不変条件**: 同意イベントは追記のみ（訂正も新イベント · ProjectRules INSERT ONLY · NFR-05）。
- **版更新時**: 旧版同意は残存。新版 binding 化（HUMAN-02-LEGAL）後に再同意が必要か否かは将来 ADR（FR-RESOLVER / `agreedTermsVersion` · §7 P4）。

### 4.2 クライアント遷移（[`遷移設計-v1.md`](./遷移設計-v1.md) と整合）

```
[idle] --GET /terms--> [viewing]（loading→条項 or error+再試行）
[viewing] --法的版/やさしい版 タブ--> [viewing(layer)]（binding は法的版のみ · 免責固定文）
[viewing] --各条 解説動画リンク--> [external/expand]（resolver 解決 · 本文不変）
[register画面 O2] --チェックON+はじめる--> POST /legal/agree（or #01 register agree_terms）
```
