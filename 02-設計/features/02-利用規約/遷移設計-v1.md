## 2. クリックパス

### 2.1 オンボーディング経由（主導線）

```text
/login (O1) ──magic link──▶ /register (O2)
                                │
                    規約リンク ──┼──▶ /terms (O3) ──戻る──▶ O2
                                │
                    チェックON + はじめる ──▶ / (ホーム)
```

| Step | 操作 | 次状態 | 必須条件 |
|:----:|------|--------|----------|
| 1 | O2 で「利用規約に同意」リンク | O3 表示 | — |
| 2 | O3 スクロール閲覧 | — | 条項全文表示 |
| 2' | 各条見出し横 **〔本音解説〕** または **解説動画** | O3 内展開 or 外部動画 | FR-16 · 小学5年生向け平易層（`docs/legal/` とは別） |
| 3 | O3「戻る」or ブラウザ戻る | O2 | — |
| 4 | O2 チェック ON +「はじめる」 | ホーム | `agree_terms` + `agree_privacy` API |

### 2.2 ログイン前ゲート（IHL 案）

| Step | 操作 | 次状態 |
|:----:|------|--------|
| 1 | `/login` で規約未チェック | 送信ボタン disabled |
| 2 | 規約リンク → `/terms` | O3 |
| 3 | 同意チェック ON | magic link 送信可 |

> legacy civ-os との差: IHL は **登録時 API 同意必須**を正とする（[`02-利用規約.md`](./02-利用規約.md) §② 補足）。

## 3. 状態（必須）

| 状態 | 表示 | 導線 |
|------|------|------|
| loading | 条項スケルトン | — |
| error | 「規約を読み込めませんでした」+ 再試行 | ホーム or 登録へ |
| 未同意（O2） | 主ボタン disabled + 理由 | 規約リンク |

---

## 4. エラー・権限

- 条項取得 404 → 運用アラート（法務ゲート 🔒）
- 既同意ユーザーの `/terms` 直アクセス → 閲覧のみ（再同意不要 unless 版更新）

---

## 5. UX 制約

- **3 クリック以内**: 登録画面から規約閲覧→戻る→同意 = **2 クリック**
- **1 主ボタン**: O3 は「同意して続ける」または「戻る」のみ（副はリンク）
- **色は意味のみ**: 同意可 = 緑 `#5CD68D`
