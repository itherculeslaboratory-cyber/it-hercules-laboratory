## 3. チャンク構成（3〜5）

| # | チャンク | 内容 |
|---|----------|------|
| 1 | **左ナビ** | 観測 / マーケット / 掲示板 / 好み / 貢献度 / 設定 + **血統 / 論文 / Builder**（9 行先） |
| 2 | **要約カード** | 貢献度 / 観測セッション / 進行中の取引 / 未読の指摘（4 カード · 1 秒で要約） |
| 3 | **主 CTA（2 つ・同等サイズ）** | 〔観測をはじめる〕→ **計測入力** · 〔検索〕→ **検索グリッド**（観測≠検索を分離） |
| 3b | **文脈バー** | ヘッダ右: 愚痴 / 改善提案 / 画面テンプレ / Builder（[ADR-H-14](../02-設計/_横断/adr/ADR-H-14-グローバル文脈バー.md)） |
| 4 | **今日の要約** | 中立なシステム説明（**最大 3 行・1 行 1 情報・具体**）。例: 「観測セッション 38 件・未読の指摘はありません。」「進行中の取引が 2 件あります。」「まず観測から始めると、マーケットに反映されます。」 |

> **「Twin」は不使用**（civ-os legacy の persona layer。IHL ユーザー UI には出さない — [`00-Twin用語-不使用方針.md`](./00-Twin用語-不使用方針.md)）。チャンク 4 は中立なシステム文言とする。
> ナビは **5〜7 行先**（[`00-遷移マップ.md`](./00-遷移マップ.md)）。要約カードは数値のみ、深掘りは各画面へ。

---

## 4. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | カードはスケルトン表示 |
| empty（新規ユーザー） | 数値 0 + 「まず観測から始めましょう」+ 主 CTA を強調 |
| error | 「読み込めませんでした」+ 再試行（raw エラー非表示） |

---

## 5. デザイン規範

- 色は意味のみ（貢献度の上昇のみ緑、その他はグレー基調）。
- 観測開始と検索は **別ボタン**（主タスクは「次の行動を選ぶ」1 つ · CTA は 2 択まで）。
- 写真はホーム要約に置かない（§C）。
- キーボード: Tab でナビ移動・Enter で実行。
