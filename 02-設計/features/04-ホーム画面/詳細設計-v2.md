## 0. この文書の位置づけ

本書は **要件 #04（ホーム画面）の `H-*` / `NF-H-*` を「実装と検証で追える設計」に厚くした v2** である。

- **重要な読み替え（IHL rebuild）**: 要件 #04 は legacy civilization-os（`HomePage.tsx` · `GET /api/home/dashboard` · 司法インボックスプレビュー · 環境 due · Twin ツアー · REQ-024 IA）を母体に書かれている。IHL では「stays in civilization-os」を **IHL rebuild（legacy = salvage 参照）** と読む。**現行の実装正本は Python FastAPI**（`apps/api/main.py` の `home_summary`）であり、legacy TS / リッチダッシュボードは **設計意図の salvage 参照**として扱う。
- **ホームの本質**: 認証後の **World 着地 = read-only サマリーハブ**。詳細操作は各 FeatureNode に委譲し、ホームは「今日の状態 + 次の一手（観測 CTA）」のみを置く（要件 §①・⑥）。IHL dev では `home_summary` が today_lines + cards + primary_cta を返す最小サマリーとして表現する。
- v2 の追加価値は **§1 境界の IHL 確定**・**§3 IHL route 契約（home/summary）**・**§4 カード/CTA データ契約**・**§6 NFR（read-only · 非 PII · フォールバック）**・**§7 retrofit 差分（legacy リッチダッシュ ↔ IHL サマリー）**の厳密化にある。

## 2. データ契約（サマリー DTO）

### 2.1 `home_summary` 応答 DTO（read-only · 非 PII）

| フィールド | 型 | 規約 |
|------------|-----|------|
| `today_lines` | str[] | 今日の方針 3 行（1 行 1 情報 · 観測色補正なし等 · NF-H-01） |
| `cards` | object[] | 入口カード（`id` · `label` · `value` · `href`） |
| `primary_cta` | object | `{ label, href }`（観測をはじめる → `/observation`） |

- **cards**（固定 4 · 入口地図）: `obs`（観測キャプチャ · value=観測件数 · `/observation`）· `paper`（進行中の論文 · `/board/paper`）· `market`（取引中 · `/market`）· `vote`（公開投票 · `/vote`）。
- **非 PII**: 集計値のみ（他ユーザー個別データを出さない · NF-H-07）。

### 2.2 観測件数の解決（フォールバック）

```
resolve_data_sources() → parquet あり → count_captures(parquet) → obs_count
   parquet なし or 例外 → obs_count = "—" / "3"（近似 · raw エラー非表示 · NF-H-03）
```

## 3. API / route 契約（IHL 実エンドポイント）

### 3.1 `GET /api/v1/home/summary`（H-010/014/020/030）

## 4. 状態機械（ホーム表示ライフサイクル）

### 4.1 サマリー取得

```
[認証後着地(#01/#03)] --GET /home/summary--> [home(today_lines + cards + primary_cta)]
[home] --primary_cta--> /observation（#05 · 1 クリック · NF-H-02）
[home] --card href--> 各 FeatureNode（read-only リンク · 画面内完結させない · H-021）
[data source 不在] --> obs_count 近似（フォールバック · エラー非表示 · NF-H-03）
```

- **read-only**: ホームは複数 Kernel 出力の閲覧ハブ（編集・fork なし · 要件 §⑥）。

### 4.2 クライアント遷移（[`遷移設計-v1.md`](./遷移設計-v1.md) と整合）

```
[loading] --200--> [home] --CTA--> [observation]
[loading] --error--> [home(fallback)]（近似値 + 代替導線 · 全機能/インボックス直接）
```
