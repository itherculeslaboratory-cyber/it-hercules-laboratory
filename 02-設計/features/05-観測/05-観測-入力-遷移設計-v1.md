## 0. 初回入場のコンテキストシート（OBS-CTX / OBS-TGT · first entry）

入力画面に **コンテキスト未設定で入った初回**（URL クエリ無し · localStorage 無し · profile 無し）は、対象・段階が空のまま記録すると後で迷うため、**05ctx ボトムシート（対象ナビゲータ）**を軽く促す（強制ではない）。

```text
[enter_input]
   │ target/stage を URL > localStorage > profile で解決
   ├─ 解決できた ─────────────▶ [edit_fields]（ヘッダ プリフィル済）
   └─ すべて空（初回）
        │ 文脈ヒント帯「対象・段階を選ぶと入力が速くなります」〔対象を選ぶ〕
        ▼
   [context_sheet (05ctx · 対象ナビゲータ)]
        │ ① ドメイン → ② ナビ（生物は亜種まで）→ ③ 確認・適用（任意「プロフィールにも保存」）
        │   └ スキップ可（空のまま記録も許可）
        ▼
   [edit_fields]（?target_id=&domain=&stage= でヘッダ プリフィル · localStorage 更新）
```

| from → to | トリガ | ガード / 副作用 |
|-----------|--------|-----------------|
| `enter_input → edit_fields` | コンテキスト解決成功 | ヘッダ 対象/段階/フェーズをプリフィル（OBS-CTX-01 · OBS-TGT-08） |
| `enter_input → context_sheet` | コンテキスト全空（初回）で〔対象を選ぶ〕 | シートは **割込み**・入力途中値は破棄しない |
| `context_sheet → edit_fields` | 〔適用〕 | localStorage 更新 · 任意 profile 保存（ADR-H-15 §6）。生物は **亜種まで or 未区別明示**（OBS-TGT-04） |
| `context_sheet → edit_fields` | 〔スキップ〕 | 空のまま記録可（対象は入力中に確定 · OBS-CTX-02） |

> **非強制**: コンテキストは記録の前提ではなく **入力を速くするプリフィル**。スキップしても保存でき、対象・段階は入力画面で確定する（OBS-SOL-04 · OBS-CTX-02）。**令（instar）・性別・採卵日は入力画面の属性**（対象選択 05ctx には置かない · ADR-H-16 §6）。

## 1. メイン状態機械（観測計測入力）

```text
[select_template] ──選択──▶ [edit_fields] ◀──────────────┐
   │  (空のまま開始)            │                          │
   └──────────────────────────▶│                          │
                                │ add_row / 値編集         │ (編集継続)
                                ▼                          │
                          [add_row] ──確定──▶ [edit_fields]┘
                                │
                                │ 計測方法=IoT かつ 機器未登録
                                ▼
                        [device_required] ──登録/戻る──▶ [edit_fields]
                                │ (機器管理へ)
                                ▼
                        （/settings/device · 別画面）
                                │ 戻る
                                ▼
                          [edit_fields]
                                │ 保存
                                ▼
                        [save_measurement] ──INSERT ONLY──▶ [saved]
                                                              │
                          （任意・テンプレ派生）             │
                                ▼                            │
[select_template] ──複製──▶ [fork_template] ──保存──▶ [edit_fields(child)]
```

### 1.1 状態定義

| 状態 | 意味 | 主アクション | UI / 状態表示 |
|------|------|--------------|---------------|
| `select_template` | テンプレ一覧（雄/雌/幼虫…）から選ぶ or 空で開始 | 選択 / 複製 / 空開始 | 一覧カード · **empty**「テンプレを選ぶ or 空で項目追加」 |
| `edit_fields` | 計測行の編集（項目/数値/単位/方法） | 値入力 · 行追加 · 性別トグル | 行スタック · **loading** スケルトン |
| `add_row` | 行追加（候補 DD / ＋自由入力） | 項目選択 or 自由入力 | DD ＋「＋ 自由入力」 |
| `device_required` | IoT 選択 × 機器未登録 | 〔機器管理へ〕 / 手入力へ戻す | 注意バナー（`#FFD66B`） |
| `save_measurement` | 保存検証（必須・単位） | 自動 | スピナ · **error** 行ハイライト |
| `saved` | measurement 行 INSERT 済 | 続けて記録 / 終了 | 保存トースト |
| `fork_template` | 既存テンプレ複製 → 編集 | 複製 → 編集 → 保存 | 複製パネル（3 クリック） |

### 1.2 許可辺表

| from → to | トリガ | ガード |
|-----------|--------|--------|
| `select_template → edit_fields` | テンプレ選択 / 空開始 | — |
| `select_template → fork_template` | 複製 | 親テンプレ存在（自他問わず複製可） |
| `fork_template → edit_fields` | 保存 | `TemplateForkEvent` INSERT（ADR-H-04 §7） |
| `edit_fields → add_row` | 〔行を追加〕 | — |
| `add_row → edit_fields` | 項目確定（候補 or 自由入力） | 自由入力時 source_type=human_added（ADR-H-13 §5） |
| `edit_fields → device_required` | 計測方法=IoT 選択 | `requires_device=true` かつ 機器未登録 |
| `device_required → edit_fields` | 登録完了 / 手入力へ戻す | — |
| `edit_fields → save_measurement` | 〔保存〕 | 必須項目・単位充足 |
| `save_measurement → saved` | 検証成功 | INSERT ONLY（新 measurement_id） |
| `save_measurement → edit_fields` | 検証失敗 | 該当行ハイライト・raw 非表示 |

> **不正遷移**: 単位欠落・数値型不一致 → `save_measurement` から `edit_fields` に戻し該当行を強調。保存衝突は新 id 発番で回避（INSERT ONLY）。

## 2. 性別変更 → 項目セット更新（sex change → field set refresh）

性別トグル切替は **既定表示の再計算のみ**（データは破棄しない · [ADR-H-13](./ADR-H-13-観測計測テンプレ契約.md) §D6）。

```text
[edit_fields(sex=male)]
   │ 性別トグル 雄→雌
   ▼
[refresh_fields]  sex_visibility_rule[female] = [both, female]
   │  ├ male 専用（角長）= 既定で淡色/非表示（値は保持・破棄しない）
   │  ├ female（産卵数）= 既定表示に追加
   │  └ both = 不変
   ▼
[edit_fields(sex=female)]  （入力済みの角長値は warn 付きで保持・任意で再表示）
```

| ケース | 挙動 |
|--------|------|
| 雄→雌 | 角長を淡色化（入力値があれば「雌では既定で非表示（値は保持）」warn） |
| 雌→雄 | 角長を再表示・産卵数は淡色化（値保持） |
| unknown/undetermined | both のみ既定表示・性別専用は「任意で表示」リンク |

> **データ非破壊**: 表示を消しても measurement の入力済み値は捨てない（誤操作復帰可）。保存対象に含めるかは行ごとの可視/有効状態で決まる。

## 3. テンプレ Fork 遷移（3 クリック以内）

```text
[select_template]
   │ ① テンプレ選択（雄 標準計測 等）
   ▼
[fork_template]
   │ ② 〔このテンプレを複製して編集〕  → child = parent の項目リスト複製
   ▼
[edit_fields(child)]
   │ ③ 〔保存〕  → TemplateForkEvent INSERT（parent_template_id → child_template_id）
   ▼
[saved]   （使用時に TemplateUsageEvent · OBS-TPL-14）
```

**クリック数**: ①選択 → ②複製 → ③保存 = **3 クリック**（OBS-NF-03 · ADR-H-13 §6.1）。

## 4. エラー・例外導線

| ケース | 遷移 | UX |
|--------|------|----|
| 必須/単位欠落 | `save_measurement → edit_fields` | 該当行ハイライト・平易文言 |
| IoT × 機器未登録 | `edit_fields → device_required` | バナー + 〔機器管理へ〕（手入力に戻す導線も併記） |
| 自由入力の重複疑い | 留まる | 「似た項目があります」alias 候補提示（採用は append-only・強制しない） |
| 保存衝突 | 新 id 発番 | INSERT ONLY のため上書きしない |
| ネットワーク失敗 | `save_measurement` で再試行 | トースト + リトライ（raw 非表示） |
