## 2. イメージ画面

### 2.1 検索グリッド（filter → grid）

![観測 検索グリッド](./mockups/ihl-05-obs-search-grid.png)

| チャンク | 内容 |
|----------|------|
| **文脈バー 種族チップ** | ヘッダ右〔🪲 ヘラクレス · L3 幼虫〕→ タップで 05ctx ボトムシート（[ADR-H-15](../02-設計/_横断/adr/ADR-H-15-観測コンテキスト.md) §3） |
| **左フィルタ** | 種 / 年 range / 性別 / ステージ / 体長mm / 角長mm / QC / 系統グループ（query whitelist · §⑪.3）。**観測コンテキストで「種」「ステージ」を初期チェック**（§2.3） |
| **検索 + 件数** | テキスト検索・結果件数（126 件） |
| **グリッド** | 4:3 個体画像 + キャプション（体長 / 性別 / QC）。**色補正なし**（§C） |
| **主操作** | 〔類似を検索〕（embedding subset cosine · OBS-IMG-04/05） |

### 2.3 観測コンテキストのフィルタ プリフィル（OBS-CTX · [ADR-H-15](../02-設計/_横断/adr/ADR-H-15-観測コンテキスト.md)）

- 受け口: `/observation?species=<正規名>&stage=<egg|larva|pupa|adult>`。
- コンテキストがあると **左フィルタの「種」「ステージ」を初期チェック**し、結果件数を即反映。
- フィルタはユーザーが自由に外せる（コンテキストは **既定値のみ**・検索の確定ではない · OBS-CTX-02）。
- 優先順 **URL クエリ > localStorage > profile**。空ならフィルタ未選択で全件表示（ADR-H-15 §5）。

### 2.2 個体詳細 + 類似検索（detail → similar）

![観測 個体詳細 + 類似](./mockups/ihl-05-obs-detail-similar.png)

| チャンク | 内容 |
|----------|------|
| **大画像** | 4:3 raw 写真（**色補正なし**） |
| **撮影条件** | 光源 / 機種 / 距離 / 背景 + 「色補正なし」明記（§C・OBS-SOL-06） |
| **計測表** | 体長 / 角長 / 胸幅 / 体重 — 各行に **`直接観測` / `画像由来`** タグ（`value_origin` · OBS-REP-IHL-02） |
| **類似個体** | スコアバー（0.92 等）付き 5 件（rerank · D-02 未決の重みは表示しない） |
| **主操作** | 〔この個体を引用〕（出品・論文・好みへ） |

## 3. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | グリッドはスケルトン・類似はスピナ |
| empty（0 件） | 「条件に合う個体がありません」+ フィルタ緩和の案内 |
| empty（類似なし） | 「類似する個体がまだありません」 |
| error | 「検索できませんでした」+ 再試行（raw 非表示） |

---

## 4. デザイン規範（§C 厳守）

- **色補正禁止** — 明度・彩度・自動補正をしない（観測の真実性）。
- 撮影条件を写真直下に必須併記。
- ダッシュボード要約には写真を置かない（一覧カードで「顔」として使う）。
- `value_origin` を **混同させない**（直接観測と画像由来を別タグ）。
- 種・亜種の **確定は常にユーザー**（候補は根拠提示のみ・OBS-SOL-04）。
