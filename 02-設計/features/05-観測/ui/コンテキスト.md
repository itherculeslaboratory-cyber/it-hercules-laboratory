## 1. 主タスク（1 つ）

**「観測対象（生物 / 器物 / デジタル / 環境 / カスタム）を選び、必要なら段階を添えて、観測画面（検索 05a / 入力 05i / テンプレ一覧 05tl）に引き継ぐ」**。

- **ボトムシート**（全画面遷移にしない）— 入力途中等の作業を失わない。
- **文字のみ**（画像 / サムネイルを出さない · OBS-TGT-02）。
- 主要導線 **3 ステップ以内**: ① ドメイン → ② ドメイン内ナビ（対象を絞る）→ ③ 確認・適用。
- 対象・段階は **既定値（プリフィル）のみ**。観測の確定は入力画面でユーザーが行う（OBS-CTX-02 · OBS-SOL-04）。

## 2. 3 ステップ UX（ドメイン → ナビ → 確認）

画面下から **約 75% 高さ**のシートがせり上がる。背景は薄暗のオーバーレイ（タップで閉じる）。**ステッパ**（①対象 ②絞り込み ③確認）を上部に出し現在地を示す。

### ステップ① ドメイン選択

| 要素 | 内容 |
|------|------|
| 見出し | 「何を観測しますか？」 |
| ドメインチップ（文字） | 〔生物〕〔器物・無機物〕〔デジタル〕〔環境〕〔カスタム〕（[`observation_target_domain.yaml`](../02-設計/_横断/schema/02-設計/_横断/schema/dictionaries/observation_target_domain.yaml)） |
| 補足 | 各チップ下に 1 行説明（例: 生物=「亜種まで選びます」） |

> ドメインによって②のナビ正本（分類木 / カテゴリ木）と用語が変わる（ADR-H-16 §3）。

### ステップ② ドメイン内ナビ（3 経路タブ · 文字のみ）

選んだドメインに応じて **3 タブ**を出す（ADR-H-16 §4）。**いずれも候補提示で確定しない**。

| タブ | biological | artifact | digital |
|------|-----------|----------|---------|
| **検索** | 学名 / 和名 検索（GBIF/Wikidata テキスト候補） | カテゴリ名・通称 | 作品名・ジャンル |
| **質問で絞る**（Phase 2） | Akinator 式（綱→目→…の選択質問） | 用途・素材・形状 | プラットフォーム・ジャンル |
| **分類 / カテゴリツリー** | 界→門→…→**亜種**（折りたたみ・目以下を主表示） | 容器›皿 等 | ゲーム›ジャンル›作品 |

- **検索結果・ツリー・候補は文字行**（学名 + 和名 + 分類パス）。**画像なし**。
- 行をタップ = その候補を **仮選択**（③へ進む / さらに下位を開く）。

### ステップ③ 確認・適用

| 要素 | 内容 |
|------|------|
| 選択中の対象 | `display_ja` + 分類パス（パンくず文字）+ ドメイン |
| **亜種ステータス（生物のみ）** | 〔亜種まで確定〕 or **〔亜種未区別（種まで）〕** トグル（OBS-TGT-04）。**未到達かつ未区別なしは〔適用〕無効** |
| 段階（生物のみ・任意） | セグメント〔卵 | 幼虫 | 蛹 | 成虫 | 不明〕（[05 §⑪.2](../01-要件/05-観測.md)）。**令（L1〜L3）は入力 05i 側**（ADR-H-16 §6） |
| タグプレビュー | path 由来の構造化タグ（例: `order:Coleoptera` `family:Scarabaeidae` `subspecies:…`）（OBS-TGT-06） |
| **適用** | 1 主ボタン〔適用〕+ トグル「プロフィールにも保存（複数端末）」+ 任意「フェーズ」自由入力 |

> 任意フェーズ（採卵・幼虫管理・羽化待ち等）は **自由テキスト・非確定メタ**（ADR-H-15 §2）。**令・性別・採卵日は対象選択に置かず入力 05i で**（ADR-H-16 §6）。

## 4. 状態（必須 · U-* DoD）

| 状態 | 表示 |
|------|------|
| loading | 検索候補 / ツリー取得中はリストにスケルトン（文字行のみ） |
| empty（候補なし） | 「候補が見つかりません」→ 「＋ 自由入力（カスタム / 種名手入力）」を強調 |
| error | 「候補を取得できませんでした」+ 再試行（raw 非表示）。手入力で続行可 |
| **亜種未到達** | 生物で種までしか選べないとき「亜種が不明なら『亜種未区別（種まで）』を選んでください」+ 〔適用〕無効（OBS-TGT-04） |
| 初回（コンテキスト空） | 入力画面の文脈ヒント帯「対象・段階を選ぶと入力が速くなります」から起動（[入力 遷移 §0](../02-設計/features/05-観測-入力/遷移設計-v1.md)） |
| 非生物 | 段階セグメントは非表示（生物のみ）。ドメイン別の補足行に置換 |

## 5. 適用後の伝播（OBS-CTX-01 / OBS-TGT-08 · [ADR-H-15](../02-設計/_横断/adr/ADR-H-15-観測コンテキスト.md) §5）

〔適用〕で localStorage（key `ihl.observation.context.v1`）の `target`（`ObservationTarget` 参照）+ `stage_name` を更新し、現在画面に応じてクエリを反映する。

| 現在画面 | 反映 |
|----------|------|
| 05i 入力 | ヘッダ 対象・段階・フェーズをプリフィル（`?target_id=&domain=&stage=` または `?species=` 後方互換） |
| 05a 検索 | 左フィルタ「対象（タグ）」「ステージ」を初期チェック・構造化タグ facet |
| 05tl テンプレ一覧 | `target_scope` 合致テンプレを上位表示 +「○○向けテンプレ N 件」（OBS-TGT-07） |

> **境界**: 対象を確定するのは観測ごとに入力画面で行う（OBS-CTX-02 · OBS-SOL-04）。シートはあくまで **既定値の差し替え**。**令・性別は入力 05i**（ADR-H-16 §6）。

---

## 6. クエリ規約（伝播パラメータ）

| パラメータ | 内容 |
|------------|------|
| `target_id` | 確定済み `ObservationTarget` の ULID（未確定下書きは省略可） |
| `domain` | `biological\|artifact\|digital\|environment\|custom` |
| `stage` | 生物の `stage_name`（egg/larva/pupa/adult/unknown） |
| `scope_route` | 文脈バー基準ルート（ADR-H-14） |
| `species`（後方互換） | 旧 種文字列。`domain=biological` の alias として解釈（ADR-H-15 §2） |

- 優先順は **URL クエリ > localStorage > profile**（ADR-H-15 §5）。
- 辞書外・不明値は無視してフォールバック（raw エラーを出さない · OBS-NF-04）。
