## 0. なぜこの画面が「すごく大事」か

ユーザー確定（2026-06-07）: **「血統と交配はすごく大事。丁寧に設計していい」**。
血統画面の価値は **(1) 親子の証明（CrossParent / OffspringAssignment）** と **(2) 世代ごとの成績を一目で再解析できること**。
本改稿は、ユーザーフィードバック（2026-06-07）で **「項目が足りていない」** とされた **令ステージ別の成長・サイズ極値・各種率（詳細一覧つき）・観測へのジャンプ** を画面に追加する。

> 集計値（令別平均体重・最大/最小・各率）は **Lineage / cross_summary Snapshot**（[ADR-H-11](../02-設計/_横断/adr/ADR-H-11-血統-Cross-設計.md) §2・[ADR-H-04](../02-設計/_横断/adr/ADR-H-04-設計規約-v1.2.md) §8）であり **Truth ではない**（Measurement / Observation / life_event から再計算）。画面に「再構築 Snapshot（真実ではない）」を明記する。

## 1. 主タスク（1 つ）と導線

| 項目 | 内容 |
|------|------|
| **主タスク** | この交配（Cross）の **親・子・世代成績** を確認し、必要なら **記録する** |
| **主ボタン（1 つ）** | 〔この交配を記録〕（緑 `#5CD68D`・右上） |
| **主要導線** | ホーム → 血統/交配一覧 →（交配を選ぶ）= **2 クリック**で本画面 |
| **ドリルダウン** | 各率の〔詳細〕→ 一覧（死亡/羽化不全）→ **個体の観測詳細** |
| **個体ジャンプ** | 子個体カード・各一覧行の〔観測詳細 →〕→ `/observation/:id`（[`05`](./05-観測-Streamlit.md)） |

```text
ホーム
  └─ 血統 / 交配一覧
       └─[交配を選ぶ]→ 交配 #CR-2041（本画面）           ← 2 クリック
            ├─[子個体カード〔観測詳細 →〕]→ 観測 個体詳細
            ├─[率カード〔詳細〕]→ 死亡一覧 / 羽化不全一覧   ← 詳細ドリルダウン
            │      └─[行〔観測詳細 →〕]→ 観測 個体詳細
            └─[系統樹を開く]→ 系統樹フルページ（Snapshot・別 PNG 予定）
```

## 2. 情報チャンク（表示順・F 型 / 上→下）

主情報は **6 チャンク**（血統は重要部位ゆえ 5+1。1 チャンク = 1 目的）。

| # | チャンク | 内容 | 由来 |
|---|----------|------|------|
| 1 | **ヘッダ / 主ボタン** | パンくず「血統 › 交配 #CR-2041」+〔この交配を記録〕 | 現在地・主タスク |
| 2 | **親（CrossParent）＋ 交配サマリー** | 親個体カード×N（役割 chip = テンプレ例）／ cross_name・planned_at・executed_at・status | Truth（Cross / CrossParent） |
| 3 | **世代集計（Snapshot）** | **令別 平均体重**表 ＋ **サイズ極値**タイル（最大重量・最大全長・最小全長） | Snapshot（Measurement 集計） |
| 4 | **率（詳細で一覧表示）** | **生体までの死亡率 / 完品率 / 羽化不全率** ＋〔詳細〕リンク | Snapshot（life_event / Observation 集計） |
| 5 | **子個体（OffspringAssignment）** | 子サムネ×N・確度・〔観測詳細 →〕 | Truth（OffspringAssignment） |
| 6 | **系統樹（Lineage Tree）Snapshot** | 祖父母→親→本交配→子（縮小版）。**「再構築・真実ではない」注記**。フル版は別ページ | Snapshot |

> チャンク 6（縮小系統樹）は本画面では **折り畳み / 下部** に置き、**多世代フル系統樹**は別 PNG（[ADR-H-11](../02-設計/_横断/adr/ADR-H-11-血統-Cross-設計.md) §4 行 3「要追加」）へ委譲する。1 画面の密度を 5±1 チャンクに保つため。

## 3. 世代集計（Snapshot）— 追加項目（フィードバック A）

### 3.1 令ステージ別 平均体重（成長テーブル）

| 列 | 内容 | 備考 |
|----|------|------|
| 令ステージ | **初令 / 二令 / 三令 初期 / 三令 後期** | 4 段（ユーザー指定） |
| 平均体重(g) | 当該ステージで記録された Measurement の平均 | `value_origin` 混在時は注記 |
| n | 集計に用いた個体・計測点数 | 母数の透明性 |

- 数値は **Measurement（体重）** を **令ステージ（life_stage / instar）でグルーピング**して平均（Snapshot 再計算）。
- ステージ判定は **Observation（状態）** ＋ **molt / eclosion（life_event = 変化）** に依存（[ADR-H-04](../02-設計/_横断/adr/ADR-H-04-設計規約-v1.2.md) §2）。
- グラフ点は **四角マーカー**（研究寄り・`preferences.md` §B）。色は装飾に使わない。

### 3.2 サイズ極値タイル

| タイル | 内容 |
|--------|------|
| **最大重量** | 当該世代の最大体重（g） |
| **最大全長** | 最大の全長（mm） |
| **最小全長** | 最小の全長（mm） |

- いずれも極値を出した **個体 ID にリンク**（タイル → 当該個体の観測詳細・将来詳細設計で確定）。
- 「再構築 Snapshot（真実ではない）」の注記をセクション直下に置く。

## 4. 率（詳細で一覧表示）— 追加項目（フィードバック A）

3 つの率カードを横並び（意味色のみ）。**いずれも〔詳細〕→ 一覧 → 観測詳細**にドリルダウンできる。

| 率 | 定義（Snapshot） | 意味色 | 詳細一覧 |
|----|------------------|:------:|----------|
| **生体までの死亡率** | 生体（成虫到達など定義段階）に至らず死亡した割合 | 赤 `#FF6B6B` | **〔詳細〕→ 死亡一覧**（必須） |
| **完品率** | 欠損・不全のない「完品」に到達した割合 | 緑 `#5CD68D` | 〔詳細〕→ 完品/非完品一覧 |
| **羽化不全率** | 羽化時に不全（翅・体形不全等）を起こした割合 | 黄 `#FFD66B` | **〔詳細〕→ 羽化不全一覧**（必須） |

> **率の定義（完品・生体・不全の段階境界）は要件確定が必要**（§8 未決）。Snapshot は分母・分子の母数を併記し、定義変更時は再計算で追従する（Truth は life_event / Observation）。

### 4.1 死亡一覧（ドリルダウン・観測ジャンプ）

率カードの〔詳細〕から開く **個体一覧 → 観測詳細ジャンプ**画面。

| 列 | 内容 |
|----|------|
| 個体ID | IND-xxxx（左に赤い状態ドット） |
| ステージ | 死亡時の令ステージ |
| 死亡事由(Event) | `death`（疑い理由を併記可・Template 定義 event_type） |
| 日付 | life_event の `created_at` 相当 |
| 観測 | **〔観測詳細 →〕**（青リンク・`/observation/:id`） |

- 注記: 「**死亡（死亡率）は life_event、状態（alive_status=dead）は Observation**。詳細は各個体の観測へ」（[ADR-H-04](../02-設計/_横断/adr/ADR-H-04-設計規約-v1.2.md) §2 の責務分離を画面で明示）。
- 羽化不全一覧も同型（事由 = 不全種別）。

## 5. ナビゲーション（フィードバック A — 観測詳細へのジャンプ）

- **子個体カード**・**世代集計の極値タイル**・**各率の詳細一覧行** から、いずれも **個体の観測詳細**（[`05`](./05-観測-Streamlit.md) `/observation/:id`）へ **1 クリック**で遷移。
- 逆方向: 観測 個体詳細の **血統タブ**から所属 Cross（本画面）へ戻れる（[ADR-H-11](../02-設計/_横断/adr/ADR-H-11-血統-Cross-設計.md) §6 影響「05 個体詳細に血統タブ」）。
- 写真は **色補正なし**・撮影条件併記（`preferences.md` §C・[`ui-reference/lineage/notes.md`](../../../ui-reference/lineage/notes.md)）。サマリ部（世代集計・率）には写真を置かない（§C）。

## 6. 状態（必須・全経路）

| 状態 | 表示 |
|------|------|
| **loading** | カード枠のスケルトン（親・世代集計表・率カード） |
| **empty（子個体 0 / 集計不能）** | 「まだ子個体の割当（OffspringAssignment）がありません」＋ 記録 CTA。集計表は「データなし」行 |
| **error** | ユーザー向け文言（raw stack 非表示）・再試行 |
| **未ログイン** | 閲覧可・記録は要ログイン導線 |
| **Snapshot 未計算** | 「集計を再計算中（Snapshot）」スケルトン（Truth は不変・再計算で復帰） |
