## 1. 設計方針

**既存 input → confirm フローを維持し、環境・設置 / 次回観測 / 3 データチャンクを実装済。**

| 維持（Phase6 · 実装済） | 追加（v2 · **実装済 2026-06-26**） |
|-------------------------|-------------------------------------|
| 発育フェーズ DD（初令〜生体） | **環境・設置** Card（binding のみ · ingest UI なし） |
| 性別 DD（任意 · unknown 既定） | **次回観測日** Card |
| 計測行 DD + 項目/行/単位「追加」 | **観測個体データ** / **写真追加** / **写真撮影時環境** 3 Card |
| 撮影条件 構造化行（`StructuredRow`） | confirm **環境・設置・次回** サマリー Card |
| カメラプレビュー + 写真選択 | テンプレ **stage interval** プリフィル |
| sire/dam 親個体 DD | sessionStorage draft + 写真 in-memory |
| テンプレ種族束縛表示（OBS-TPL-19） | |
| confirm テンプレ保存（secondary · OBS-TPL-18） | |
| confirm 主導線 → done | |

**撤去・変更しない**: 計測行ごとの IoT 必須バナー（OBS-INPUT-06/07 = ver2 OUT）。環境は **環境・設置チャンク**に集約。

## 2. チャンク構成（5 · OBS-RX-UX-01）

```text
PageColumn + Stack（gap 16–24px · preferences §A）

┌─ Card 1: 観測対象 / コンテキスト ─────────────────────┐
│  種族 · scope · 前回サマリー（QR/continue 時）           │
│  発育フェーズ DD · 性別 DD                              │
│  計測テンプレ選択 + 種族一致バッジ                       │
├─ Card 2: 個体命名 ─────────────────────────────────────┤
│  表示名 · 命名テンプレ · sire/dam DD（既存）            │
├─ Card 3: 【環境・設置】★ NEW ──────────────────────────┤
│  Placement DD · Devices[]（role+device）· □snapshot    │
├─ Card 4: 【次回観測】★ NEW ────────────────────────────┤
│  date picker · テンプレ理由1行 · □今回は設定しない       │
├─ Card 5: 計測行 ───────────────────────────────────────┤
│  行 DD（項目/値/単位/方法）· ＋追加（既存）              │
├─ Card 6: 写真 / 撮影条件 ──────────────────────────────┤
│  写真選択 · カメラ · 条件構造化行（既存）               │
└─ 副次: 〔確認へ〕（Page 内唯一の forward CTA）─────────┘
```

> **Card 数**: 画面上は 6 Card だが **環境+次回は各1行コンパクト**とし、認知チャンクは **5 塊**（対象+命名=1 · 環境 · 次回 · 計測+写真=2）として §A 上限内に収める。

## 3. ワイヤ（ASCII · 1280px 想定）

### 3.1 入力 — 上部（維持 + 前回サマリー）

```text
+----------------------------------------------------------+
| 観測入力                                    [コンテキスト] |
+----------------------------------------------------------+
| ■ 観測対象                                                |
|   Dynastes hercules hercules  (biological)               |
|   [前回: 2令 · 体長 45mm · 2026-05-10]  ← QR/continue のみ|
|   フェーズ [2令 ▼]   性別 [unknown ▼]                    |
|   テンプレ [標準幼虫 ▼]  ✓ 種族一致                       |
+----------------------------------------------------------+
```

### 3.2 入力 — 環境・設置（NEW · OBS-RX-UX-04 · §4.18 スナップショット）

```text
+----------------------------------------------------------+
| ■ 環境・設置                              obs-env-placement-chunk |
|   棚 [ 幼虫棚 A ▼ ]  [棚を追加]                          |
|   設置開始日 [ 2026-06-26 ]  （終了日は入力しない）        |
|   機器 [温湿度 ▼] [SwitchBot リビング ▼]                  |
|   ※ ingest / スナップショット UI は **ここに置かない**     |
+----------------------------------------------------------+
```

- **device DD**: `GET /api/v1/devices` · `display_name` 表示（secret 非露出 · Phase6 §4.4）
- **スナップショット取得**: `GET /api/env/devices/{id}/latest`（Tier B ingest · **サーバ SwitchBot 禁止** · ADR-H-30）
- **StructuredRow** `group=env_snapshot` — 項目候補は温度/湿度のみ · `source=ingest_snapshot` 時 **値ロック**
- **B モデル**: 撮影完了時に温湿度を **撮影条件行へ自動注入しない**（§4.18 OBS-RX-ROW-03）
- **管理リンク**: `/settings/devices`（既存パターン維持）
- **snapshot OFF** でも `devices[]` 宣言は commit に載せ binding 派生可

#### 3.2.1 StructuredRow（OBS-RX-ROW-01/02）

| `group` | 項目候補 | 方法列 | device 行 |
|---------|----------|--------|-----------|
| `measurement` | 計測辞書 + カスタム | 手入力 / IoT | IoT 時のみ |
| `photo_condition` | 照明・アスペクト比・照度レベル等 | 手入力 / IoT（**照度レベルのみ**） | IoT 時のみ |
| `env_snapshot` | 温度 / 湿度 | 手入力 / 機器から取得（表示） | チャンク上部で一括 · binding 機器から `/latest` |

**UIbuilder**: `<StructuredRow group="measurement" row={…} itemChoices={…} />` — 配置は Card 設定で差し替え可（#16 連携）。

### 3.2 旧ワイヤ（参照 · スナップショット行なし版は廃止）

### 3.3 入力 — 次回観測（NEW · OBS-RX-UX-10）

```text
+----------------------------------------------------------+
| ■ 次回観測                                                |
|   次回観測日 [ 2026-07-21 📅 ]                           |
|   ℹ テンプレ: 2令→3令 3ヶ月（上書き可）                   |
|   □ 今回は設定しない                                      |
+----------------------------------------------------------+
```

- テンプレ interval 不一致: プリフィルなし · 小さな注意文のみ（OBS-TPL-19 同型）
- `source=template_default` 時は Card 内に情報色バッジ

### 3.4 入力 — 観測データ 3 チャンク（v2.1 · 2026-06-25）

計測・写真・撮影時環境を **別 Card** に分離（1 Card 内の入れ子を廃止）。

```text
+----------------------------------------------------------+
| ■ 観測個体データ                          obs-chunk-individual-data |
|   StructuredRow group=measurement（体長・体重等）        |
|   [行を追加] [一括取得]                                  |
|   □ 環境スナップショット同梱（写真なし時のみ）            |
+----------------------------------------------------------+
| ■ 写真追加                                obs-chunk-photo-add |
|   [撮影する] [写真を選択]   preview                       |
+----------------------------------------------------------+
| ■ 写真撮影時 環境データ（hasPhoto 時のみ） obs-chunk-photo-env |
|   photo_condition: アスペクト比・色補正・撮影角度等のみ           |
+----------------------------------------------------------+
|              [ 確認へ ]                                   |
+----------------------------------------------------------+
```

**TELEMETRY_NOT_FOUND**: 計測チャンクの環境スナップショット（写真なし + 同梱 ON）のみ · 友好メッセージ + 手入力フォールバック（`formatEnvTelemetryFetchError`）。写真あり時の撮影条件 Card には温湿度 UI なし。

### 3.4 旧ワイヤ（参照 · 単一「計測行」Card 版は廃止）

```text
+----------------------------------------------------------+
| ■ 計測行                                                  |
|   [体長▼] [45    ] [mm▼] [手入力▼]  [＋行] [＋項目]      |
|   ...                                                     |
+----------------------------------------------------------+
| ■ 写真 / 撮影条件                                         |
|   [写真を選択] [撮影する]   preview                       |
|   条件行: [照明▼][LED   ][  ▼]  …                        |
+----------------------------------------------------------+
|              [ 確認へ ]                                   |
+----------------------------------------------------------+
```

**維持**: `MediaDevices.getUserMedia` プレビュー · IoT 行は計測行内 device 引用可（**必須化しない**）· 撮影後の環境は **環境・設置スナップショットのみ**（B モデル · §4.18 · 撮影条件へ自動注入 **禁止**）。

### 3.5 確認（拡張 · OBS-RX-UX-03/08）

```text
+----------------------------------------------------------+
| 観測内容の確認                                            |
|   個体 · フェーズ · 計測一覧 · 写真条件                   |
|   --- 変更サマリー（device/placement 変更時のみ）---     |
|   ⚠ 温湿度: リビング → 寝室 に変更（本観測から有効）      |
|   次回観測: 2026-07-21（テンプレ由来）                    |
|                                                           |
|   [ 観測を確定 ]  ← 主ボタン1つのみ                       |
|   [ この設定をテンプレ保存 ]  ← secondary                 |
|   [ 戻って編集 ]                                          |
+----------------------------------------------------------+
```

## 5. インタラクション規則

| 規則 | 根拠 |
|------|------|
| 1 画面 1 主ボタン（入力=確認へ · confirm=観測を確定） | OBS-RX-UX-03 · preferences §A |

## 6. 状態表示

| 状態 | 環境・設置 | 次回観測 | 計測/写真 |
|------|------------|----------|-----------|
| loading | placement/devices スケルトン | date 無効 | 行スケルトン |
| empty | 「棚未登録」+ 管理リンク | プリフィルなし | テンプレ空状態 |
| gap | snapshot「未取得」· 補間なし | — | — |
| error | devices API 失敗 + 再試行 | — | 行ハイライト |
