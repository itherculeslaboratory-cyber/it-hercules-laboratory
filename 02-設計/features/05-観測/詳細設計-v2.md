| フィールド | 規約 | enum / 既定 |
|------------|------|-------------|
| `measurement_name` | 計測項目（体長→`body_length_mm` · 角長→`horn_length_mm` 等） | 辞書 / 自由入力 |
| `measurement_value` / `_text` | 数値 or テキスト（`value_type` に応じ） | — |
| `measurement_unit` | 単位（候補 or 自由） | — |

### 3.1 `POST /api/captures`（OBS-SOL-01/02/03/07 · OBS-ENV-01/02/05）

| 項目 | 契約 |
|------|------|
| 入力 | `species`（必須）· `sex`/`stage_name`/`view_type`（既定）· `actor_id` · `device_id`/`placement_id`/`include_env_measurements`（任意 env chain） |
| 201 | `{ status: "committed", capture: { capture_id(cap_*), individual_id, ... }, measurements: [...], telemetry_bucket }` |

| 404 | `device_id` 不在（`DeviceNotFoundError`）· `TELEMETRY_NOT_FOUND` |
| 400 | その他 `ValueError` コード |

### 3.2 `POST /api/measurements`（OBS-TPL-04/05/06）

| 項目 | 契約 |
|------|------|
| 入力 | `individual_id` · `capture_id` · `rows[]`（name/value/unit/method/origin）· `from_device_telemetry` + `device_id` |
| 200 | `{ status: "saved", measurement_ids: [...], items: [...] }` |
| telemetry | `from_device_telemetry` 時 device telemetry → iot_switchbot measurements |
| 400 | `DEVICE_ID_REQUIRED` · `TELEMETRY_EMPTY_READINGS` · `MEASUREMENT_ROWS_REQUIRED` |

### 3.3 `/api/v1/observation/*`（検索 · 登録 · 詳細 · テンプレ · 計測）

| route | 契約 | req |
|-------|------|-----|
| `POST /search` | filters は whitelist のみ · `{status, items, total}` · データ無は `empty` | OBS-TAX-01 / NF-04 |
| `POST /upload` | capture register（run_id=api_upload）· 任意 pipeline | OBS-SOL-01 |
| `GET /{capture_id}` | capture detail + similar（locator 有時）· 404 不在 | OBS-IMG-04/05 |
| `GET /templates` | テンプレ LIST | OBS-TPL-16 |
| `GET /templates/{id}` | テンプレ DETAIL · 404 不在 | OBS-TPL-17 |
| `POST /measurements` | 体長/角長→正規名 · direct_observed | OBS-TPL-03 |

### 3.3.1 ver2 追補 — 詳細 Truth 縦持ち · manifest · digest

**読み分け（不変）**: `POST /search` = **`searchable_capture_set` 横投影**（whitelist · `key_measurements` サマリー最大 3 件）。**詳細の代替にしない**。`GET /{capture_id}` = **Truth 縦持ち**（event store）。

#### `GET /api/v1/observation/{capture_id}`

| フィールド | 型 | 説明 |
|------------|-----|------|
| `capture` | object | 横サマリー（検索 enrichment 同等）+ `has_photo` · `image_url` |
| `measurements[]` | array | 全計測行（1〜100+）· `name`/`label`/`value`/`unit`/`method`/`value_origin`/`device_id`/`source` |
| `photo_conditions[]` | array | `item`/`value`/`unit`/`method`/`device_id` |
| `devices[]` | array | `device_id`/`role`/`source`/`linked_measurement_names?` |
| `environment_snapshot` | object \| null | B モデル点 snapshot（写真なし commit 時） |
| `similar[]` | array | locator 有時 · ADR-H-12 |

#### `GET /api/v1/observation/{capture_id}/image`

commit 済み blob（`raw/{capture_id}.jpg` 等）を `image/jpeg` で返す。不在は 404。

#### `GET /api/v1/observation/{capture_id}/reanalysis-manifest`

ver1 最小メタ（OBS-RX-RD-06 · OBS-RX-REP-06）:

`capture_id` · `individual_id` · `observed_at` · `committed_at` · `prior_capture_id` · `devices[]` · `placement_id` · `schema_version` · `clientContentDigest` · `image_path` · `has_photo` · `measurement_count` · `photo_condition_count` · `implementation_hints`

#18 embedding パイプライン出力は **含めない**（ver3 境界）。

#### `clientContentDigest` canonical（OBS-RX-REP-05 · ver2 IN）

IHL commit 用 canonical JSON（**sorted keys · compact** · 画像バイナリ除外）:

```json
{
  "v": 1,
  "species": "…",
  "stage_name": "…",
  "sex": "…",
  "individual_id": "…",
  "prior_capture_id": "…",
  "entry_mode": "…",
  "placement_id": "…",
  "devices": […],
  "rows": […],
  "photo_conditions": […],
  "environment_snapshot": {…} | null,
  "has_photo": true|false,
  "schema_version": 1
}
```

`clientContentDigest = SHA-256(hex)` of UTF-8 canonical string. **サーバが commit 時に計算**し capture イベント + 201 応答に含める。クライアント送信値がある場合は一致検証（不一致は 400 `DIGEST_MISMATCH`）。

## 4. 状態機械（固体 commit + env chain）

```
[capture 入力] --POST /api/captures--> [capture INSERT(cap_*)]
   include_env_measurements?
     ├ false → [committed(measurements=[])]
     └ true  → resolve_device_telemetry
                 ├ device 不在 → 404 DEVICE_NOT_FOUND
                 ├ telemetry 不在 → 404 TELEMETRY_NOT_FOUND
                 └ あり → write_iot_switchbot_measurements(2行 · environment_derived)
                          → [committed(measurements=2 · telemetry_bucket)]
[measurement 追加] --POST /api/measurements--> rows + (from_device_telemetry ? iot行) INSERT
[検索] --POST /search--> whitelist filter → items（whitelist外=400）
[詳細] --GET /{id}--> capture + similar（ADR-H-12）
```

- **INSERT ONLY**（OBS-R2-01/02 · OBS-NF-01）: 修正は新 capture_id / 新 measurement で表現。同一キー上書き禁止。

| `next_observation_source` | △ | `user` \| `template_default` | テンプレプリフィル確定時 | OBS-TPL-23 |
| `measurements[]` | △ | 縦持ち行 | 各行 `source` + `value_origin` 必須 | OBS-RX-RD-01 |
| `photo_conditions[]` | △ | 構造化行 | OBS-PHOTO-01 · OBS-RX-RD-04 | OBS-PHOTO-01 |
| `taxonomy` | △ | user-confirmed only | OBS-TAX-07 | OBS-RX-RD-04 |
| `sire_id` / `dam_id` | △ | `ind_*` | 親個体（既存 UX 維持） | OBS-IND-02 |
| `clientContentDigest` | △ | SHA-256 hex | canonical body | OBS-RX-REP-05 · **ver2 未配線** |
| `schema_version` | ○ | string | 再現性 | OBS-RX-RD-06 |

**後方互換**: 既存単体 `device_id` → `devices[{ role: "temp_humidity", device_id, source: "registry_poll" }]` へサーバ正規化（breaking なし · ADR-H-33 §11）。

### 9.3 `derive_bindings_from_observation()`（**ver1 実装済**）

**配置**: `libs/ihl/observation/derive_bindings.py`（`commit_solid_observation` TX 内から呼出）

**入力**: `ObservationCommitBody` + 現在 open binding/occupancy（`PlacementStore`）

**処理**（INSERT ONLY）:

| 条件 | 派生 |
|------|------|
| `(placement_id, device_id, role)` ≠ open binding | `device.binding.ended`（旧）+ `device.binding.started`（新 · `trigger_capture_id` · `source=observation_commit`） |
| `placement_id` or `individual_id` ≠ open occupancy | `occupancy.ended` + `occupancy.started`（`subject_ref=@individual/{id}`） |
| 宣言が前回と同一 | 新イベントなし（`source=unchanged`） |
| 409 | 同一 `(placement_id, device_id, role)` の **未終了 binding 重複** | OBS-RX-RD-08 |

**出力**（commit 201 応答）:

```json
{
  "status": "committed",
  "capture": { "capture_id": "cap_…", "observed_at": "…", "committed_at": "…" },
  "derived_bindings": [
    { "event": "device.binding.started", "device_id": "…", "role": "temp_humidity", "started_at": "…" }
  ],
  "observation_schedule": { "scheduled_at": "2026-07-21", "source": "template_default" }
}
```

### 9.4 `observation_schedule.scheduled` イベント（INSERT ONLY）

| フィールド | 必須 | 説明 |
|------------|------|------|
| `event_type` | ○ | 固定 `observation_schedule.scheduled` |
| `individual_id` | ○ | 対象個体 |
| `set_by_capture_id` | ○ | 本 commit の `capture_id` |
| `prior_capture_id` | △ | OBS-FUP-02 連鎖 |
| `scheduled_at` | ○ | ユーザー指定次回日 |
| `source` | ○ | `user` \| `template_default` |
| `template_id` | △ | プリフィル元 |
| `stage_at_set` | △ | 例 `L2` |
| `interval_applied` | △ | `{ unit, value }` 監査用 |

**クエリ正本**: individual ごと `committed_at` 降順 **先頭 1 件** = 現在有効次回予定（UPDATE/DELETE 禁止 · §4.17.2）。

### 9.5 計測テンプレ interval（OBS-TPL-22/23）
