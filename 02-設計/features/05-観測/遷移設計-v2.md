## 0. ルート一覧（IHL `apps/web`）

| ルート | 画面 | 役割 |
|--------|------|------|
| `/observation` | 観測ホーム | 分岐 · 検索入口 |
| `/observation/context` | コンテキスト | WorkflowContext 設定（任意） |
| `/observation/input` | **入力** | 5 チャンク · draft 編集 |
| `/observation/input/confirm` | **確認** | 読取サマリー · 主ボタン 1 つ |

### 0.1 クエリパラメータ（入力）

| param | 用途 | entry_mode |
|-------|------|------------|
| `individual_id` | 個体プリフィル | `qr` if `from=qr` |
| `from=qr` | QR 経路 | `qr` |
| `from=continue` | 続ける経路 | `continue` |
| `species` / `target_id` / `scope_route` | コンテキスト（ADR-H-15/16） | `manual` |
| `edit=env` \| `edit=measurement` | confirm から部分編集 | — |
| `prior_capture_id` | サーバ解決 or 明示（続ける） | — |

## 1. メインフロー（ver1 · MVP）

```text
[観測ホーム / ホーム CTA / 個体詳細]
        │
        ├─ QR ──▶ /scan ──▶ /observation/input?individual_id=&from=qr
        ├─ 続ける ──▶ /observation/input?individual_id=&from=continue
        └─ 新規 ──▶ /observation/context?（任意）──▶ /observation/input
                                │
                                ▼
                    [input: draft 編集 · 5 チャンク]
                                │ 〔確認へ〕
                                ▼
                    [confirm: サマリー + 観測を確定]
                                │ POST commit（単一 TX）
                                ├─ 201 ──▶ /observation/done
                                ├─ 400 ──▶ confirm（理由表示）→ 入力へ戻る
                                ├─ 409 ──▶ confirm（binding 競合 · 修正導線）
                                └─ 5xx ──▶ confirm（再試行）
```

**3 クリック例（OBS-NF-03 · OBS-RX-UX-02）**:

1. ホーム today_line タップ → 2. 入力で値確認 → 3. confirm「観測を確定」

## 2. 入力画面状態機械

```text
[hydrate_draft]
   │ URL/localStorage/profile 解決
   ├─ from=qr|continue ──▶ [prefill_from_latest_capture]
   │                         prior_capture_id 設定 · placement/devices/phase プリフィル
   └─ 新規 ──▶ [default_draft]
        │
        ▼
[edit_chunks]  ←────────────────────────────┐
   │  chunk: 個体 / 環境・設置 / 次回観測 / 計測 / 写真   │
   │  部分編集（edit=env 等）も同状態                      │
   │  バリデーション（client）                             │
   ├─ 計測行不足 ──▶ 留まる（行ハイライト）                │
   ├─ 409 競合は confirm/commit 側                         │
   └─ 〔確認へ〕 ──▶ [confirm_readonly]                    │
                        │ 戻る                           │
                        └────────────────────────────────┘
```

### 2.1 状態定義

| 状態 | 意味 | 出口 |
|------|------|------|
| `hydrate_draft` | sessionStorage + query マージ | `edit_chunks` |
| `prefill_from_latest_capture` | GET latest capture · schedule · binding context | `edit_chunks` |
| `edit_chunks` | 5 チャンク編集 | `confirm_readonly` |
| `confirm_readonly` | 読取 + 副次 CTA | commit / 入力戻り |
| `committing` | POST in flight | done / error |
| `done` | 完了表示 | 続ける / ホーム |

### 2.2 チャンク内遷移（環境・設置 · 新規）

```text
[env_chunk_idle]
   │ placement 選択
   ▼
[env_devices_select]  devices[] 追加（role DD + device DD）
   │ □ include_env_snapshot
   ├─ snapshot ON + poll データあり ──▶ [env_snapshot_ready]
   ├─ snapshot ON + gap ──▶ [env_snapshot_gap]（「未取得」表示 · imputed 禁止）
   └─ snapshot OFF ──▶ [env_declaration_only]（binding のみ派生）
```

**禁止遷移**: サーバ secret による live SwitchBot fetch（ADR-H-30）— UI は Tier B 最新 or 手入力のみ。

### 2.3 次回観測チャンク

```text
[next_obs_idle]
   │ テンプレ + stage 一致
   ├─ interval あり ──▶ [next_obs_prefilled]（source=template_default バッジ）
   └─ なし ──▶ [next_obs_empty]
   │ date picker 変更 ──▶ source=user
   │ □ 今回は設定しない ──▶ skip（commit 時 schedule 不発行）
   ▼
[edit_chunks] 続行
```

## 3. QR / 観測を続ける（OBS-FUP-03 · OBS-QR-*）

```text
[QR scan success]
   ▼
GET /api/individuals/{id} + GET latest capture summary
   ▼
/observation/input?individual_id=&from=qr
   │ entry_mode=qr（draft）
   │ prior_capture_id=latest.capture_id
   │ プリフィル: phase · 計測サマリー（body/horn のみ · 過剰禁止）
   │            placement · devices[] · next_observation 未設定（新規指定）
   ▼
[edit_chunks] → confirm → commit（新 cap_* INSERT）
```

```text
[個体詳細 · 観測を続ける]
   ▼
/observation/input?individual_id=&from=continue
   │ entry_mode=continue
   │ （QR と同一プリフィル規則）
   ▼
[edit_chunks]
```

| ガード | エラー |
|--------|--------|
| `prior_capture_id` が他 individual | 400 `PRIOR_CAPTURE_MISMATCH` |
| `prior_capture_id` 不存在 | 400 `PRIOR_CAPTURE_NOT_FOUND` |
| individual 権限なし | 403 |

## 4. confirm → commit 遷移

```text
[confirm_readonly]
   │ 主ボタン〔観測を確定〕
   ▼
[committing]  POST /api/solid-observation/commit
   │
   ├─ 201 committed
   │     capture INSERT
   │     measurements INSERT
   │     derive_bindings_from_observation()
   │     observation_schedule? INSERT
   │     ▼
   │   /observation/done?capture_id=
   │
   ├─ 400 validation（source 欠落 · prior 不正）
   │     ▼ confirm + 「入力を修正」→ input
   │
   ├─ 409 FR-ENV-02（placement, device, role 重複 open binding）
   │     ▼ confirm バナー + device/placement 修正導線
   │
   └─ 5xx / network
         ▼ confirm トースト + 再試行（draft 保持）
```

**副次 CTA（secondary · OBS-RX-UX-03）**:

- 「この設定をテンプレートとして保存」→ POST templates（commit **前後どちらも可** · 現行は confirm 内 · 維持）
- 「戻って編集」→ `/observation/input?edit=…`

## 6. エラー・例外導線一覧

| ケース | HTTP | 遷移 | UX（OBS-NF-04 · OBS-RX-UX-09） |
|--------|------|------|--------------------------------|
| 計測行空 | — | input 留まる | 「計測値を1行以上入力してください」 |
| capture UPDATE 試行 | 405 | — | API のみ（UI に編集なし） |
| prior_capture 不正 | 400 | confirm→input | 理由 + 連鎖リセット提案 |
| binding 409 | 409 | confirm | 競合キー 3 要素表示 |
| telemetry gap | — | input env | 「未取得」· 補間しない |
| secret poll 試行 | — | — | **経路自体存在しない** |
| commit 失敗 | 5xx | confirm | 再試行 · draft 保持 |
| devices 未登録 | — | input env | `/settings/devices` リンク（**計測行 IoT 必須は ver2**） |

## 7. 旧 v1 遷移設計との差分

| 項目 | v1（`05-観測-入力-遷移設計-v1`） | v2 |
|------|----------------------------------|-----|
| スコープ | 計測テンプレ行編集中心 | **フル commit フロー** + env/schedule |
| device_required | 計測行 IoT 必須 | **ver2 OUT** · 環境チャンクへ移管 |
| confirm | 未定義 | **主ボタン 1 つ** + binding/schedule サマリー |
| QR/continue | 部分 | **prior_capture_id** 連鎖確定 |
| ホーム | なし | today_lines upcoming/overdue |
