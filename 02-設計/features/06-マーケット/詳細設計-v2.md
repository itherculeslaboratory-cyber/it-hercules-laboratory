## 0. この文書の位置づけ

本書は **要件 #06（マーケット）の `FR-MKT-*` / §11 取引ステージ を「実装と検証で追える設計」に厚くした v2** である。

- **IHL rebuild（salvage-adapt · ADR-H-21）**: legacy civ-os `/market/*`・`market.ts`・`marketListingStateStore.ts` は **salvage 参照のみ**。IHL では **Python event store + 状態機械**（`libs/ihl/economy/market_state.py`）が実装正本。
- **マーケットの本質**: **非エスクロー P2P マッチング基盤**。OS は資金預かり・配送保証をしない（`mkg_p2p_core`）。透明性は **公開 trade-events（append-only）**＋カルマ連動で担保。
- **取引成立 ≠ マッチング**（§11.0.1）: listing `sold` / auction `settled` 単独では成立しない。**配送完了確認 + 評価確定**（手動 or 1 ヶ月自動「良い」）で成立し、**8% 貢献費クロック**が起算。
- v2 の追加価値は **§3 IHL route 契約（listings/transition/match/transfer）**・**§4 出品状態機械（7 状態・許可辺）**・**§5 取引ステージ機械（0–3）**・**§6 8% transfer/振込コード**・**§7 retrofit 差分（legacy 全チャネル/抽選/優先順 ↔ IHL コア）**の厳密化にある。

## 3. API / route 契約（IHL 実エンドポイント）

### 3.1 `GET /api/v1/market/listings`（FR-MKT-01/12）

| 項目 | 契約 |
|------|------|
| query | `channel`（任意フィルタ） |
| 200 | `{ items: [{ listing_id, title, price_pt, channel, status, listing_state, seller_handle, ... }] }` |
| status | `listed*` → `open` · それ以外は listing_state |

### 3.2 `GET /api/v1/market/listings/{id}`（詳細 + board）

- 200: listing + `board`（trade stage/step/messages/payment_deadline）。404: 不在。

### 3.3 `POST /api/v1/market/listings/{id}/transition`（FR-MKT-02）

| 項目 | 契約 |
|------|------|
| 入力 | `{ to_state, actor_id }` |
| 200 | `{ status: "ok", event }` |
| 409 | 不正遷移 / 未知状態（`InvalidTransitionError`） |

### 3.4 `POST /api/v1/market/listings/{id}/match`（Stage 0→1）

- 入力 `{ buyer_handle }` → `sold` 遷移 + trade Stage 1（private_board）· `{ status: "matched", trade_event }`。409 不正。

### 3.5 `GET /api/v1/market/transfer/{id}`（FR-MKT-07/09）

- `obligor_user_id` から `transfer_code` · 8% `amount_jpy` · stub tier · `awaiting_deposit`。live は `P0-NEXT-GMO-LIVE-EXEC` 人間ゲート。

## 4. 出品状態機械（FR-MKT-02 · 遷移設計 v1 正本）

### 4.1 7 状態・許可辺

```
unlisted → {listed_fixed, listed_auction, listed_offer}
listed_fixed → {sold, delisted}
listed_auction → {sold, delisted}
listed_offer → {offer_review, delisted}
offer_review → {sold, listed_offer, delisted}
delisted → {listed_fixed, listed_auction, listed_offer}
sold → {}（終端）
```

- 許可辺以外は **409**（`InvalidTransitionError`）。`unlisted` への逆行不可（例: listed_fixed→unlisted）。
- INSERT ONLY: 状態は **イベント追記**で表現（上書きしない · FR-MKT-03 · NFR-MKT-02）。

## 5. 取引ステージ機械（§11.0 · 非エスクロー）

```
Stage 0 matching       … 公開（出品・入札）
Stage 1 private_board  … マッチング後 当事者 2 人のみ（振込 2 週間期限） ← start_trade
Stage 2 delivery_confirm … 買い手受取申告（評価ウィンドウ最大 1 ヶ月開始）
Stage 3 completed      … 評価確定 = 取引成立 → 8% クロック起算（§11.0.1）
```

- `advance_trade_stage`: **後退不可**（`to_stage < current` は 409）。
- **成立 ≠ マッチング**: Stage 1（sold/matched）は成約だが取引成立ではない。Stage 3 で成立 + 8% 義務発火（FR-MKT-13）。
