# 07 掲示板 — UI 設計（イメージ画面）

> **ステータス**: **草案 · 人間目視レビュー待ち**
> **正本**: [`../01-要件/07-掲示板.md`](../01-要件/07-掲示板.md) · 指摘入口は [`11-裁判.md`](../01-要件/11-裁判.md) §3

---

## 1. 主タスク（1 つ）

**「スレッドを読み、投稿・引用・指摘する」**。国際 BBS。コンテンツモデレーションは **NG ワード自動削除ではなく、人の指摘 + 対話**（11 §3.1）。

---

## 2. イメージ画面

![掲示板 スレッド](./mockups/ihl-07-board-thread-post.png)

| チャンク | 内容 |
|----------|------|
| **カテゴリ** | 愚痴 / 改善提案 / 論文 / 裁判 / 説明（`menuConfig` · FR-BBS-06） |
| **スレッド** | 投稿カード（アバター / handle / 時刻 / 本文）の縦リスト |
| **投稿アクション** | `…` → **引用 / 指摘**（**通報ボタンは置かない** · 11 §3.2） |
| **投稿欄** | テキスト + 〔投稿〕（主操作） |

> 指摘は **タグ選択（必須）+ 理由テキスト（必須）** が揃って初めて二人部屋が開く（FR-DSP-02 · [`11-争い-二人部屋.md`](./11-争い-二人部屋.md)）。

---

## 3. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | スレッドはスケルトン |
| empty | 「まだ投稿がありません」+ 投稿誘導 |
| error | 「読み込めませんでした」+ 再試行 |
| **投稿失敗（rescue）** | raw error を見せず、失敗理由 + 再試行 + 保存境界を明示（FR-BBS-07 · REQ-024） |

---

## 4. デザイン規範

- 追記文化（INSERT ONLY）— 削除でなく訂正は追記（NFR-BBS-01）。
- 「指摘」は控えめに強調（攻撃的な通報 UI にしない）。
- 国際 BBS は **登録言語** で表示・二人部屋は **クライアント翻訳** 許容（FR-DSP-11 · [`21-翻訳-言語.md`](../01-要件/21-翻訳-言語.md)）。
- ユーザー向け「未実装」文言を出さない（NFR-BBS-04）。

---

## 5. 板サブツリー（投稿先の正しさ）

掲示板は **単一の汎用画面ではなく板サブツリー**。ハブで板（**愚痴 / 改善提案 / 論文 / その他** + 裁判 / 説明）を選び、スレッド画面では **パンくず + 左ナビハイライト**で現在板を常時明示する（投稿先の取り違え防止）。

![掲示板ハブ](./mockups/ihl-07-board-hub.png)
![掲示板 愚痴（タブ active）](./mockups/ihl-07-board-post-愚痴.png)
