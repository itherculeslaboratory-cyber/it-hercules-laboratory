## 3. API / route 契約（IHL 実エンドポイント）

### 3.1 `GET /api/v1/board/categories`（FR-BBS-06/14）

- 200: 4 入口（general/paper/rant/component · 各 id/label/description）。Research Board 独立板なし（ADR-H-07）。

### 3.2 `GET /api/v1/board/{category}/threads`（FR-BBS-05）

- 200: `{ category, threads: [{ thread_id, title, post_count, last_activity, case_chip, ... }] }`。last_activity 降順。

### 3.3 `POST /api/v1/board/{category}/threads`（FR-BBS-05/15）

- 入力 `{ title, actor_id, case_chip? }` → `{ status: "created", thread_id, event }`。論文板は case_chip（FR-BBS-15）。

### 3.4 `POST /api/v1/board/{category}/threads/{id}/posts`（FR-BBS-05 · 追記）

- 入力 `{ actor_id, body }` → `{ status: "posted", event }`。post_count 増（投影）。INSERT ONLY。

### 3.5 `GET /api/v1/component-board`（FR-BBS-10 · #19 連携）

- GitHub BOARD/Discussion URL を component_id ごとに mirror。スレッド無しでも GitHub 項目を列挙（`source: "github"`）。

## 4. 状態・投影（スレッド index）

```
[thread_create イベント] → list_threads が thread index に投影（post_count=0）
[post_append イベント] → 該当 thread の post_count++ · last_activity 更新
[list_threads] → last_activity 降順ソート
[thread_posts] → thread_id の post_append を時系列で返す
```

- **削除なし**（NFR-BBS-01）: 訂正は追記 post。index は **イベント再投影**（projection · 状態を別途持たない）。
