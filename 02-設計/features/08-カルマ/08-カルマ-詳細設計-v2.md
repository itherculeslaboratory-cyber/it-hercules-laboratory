## 1. 二層モデル（値 vs count）

| 層 | フィールド | 更新契約 |
|----|-----------|----------|
| **値（karma_value）** | 連続スコア · Fib 減衰 | 月次バッチ + 争い Δ（11 §6.4） |
| **count（karma_count）** | 整数イベント積み上げ | append-only `karma_event` |

> **統合禁止**（ADR-H-08）: 貢献度・マーケット評価と同一カードに混ぜない。

---

## 2. 月次バッチ UI 契約

| 項目 | 内容 |
|------|------|
| 実行タイミング | 毎月 1 日 00:00 JST（cron · `karma_monthly_batch`） |
| ユーザー表示 | `/karma` に「前回バッチ: YYYY-MM-DD」+ 次回予定 |
| バッチ中 | `batch_running` バナー · 読み取り専用 |
| 失敗 | `batch_failed` · 管理者 runbook リンク（ユーザー向けは「再計算中」） |

### 2.1 API

| メソッド | ルート | 内容 |
|---------|--------|------|
| `GET` | `/api/karma/summary` | value · count · last_batch_at · fee_unpaid フラグ |
| `GET` | `/api/karma/events?cursor=` | append-only イベントページング |
