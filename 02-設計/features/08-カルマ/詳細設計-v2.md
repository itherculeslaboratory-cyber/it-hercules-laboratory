## 0. この文書の位置づけ

本書は **要件 #08（カルマシステム）の `FR-KRM-*` を「実装と検証で追える設計」に厚くした v2** である。

- **二層モデル**（§12 ユーザー確定）: **カルマ値**（[-100,+100] · 信用残高）と **カルマカウント**（count · 未解決争い等の累積）の 2 層。count 増加が値を Fibonacci で削る。
- **Fibonacci ペナルティ**（FR-KRM-03 · §2.3）: count n-1→n でカルマ値 **-Fib(n)**。`fib_tier_delta` が決定的に算出（IHL stub）。
- **回復・BAN**: 毎月 25 日に count≥1 なら count-1（FR-KRM-04）· count=0 の月のみ値 +10（FR-KRM-05）· 値 ≤ -100 で **永久 BAN**（FR-KRM-06）。
- **IHL 現状**: `EconomyStore` が **karma_event（layer=value/count · INSERT ONLY）**・`fib_tier_delta`・`fee_unpaid` 起点 Δcount・`karma_snapshot`・`profile/metrics` を実装。25 日サイクル/BAN 判定/免罪符 count-1 は **gap/部分**（§7 で明示）。
- v2 の追加価値は **§3 IHL route 契約（profile/metrics）**・**§4 karma_event データ契約（二層 · INSERT ONLY）**・**§5 Fib 状態機械**・**§6 NFR（決定的 · BAN 統一）**・**§7 retrofit 差分（25 日/BAN/免罪符 ↔ IHL 実装）**の厳密化にある。

## 2. データ契約（karma_event · 二層 · INSERT ONLY）

### 2.1 karma_event（`economy/karma_event` · append-only）

| フィールド | 規約 |
|------------|------|
| `actor_id` | 対象ユーザー |
| `layer` | `value`（信用残高）/ `count`（累積カウント） |
| `delta` | float（value は ±Fib · count は ±1 等） |
| `reason_code` | `fee_unpaid` / `monthly_recover` / `indulgence` 等 |

- **二層分離**（§12）: count 増 → 値減（FR-KRM-03 Fib）。値直接加減 API は設けない（FR-KRM-08 · 管理者例外 TBD）。

### 2.2 Fib count delta（FR-KRM-03 · §2.3）

```
fib_tier_delta(n): n<1 → 0 · else FIB_SEQUENCE[min(n-1, 9)]
  1→1 · 2→1 · 3→2 · 4→3 · 5→5 · 6→8 · 7→13 · 8→21 · 9→34 · 10→55
```

### 2.3 karma_snapshot 投影

- `value` = 128.0 + Σ(layer=value delta) · `count` = Σ(layer=count delta) · `trend` = count≥0 ? up : down。

## 4. 状態機械（Fib ペナルティ · 25 日回復 · BAN）

```
[count n-1 → n] → karma 値 -Fib(n)（逐次適用 · FR-KRM-03）
[毎月25日] count≥1 → count-1（FR-KRM-04）
[25日サイクル] count=0 の月 → 値 +10（FR-KRM-05 · count≥1 月は付与なし）
[免罪符購入] → count-1（FR-KRM-07 · #22 連携）
[値 ≤ -100] → 永久BAN（ログイン拒否 · アカウント非削除 · FR-KRM-06）
[値クリップ] 値は [-100, +100]（FR-KRM-02）
```

- INSERT ONLY（FR-KRM-09 · NFR-KRM-01）: 全ミューテーションは karma_event 追記。UPDATE/DELETE 禁止。
