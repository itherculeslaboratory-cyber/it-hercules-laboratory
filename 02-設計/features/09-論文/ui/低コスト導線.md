## 1. 目的（このページが守ること）

**研究を前に進める操作を、画面往復ゼロ・下書き消失なしで最短に。** 「論文＝完成品」の重圧を外し、**進行中（in_progress）を一級**として扱う。後から探すための **case チップ**（[ADR-H-07](../02-設計/_横断/adr/ADR-H-07-掲示板-入口4つ-論文内研究.md)）は維持。

---

## 2. 主導線（一本道）

```text
個体 → 撮影(Capture) → 計測/観察(Measurement/Observation) → 交配(Cross)
                                                              ↓ 〔研究を始める〕
                                              研究ノート(research_note · in_progress)
                                                ├─〔ケースを追加〕並行検証（温度A/B/C…）
                                                ├─〔記録を追加〕結果を append（消えない）
                                                ├─〔仮説にする〕hypothesis
                                                ├─〔引用に追加〕Citation（観測/Cross/他論文）
                                                └─〔論文にする〕paper（公開）
```

- **どの研究画面からでも観測ループに戻れる**: 「試す」は新画面ではなく Capture/Measurement に戻るだけ。
- **status 遷移で完結**: research_note → hypothesis → (testing) → paper。**画面を増やさない**。

## 3. クリックバジェット（3 クリック原則）

| タスク | クリック | 経路 |
|--------|:-------:|------|
| 計測 1 件記録 | 2 | 個体詳細 →〔計測を追加〕→ 保存 |
| 研究を始める | 1 | 観測詳細 →〔研究を始める〕 |
| 仮説に進める | 1 | 研究 →〔仮説にする〕 |
| 並行ケース追加 | 2 | 研究 →〔ケースを追加〕→ 条件入力 |
| 結果を記録 | 2 | 研究 →〔記録を追加〕→ append |
| 引用を足す | 1 | 各画面〔引用に追加〕 |
| 論文公開 | 2 | 研究 →〔論文にする〕→ 確認 |

**一周（観測→引用付き進行中論文）**: 約 7〜9 クリック・往復なし・保存不安なし。

## 4. 画面イメージ（進行中論文・テンプレ駆動）

> **フィードバック反映（2026-06-07）**: ユーザー要望「**前提・条件をテンプレ化し、空欄を埋めればほぼ完成形の論文**にしたい。**今この phase** が分かり、**どんな条件で何を検証しているか**が読み手に伝わって、**『自分の環境でも貢献できそう』『条件を合わせてデータを足そう』** と思える形に」。

左に **論文テンプレートの節**（目的 / 仮説 / **条件（温度・湿度・餌）** / 検証したいこと / **現在のフェーズ** / 必要なデータ・ギャップ）、右に **5 ステップ進行**（観測→仮説→試す→記録→引用・現在地ハイライト）と CTA **〔同じ条件でデータを追加する〕**。上部にステータス **「今この phase: 試す」**。下に **〔← 観測キャプチャへ〕〔引用を追加〕**。

- 上部に **case チップ行**（論文/観察/繁殖ログ/分析/査読/追試/仮説/その他）— 後から探す用（[ADR-H-07](../02-設計/_横断/adr/ADR-H-07-掲示板-入口4つ-論文内研究.md)）。

### 4.1 論文テンプレート（穴埋め）

**前提・条件をテンプレ化**し、**空欄を埋めるとほぼ完成形**になる入力画面。各節はラベル + 値（埋め済みは緑チェック・未記入はプレースホルダ）。上部に **完成度メータ**、下部に **〔プレビュー / 論文にする〕** と **〔観測から差し込む（自動）〕**、注記 **「進行中のまま保存（消えません）」**（append-only・[ADR-H-09](../02-設計/_横断/adr/ADR-H-09-研究フロー-低コスト設計.md) §1「下書きという概念を使わない」）。

![論文テンプレート（穴埋め）](./mockups/ihl-09-paper-template-fill.png)

**テンプレ節（構造のみ・Phase 1 / [ADR-H-09](../02-設計/_横断/adr/ADR-H-09-研究フロー-低コスト設計.md) §4「構造のみ Phase 1・装飾は後」）**:

| 節 | 役割 | 読み手への意味 |
|----|------|----------------|
| **目的** | 何を明らかにするか | 研究の射程が 1 行で分かる |
| **仮説** | P⇒Q の予想 | 賛否・追試の対象が明確 |
| **条件（温度/湿度/餌等）** | 飼育・計測条件 | **「自分の環境と合うか」** を即判定 → 参加可否 |
| **検証したいこと** | 検証対象の言語化 | どのデータが効くか分かる |
| **現在のフェーズ** | 今この phase（観測/仮説/試す/記録/引用） | 進行中の一級表示（[ADR-H-09](../02-設計/_横断/adr/ADR-H-09-研究フロー-低コスト設計.md) §1） |
| **必要なデータ / ギャップ** | 不足キー（〔不足〕タグ） | **「自分が足せるデータ」** が見え、貢献導線になる |

> 美学テンプレ（文体スキン）は **後追い**（[ADR-H-09](../02-設計/_横断/adr/ADR-H-09-研究フロー-低コスト設計.md) §4・[`09-論文.md`](../01-要件/09-論文.md) FR-PAPER-04「構造と文体分離」）。Phase 1 は **構造（節・条件・state・ギャップ）** のみ。

## 5. 状態（必須・全経路）

| 状態 | 表示 |
|------|------|
| **loading** | スケルトン（ケース行・タイムライン） |
| **empty（研究 0 件）** | 「まだ研究記録がありません。観測から〔研究を始める〕で起票できます」＋ CTA |
| **in_progress** | 進行中チップ・ケースは追記可・**下書きという概念を使わない**（append-only で消えない） |
| **error** | ユーザー向け文言（raw stack 非表示） |
| **未ログイン** | 閲覧可・記録は要ログイン導線 |
