## 1. 主タスク

**「2 枚の個体画像から好きな方をタップして好みを記録する」**。一般ユーザー向け **好み導線**（`/match` — dev-only ではない・ADR-H-02）。

**副次（2026-06-09 レビュー反映）**: 学習した好みに基づく **表示プレビュー帯**（観測グリッド・マーケット一覧での並び替えイメージ）を同一画面下部に常設。pairwise のみでは機能が薄いとの指摘への対応。

## 3. pairwise 画面ワイヤー（2 枚 + 大ボタン）

```text
┌───────────────────────────────────────────────┐
│  [発見 ●] — [精緻化 ●] — [マッチ ○]           │  ← mch-progress-chip
│  ← おすすめに戻る                    3/10 回  │  ← mch-back-to-rec · ラウンド表示
├───────────────────────┬───────────────────────┤
│                       │                       │
│      ┌──────────┐     │     ┌──────────┐      │
│      │  左 画像  │     │     │  右 画像  │      │  ← 4:3・色補正なし
│      │  (4:3)   │     │     │  (4:3)   │      │
│      └──────────┘     │     └──────────┘      │
│   撮影条件: 光源/機種  │   撮影条件: 光源/機種  │  ← §C 撮影条件併記
│                       │                       │
│      〔 左 〕 ★       │      〔 右 〕 ★       │  ← 大ボタン（主操作・左右対）
├───────────────────────┴───────────────────────┤
│            〔 どちらも × 〕   ・   詳しく ▸     │  ← mch-pair-neither / 副次
└───────────────────────────────────────────────┘
```

> **N ラウンド到達（FR-MCH-UX-07）**: 10 回（§AI 仮定）で `mch-pair-converged` バナー → §5 収束サマリへ自動遷移可。

### 3.1 ボタン

| ボタン | choice | 役割 | スタイル |
|--------|--------|------|----------|
| **〔左〕** | `left` | 左を好む（主操作） | 大・対称（色は意味のみ・装飾多色化しない） |
| **〔右〕** | `right` | 右を好む（主操作） | 大・対称 |
| 〔どちらも ×〕 | `neither` | 両方不採用（`mch-pair-neither`） | 副次・控えめ |
| 詳しく ▸ | — | ValueCheck 詳細オーバーレイへ | テキストリンク（任意） |
| （暗黙）skip | `skip` | 判断保留で次へ | 小・端 |

> 色は **意味のみ**（preferences §A）。緑/赤は「成功/失敗」意味であり **左右の優劣に多色を使わない**（左右は等価選択）。fb_00046 の緑/赤理想は「採用/不採用」の意味づけにのみ適用（どちらも× = 不採用＝赤系の意味は可）。

## 5. 収束サマリ（converged）

```text
┌─ あなたの好み（暫定） ─────────────────────┐
│ 収束度: ▓▓▓▓░ 78%   評価数: 42             │
│ 好む傾向:  大型・太い角・黒系               │
│ 避ける傾向: 小型・短角                       │
│         〔 続ける 〕   〔 検索に反映 〕      │
└────────────────────────────────────────────┘
```

| 要素 | 内容 |
|------|------|
| 収束度 | 直近の profile 変化量から算出 |
| 傾向 | preference_profile の prefer/avoid（短文・1 行 1 情報） |
| 反映 | 検索 rerank の boost へ（重みは H-05 待ち） |

## 6. 状態（必須）

| 状態 | 表示 |
|------|------|
| **loading** | ペア取得スピナ。画像個別ロードはプレースホルダ |
| **empty** | 候補 < 2 → 「まだ評価できる個体がありません」+ 観測導線案内 |
| **error** | 記録失敗トースト + 再試行（楽観的 UI ロールバック・raw 非表示） |
| **未ログイン** | ログイン誘導（profile は本人限定・NFR-MCH-02） |
| **収束** | §5 サマリ |

---

## 7. デザイン規範（preferences.md 整合）

| 項目 | 適用 |
|------|------|
| 画像 | **色補正しない**・4:3・枠 1px `#2A2A2A`・角丸 12px・影なし（§B/§C） |
| 撮影条件 | 画像直下に必須併記（光源/機種/距離/背景）（§C） |
| 色 | 意味のみ。左右ボタンは等価（多色化しない） |
| チャンク | 1 画面の主情報は 2 画像 + 主操作（3〜5 塊以内） |
| 文言 | 「好み」など平易・最大 3 行 |
| アクセシビリティ | キーボード: ←=左 / →=右 / ↓=どちらも× / Enter=次（色だけに依存しない） |
