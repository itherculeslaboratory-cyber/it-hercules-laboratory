## 1. 主タスク（1 つ）

**「2 枚の個体画像から好きな方をタップして好みを記録する」**。ルートは **`/match` = 一般ユーザーの好み**（`/dev` 配下にしない · ADR-H-02）。

---

## 2. イメージ画面

![好み pairwise](./mockups/ihl-10-preference-pairwise.png)

| チャンク | 内容 |
|----------|------|
| **進捗（控えめ）** | 収束度バー（右上・小さく） |
| **2 画像** | 左右 4:3・等価・**色補正なし**・撮影条件キャプション（§C） |
| **大ボタン** | 〔左〕〔右〕— **大きく等価**（`choice=left/right`・FR-MCH-PAIR-01） |
| **副次** | 〔どちらも ×〕（`neither`・FR-MCH-PAIR-02）+ 任意「詳しく ▸」（ValueCheck 詳細） |

> 左右は **等価選択** — 多色化しない（左右に優劣の色を付けない）。緑/赤は「採用/不採用」意味にのみ（10-UI v1 §3.1）。

---

## 3. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | ペア取得スピナ・画像はプレースホルダ |
| empty（候補 < 2） | 「まだ評価できる個体がありません」+ 観測導線 |
| error | 記録失敗トースト + 再試行（楽観的 UI ロールバック） |
| 未ログイン | ログイン誘導（profile は本人限定 · NFR-MCH-02） |
| 収束 | 好み傾向サマリ → 〔検索に反映〕 |

---

## 4. デザイン規範

- 写真 **色補正しない**・4:3・枠 1px・角丸 12px（§B/§C）。
- キーボード: ←=左 / →=右 / ↓=どちらも× / Enter=次（色だけに依存しない）。
- **サンプルデータ表示**（NFR-MCH-04）— 本番 capture 接続は H-06 後。
- 記録は `preference_event`（append-only · ADR-H-02）。
