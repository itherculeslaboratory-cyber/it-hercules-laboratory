## 2. データ契約（preference_event · INSERT ONLY · プライバシー）

### 2.1 preference_event（`events/preference_event` · append-only）

| フィールド | 規約 |
|------------|------|
| `preference_event_id` | `pref_*` 一意 |
| `user_id_hash` | sha256(salt:voter_handle) — **生 handle 非保存**（NFR-MCH-02） |
| `choice` | `left` / `right` / `neither` / `skip`（不正は left に丸め） |
| `left_capture_id` / `right_capture_id` | 比較対象 capture |
| `recorded_at` / `schema_version` | メタ |
| ~~`dimension_matrix`~~ | **Truth 非永続**（`_NON_PERSIST_KEYS` · schema-pack §6 #2） |

### 2.2 離散モデル（NFR-MCH-01）

- 連続数値合計スコアを定義しない。選好は離散イベント列として蓄積し、集約は再生成（FR-MCH-16 上書き禁止）。

## 3. API / route 契約（IHL 実エンドポイント）

### 3.1 `GET /api/v1/match/pair`

| 項目 | 契約 |
|------|------|
| 200（ペア有） | `{ status: "ok", pair: { pair_id, left:{capture_id,label}, right:{...}, ... } }` |
| ペア無 | `{ status: "empty", pair: null }` |

### 3.2 `POST /api/v1/match/vote`

| 項目 | 契約 |
|------|------|
| body | `{ pair_id, chosen, voter_handle }` |
| 200 | `{ status: "recorded", preference_event_id }` |
| 404 | pair_id 不在（「ペアが見つかりません」） |
| choice 不正 | left に丸め（落とさない） |
| プライバシー | persisted event に dimension_matrix 無 · voter は hash |

## 4. 状態（選好ループ）

```
[GET /pair] カタログ → ペア提示（無→empty）
[POST /vote] pair_id 解決（無→404）→ choice 正規化 → preference_event 追記（hash · 非永続 dimension）→ recorded
[集約] preference_event 列 → 再生成ビュー（上書き禁止 · FR-MCH-16）
```
