## 0. エントリー → ① おすすめ個体一覧 → ② pairwise（全体フロー）

```text
[ナビ /match] ──▶ [① recommend_list] ──「好みを精緻化する」──▶ [② pairwise ループ]
      │                    │  ▲                                    │
      │                    │  └──「おすすめに戻る」(状態保持)────────┘
      │              candidates = 0
      │                    ▼
      │            [recommend_empty] ──「まず②で好みを教えて」──▶ [② pairwise ループ]
      │
[リピート /match/pairwise] ──(初回不可)──▶ [② pairwise ループ]
```

**全画面共通**: 画面上部に progress chip（`mch-progress-chip`）— **発見 → 精緻化 → マッチ**（FR-MCH-UX-01）。③ マッチは post-v1 · chip 第 3 段は muted。

### 0.1 状態定義

| 状態 | 意味 | UX |
|------|------|----|
| `recommend_list` | ① おすすめ個体グリッド表示 | カード一覧（`mch-rec-grid`）+ 理由 1 行（`mch-rec-reason`）+ 詳細リンク（`mch-rec-detail-link`）+ 「好みを精緻化する」ボタン |
| `recommend_empty` | おすすめ候補なし（cold start） | 空状態（`mch-rec-empty`）+ 主 CTA「まず②で好みを教えて」（`mch-empty-to-pairwise`）+ 観測導線 |

### 0.2 遷移

| from → to | トリガ | ガード |
|-----------|--------|--------|
| `[ナビ] → recommend_list` | `/match` 到達 | candidates ≥ 1 · progress chip step=発見 active |
| `[ナビ] → recommend_empty` | `/match` 到達 | candidates = 0 · progress chip step=発見 active |
| `recommend_list → [pairwise load]` | 「好みを精緻化する」ボタン押下 | progress chip step=精緻化 active |
| `recommend_empty → [pairwise load]` | `mch-empty-to-pairwise` 押下 | — |
| `recommend_list → [individual detail]` | カード / `mch-rec-detail-link` 1 タップ | 観測 #05 または血統詳細ルート |
| `[individual detail] → recommend_list` | ブラウザ戻る / 閉じる | ① 状態 best-effort 復元 |
| `[pairwise *] → recommend_list` | `mch-back-to-rec` 押下 | pairwise セッション状態保持（FR-MCH-UX-04） |
| `[リピート] → [pairwise load]` | `/match/pairwise` 直アクセス | **初回訪問不可**（`preference_event = 0` → `/match` へリダイレクト） |

> **FR 参照**: FR-MCH-REC-01〜07 · FR-MCH-UX-01〜06

## 1. pairwise 評価ループ（本線）

```text
   [load] ──capture_set snapshot──▶ [pair_presented]
        │                                  │
        │                    choice(left/right/neither/skip)
        │                                  ▼
        │                         [recording] ──append preference_event──▶ [next_pair]
        │                                  │                                    │
        │                                  │ error(retry)                       │ round < N & not converged
        │                                  ▼                                    ▼
        │                         [record_error] ──retry──▶ [recording]   [pair_presented]
        │
        └─ no candidates ──▶ [empty]
                                  │ round ≥ N (§AI 仮定 N=10) / converged / 母集団枯渇
   [pair_presented] ─────────────┴──────────────────────────▶ [converged]
```

### 1.1 状態定義

| 状態 | 意味 | UX |
|------|------|----|
| `load` | capture_set snapshot 取得・最初のペア準備 | ローディング · progress chip step=精緻化 active |
| `pair_presented` | 2 枚提示・入力待ち | [左] [右] [どちらも×]（`mch-pair-neither`）· 「おすすめに戻る」（`mch-back-to-rec`）· 任意「詳しく」 |
| `recording` | 選択を `preference_event` へ append | 楽観的 UI（即次へ） |
| `record_error` | 記録失敗 | トースト + 再試行（raw エラー非表示） |
| `next_pair` | 次ペア選択（pair_strategy 適用） | ラウンドカウンタ更新 |
| `converged` | 収束 or N ラウンド到達 or 母集団枯渇 | 結果サマリ（`mch-pair-converged` · profile・収束度） |
| `empty` | 候補 capture なし | **空状態**: 「まだ評価できる個体がありません」 |

### 1.2 許可辺表

| from → to | トリガ | ガード |
|-----------|--------|--------|
| `load → pair_presented` | snapshot 取得成功 | 候補 ≥ 2 |
| `load → empty` | 候補 < 2 | — |
| `pair_presented → recording` | choice 入力 | choice ∈ {left,right,neither,skip} |
| `recording → next_pair` | append 成功 | INV-PE-01（append-only） |
| `recording → record_error` | append 失敗 | — |
| `record_error → recording` | 再試行 | — |
| `next_pair → pair_presented` | 次あり & round < N & 未収束 | 重複ペア回避（session 内） |
| `pair_presented / next_pair → converged` | round ≥ N **または** 収束 **または** 枯渇 | N 既定 **10**（FR-MCH-UX-07 · §AI 仮定） |
| `pair_presented → recommend_list` | `mch-back-to-rec` | pairwise 状態を Context に保持 |

> **skip**: 記録（`choice=skip`）した上で `next_pair` へ（情報は残すが弱シグナル）。
> **neither**（どちらも× · `mch-pair-neither`）: `choice=neither` を記録 → `next_pair`（正常ラウンドとして N カウントに含む）。

## 2. ValueCheck 詳細オーバーレイ（任意・副次）

pairwise 中に「詳しく」を選ぶと、**詳細オーバーレイ**で多次元評価へ。

```text
[pair_presented] ──「詳しく」──▶ [valuecheck_overlay]
   [valuecheck_overlay] ──dimensions 入力──▶ [valuecheck_overlay]
   [valuecheck_overlay] ──送信(overall_fit)──▶ [recording(dimension_matrix)] ──▶ [next_pair]
   [valuecheck_overlay] ──閉じる──▶ [pair_presented]（本線へ戻る）
```

| 状態 | writer | source |
|------|--------|--------|
| `valuecheck_overlay` | `preference_event(dimension_matrix)` + `ValueCheckSessionRecord` | `valuecheck_detail` |

> **境界（INV-PE-03）**: dimension_matrix は **詳細オーバーレイのみ**。本線 [左]/[右]/× は pairwise_choice。両者単一ストリーム。

## 3. convergence 状態（収束）

| 判定要素 | 内容 |
|----------|------|
| 収束条件 | 直近 K 回の preference_profile 変化量 < 閾値（詳細設計 §5） |
| **N ラウンド上限** | 連続 pairwise が **N=10** 回（§AI 仮定 · FR-MCH-UX-07）→ `mch-pair-converged` メッセージ + 結果サマリ |
| 枯渇条件 | snapshot 母集団のペア候補を出し切った |
| 収束後 UX | 結果サマリ（推定 prefer/avoid 特徴・収束度・寄与イベント数）+ 「続ける」「検索に反映」 |
| 続行 | `converged → pair_presented`（diversity 戦略で探索再開）可 |

## 4. エラー・例外導線

| ケース | 遷移 | UX |
|--------|------|----|
| 記録失敗 | `recording → record_error → recording` | トースト + 再試行（楽観的 UI のロールバック） |
| 画像ロード失敗 | ペア再取得 | プレースホルダ + スキップ可 |
| 候補なし | `load → empty` | 空状態 + 観測導線案内 |
| 未ログイン | ループ開始不可 | ログイン誘導（profile は本人限定・NFR-MCH-02） |
| 初回 `/match/pairwise` | リダイレクト | `/match` へ（① 必須 · FR-MCH-UX-06） |
