# 11 争い — 二人部屋（公開観覧） UI 設計（イメージ画面）

> **ステータス**: **草案 · 人間目視レビュー待ち**
> **正本**: [`../01-要件/11-裁判.md`](../01-要件/11-裁判.md) §3（指摘・二人部屋）· §6（マーケット争い）· §14（哲学）

---

## 1. 主タスク（1 つ）

**「当事者二人が対話して合意するか、期限で終わる」**。開発者は裁判官ではない。**人の指摘 → 二人部屋対話 → 合意 / 強制クローズ**（§1.1 · §14.1）。

---

## 2. イメージ画面

![争い 二人部屋](./mockups/ihl-11-dispute-tworoom.png)

| チャンク | 内容 |
|----------|------|
| **指摘サマリ** | タグ（例: 虚偽表現）+ 理由テキスト + ルーム ID |
| **公開観覧バナー** | 「公開観覧中・投稿できるのは当事者2人のみ」（FR-DSP-04） |
| **TTL チップ** | 「残り 27日でクローズ」（合意なし 1 ヶ月で強制クローズ · FR-DSP-08） |
| **対話** | 当事者の往復 + **修正案を提案 → 〔合意する〕**（§3.5） |
| **観覧者** | 閲覧のみ（介入・判定なし · スペクテーター） |
| **プラチナ投票（任意）** | 配送完了後 Y03/Y04/Y06 のみ opt-in · 7日 · 1票=1PT · 二択 · 集計表示（§6.3） |

---

## 3. 二人部屋とプライベートボードの違い（混同禁止）

| 種別 | 公開性 | 用途 | 正本 |
|------|--------|------|------|
| **二人部屋（本画面）** | **公開観覧可**（第三者は閲覧のみ） | 掲示板・出品の **指摘** 対話 | 11 §3.3 |
| **プライベートボード** | **第三者非公開** | マーケット Stage 1 の振込・発送 P2P | 06 §11.0 · [`06-マーケット.md`](./06-マーケット.md) |

---

## 4. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | 対話はスケルトン |
| 議論中 | 「指摘中／議論中」表示・重複指摘は不可（FR-DSP-05） |
| 期限切れ | 「合意なしでクローズ」+ 未解決累計の説明（5 倍数で Δcount +1） |
| PT 不足 | 指摘 30 回ごとに PT 1 枚 → 不足時は理由明示（FR-DSP-10） |

---

## 5. デザイン規範

- **通報ボタンを置かない**（指摘 + 対話のみ）。
- 合意は **明示操作**（提案 + OK）。自動検知しない（§3.5）。
- 旧投稿は削除せず **新タイムスタンプが表示勝ち**（FR-DSP-07）。
- プラチナ投票 UI は legacy `EconomyVote` 簡素版（2 ボタン + 集計 · §14.7.4）。
- 二人部屋は **クライアント翻訳** 許容（FR-DSP-11）。
