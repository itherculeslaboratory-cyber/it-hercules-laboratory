## 2. データ契約（dispute_kind · anchor · dispute_event · INSERT ONLY）

### 2.1 dispute_kind enum（設計正本 · 段階実装）

| `dispute_kind` | anchor | 可視性 | 実装 |
|----------------|--------|--------|------|
| `board_pointer` | `board_message` / `listing_engagement` | 公開観覧可 | gap（#07 連携） |
| `listing_pointer` | `listing` | 公開観覧可 | gap |
| `trade_private` | `trade_private` | 非公開 | **partial**（`market_dsp_ref`） |
| `trade_exchange` | `trade_public_exchange` | 公開 opt-in | gap |

### 2.2 dispute_event（`governance/dispute_event` · append-only）

| フィールド | 規約 |
|------------|------|
| `dispute_id` | `dsp_{thread_id}` 等 · 一意 |
| `event_kind` | `opened` / `evidence_added` / `agreed` / `force_closed` / … |
| `actor_id` | 当事者 ID |
| `market_dsp_ref` | 任意 · `trade_private:{trade_id}` |
| `recorded_at` | メタ |

- **UPDATE/DELETE 禁止**（NFR-DSP-02 · ProjectRules）。
- **現行実装**: `opened` · `evidence_added` のみ（retrofit スモーク）。

### 2.3 錨ルール（U-MKT-DSP-03 · 設計）

- 争点は `(anchor_type, anchor_id)` で正規化。
- **同一 anchor に open 争い 1 件** — 重複指摘拒否（FR-DSP-05 · gap）。

## 3. API / route 契約（IHL 実エンドポイント · retrofit）

### 3.1 `GET /api/v1/dispute/{thread_id}`

| 項目 | 契約 |
|------|------|
| 200 | `{ thread_id, dispute_id, status, parties, public_view, messages[], deadline, pt_required, market_dsp_ref? }` |
| `public_view` | **true** = 第三者閲覧可 · 投稿は当事者のみ（FR-DSP-04） |

### 3.2 `POST /api/v1/dispute/{thread_id}/messages`

| 項目 | 契約 |
|------|------|
| body | `{ actor_id, body }` |
| 200 | `{ status: "posted", actor, body(redacted), at }` |
| PII | `redact_pii_text(body)` 適用（FR-DSP-18 方向） |
| 副作用 | `evidence_added` dispute_event 追記 |

### 3.3 `DisputeService.open_market_dispute`（service 層 · テスト直接）

| 項目 | 契約 |
|------|------|
| 入力 | `thread_id`, `trade_id`, `actor_id` |
| 出力 | room に `market_dsp_ref = trade_private:{trade_id}` |
| イベント | `opened` + ref |
