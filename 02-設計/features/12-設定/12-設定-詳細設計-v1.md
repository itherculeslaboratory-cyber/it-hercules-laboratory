## 1. 設計方針

> **PII は公開ボード・チャットに打ち込まない**。取引 **前**に設定へ登録し、マッチング成立後の **当事者にのみ**最小限を開示する（FR-DSP-19 · §④.5）。

> **PII 最小化（HUMAN-CONFIRMED 2026-06-07）**: Truth/R2 に平文 PII を永続しない。`trade_pii` は取引時のみ暗号化または GMO/配送外部委譲し、取引完了後 purge。監査は `pii_access_event` のメタデータのみ。正本: [`schema-pack-v1.md` §7](../02-設計/_横断/schema-pack-v1.md#7-pii-minimization-policyhuman-confirmed-2026-06-07)。

| 原則 | 内容 |
|------|------|
| 最小開示 | 取引相手に渡すのは「配送に必要な項目のみ」。プロフィール公開には載せない |
| Truth 非永続 | 氏名・住所・口座等の **平文は IHL Truth/R2 に載せない**（§7） |
| append/PATCH | 設定値は preferences PATCH（許可フィールドのみ）。秘密値は平文ログ禁止（FR-SET-09）|
| 参照は成立後 | PII 参照は **マッチング成立（取引相手確定）後**のみ。それ以前は本人だけ |
| 監査 | PII 参照は `pii_access_event`（append-only）で **誰が・いつ・種別**のみ。本文は行に含めない |

## 2. データ契約（取引前設定）

`me_settings.trade_pii`（PATCH 対象 · **短命/外部 KV**。Truth/R2 平文永続は §7 禁止）:

```yaml
trade_pii:
  poste_restante:                 # 局留め（FR-SET-16・推奨）
    full_name: string             # 局留めに必要な氏名（フルネーム）
    office_name: string           # 郵便局名
    postal_code: string
  shipping_address:               # 配送先（FR-SET-17・推奨）
    recipient_name: string
    postal_code: string
    address_line: string          # 番地まで
    phone: string                 # 任意
  bank_account:                   # 振込口座（FR-SET-18・推奨）
    bank_name: string
    branch: string
    account_type: string          # 普通/当座
    account_number: string
    account_holder: string
  updated_at: string              # ISO8601
  completeness: { poste: bool, shipping: bool, bank: bool }  # 誘導判定用（FR-SET-19）
```

> `bank_account.account_number` 等は **保存時のみ送信**・レスポンスでは **マスク**（末尾 4 桁のみ）。

## 3. API スキーマ

| メソッド | ルート | 認可 | 内容 |
|---------|--------|------|------|
| `GET` | `/api/me/settings/trade-pii` | 本人のみ | 自分の trade_pii（口座番号はマスク）|
| `PATCH` | `/api/me/settings/trade-pii` | 本人のみ | 許可フィールドのみ更新（部分更新）|
| `GET` | `/api/trades/{trade_id}/counterparty-pii` | **取引相手のみ・成立後** | 相手が開示した最小項目 |

### 3.1 `PATCH /api/me/settings/trade-pii`（リクエスト例）

```json
{ "shipping_address": { "recipient_name": "…", "postal_code": "…", "address_line": "…" } }
```

レスポンス: 更新後の trade_pii（口座マスク）+ `completeness`。

### 3.2 `GET /api/trades/{trade_id}/counterparty-pii`（アクセス制御）

```text
前提: trade_id が「成立(matched/agreed)」状態で、リクエスタが当事者2名のいずれか
許可: 相手が「この取引で開示」を選んだ項目のみ返す（配送方法に応じた最小集合）
拒否: 当事者でない / 未成立 → 403 + 理由（raw エラー非表示・U-* DoD）
記録: 参照成功は pii_access_event に append（accessor / trade_id / fields / at）
```

## 4. アクセス制御マトリクス

| 主体 | 自分の trade_pii | 相手の PII（成立前）| 相手の PII（成立後）| 公開ボード |
|------|:---:|:---:|:---:|:---:|
| 本人 | RW | — | — | — |
| 取引相手 | — | ✕（403）| **最小開示のみ** | ✕（PII 直書き禁止）|
| 第三者/観覧者 | — | ✕ | ✕ | ✕ |
| 管理者 | ✕（付与は別）| ✕ | ✕（争い証拠は redact 済参照のみ）| — |

- 争い（11 裁判）に出る場合も **構造化参照のみ**・公開エクスチェンジ移行時は **PII redact**（FR-DSP-18 · §6.8.2）。

## 5. 取引前誘導（FR-SET-19）

```text
マーケット申込（Stage 0→1）時:
  completeness を確認 →
    不足あり → 「配送先/口座が未設定です。設定すると相手にボードで住所を書かずに渡せます」
              → [設定へ] (1 クリック) → 戻る
    完了      → そのまま進行
必須化: 既定は「強く推奨（ブロックしない）」。必須化の最終判断は人間ゲート（U-MKT-DSP-10）
```
