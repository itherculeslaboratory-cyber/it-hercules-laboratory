## 2. データ契約（preferences · Tier C 投影）

### 2.1 `preferences` ドキュメント（`preferences/v1/{actor_id}/settings.json`）

| フィールド | 型 | 規約 | FR |
|------------|-----|------|-----|
| `language` | string | 表示言語（`ja` / `en` 等 · #21 同一集合） | FR-SET-11/15 |
| `notifications` | bool | Push/通知 opt-in 投影 | FR-SET-07（部分） |
| `default_device_id` | string \| null | 環境 IoT 既定デバイス（#13 xref） | — |
| `counterparty_pii_mode` | string | 既定 `session_only` — Truth 非保存 | FR-SET-17/18 方向 |
| `truth_pii_policy` | string | 既定 `no_plaintext` — 書込禁止 | FR-SET-09 · §7 |
| `updated_at` | ISO8601 | PATCH 時更新 | FR-SET-09 |

- **PATCH 許可フィールドのみ**更新（`allowed` frozenset · それ以外は無視）（FR-SET-09）。
- **API Key / 口座番号平文**は本投影に載せない（NFR-SET-01 · schema-pack §7）。

### 2.2 `trade_pii`（v1 設計 · **IHL 未実装 · gap**）

[`12-設定-詳細設計-v1.md`](./12-設定-詳細設計-v1.md) §2 の YAML 契約を正とする:

- `poste_restante` · `shipping_address` · `bank_account` · `completeness`
- 保存: 短命 KV または外部委譲 — **Truth/R2 平文永続禁止**
- レスポンス: 口座番号マスク（末尾 4 桁）

### 2.3 `TradePiiSession`（セッション · 実装スモーク）

| 操作 | 契約 |
|------|------|
| `store(trade_id, contact_token)` | セッション dict のみ · `pii_access_event` append |
| `get_masked(trade_id)` | 本文非返却 · `mode: session_active \| masked` |

## 3. API / route 契約（IHL 実エンドポイント · retrofit）

### 3.1 `GET /api/v1/me/preferences`

| 項目 | 契約 |
|------|------|
| query | `actor_id`（テスト/デモ用 · 本番は session から解決） |
| 200 | `DEFAULT_PREFERENCES` マージ済み JSON |
| 秘密値 | 返さない（NFR-SET-01） |

### 3.2 `PATCH /api/v1/me/preferences`

| 項目 | 契約 |
|------|------|
| body | `{ language?, notifications?, default_device_id?, counterparty_pii_mode?, actor_id }` |
| 200 | 更新後 preferences 全文 |
| 拒否 | 許可外キーは無視 · 平文 PII キーは store 層で禁止 |

### 3.3 `GET /api/v1/settings` · `GET /api/v1/me/settings`

| 項目 | 契約 |
|------|------|
| 200 | preferences 投影と同一（読取専用エイリアス） |
| 用途 | 設定ハブ load（遷移設計 SET state） |

### 3.4 `POST /api/v1/settings/pii-session`

| 項目 | 契約 |
|------|------|
| query | `trade_id`, `contact_token`, `actor_id` |
| 200 | `{ trade_id, mode: session_only, ref: trade_private:{id} }` |
| 副作用 | `pii_access_event` append（accessor ハッシュ · 本文なし） |
| 設計差分 | v1 の `GET /api/trades/{id}/counterparty-pii` は **未実装**（gap） |

### 3.5 認可（FR-SET-01 · 部分）

- **契約**: 認証済みユーザーのみ `/me/*` — IHL 現行は `actor_id` クエリ（テスト用）· **本番 auth middleware は #01 連携 gap**。
- **admin 導線変更**（FR-SET-05）: `isAdminLikeUser` ガード — UI/API 未実装（gap）。
