## 2. 設定ハブ state machine

```text
[load] ──GET settings──▶ [hub_ready]
                              │
         セクション選択 ───────┼──▶ [section_edit] ──PATCH──▶ [saving]
                              │                                    │
                              │                         success ───┼──▶ [save_ok] ──▶ hub_ready
                              │                         fail ──────┼──▶ [save_error] ──retry──▶ saving
                              │
         戻る（各所） ──────────┴──▶ [hub_ready]
```

| 状態 | UX |
|------|-----|
| `load` | スケルトン |
| `hub_ready` | セクション一覧 · 各 completeness バッジ |
| `section_edit` | フォーム編集 |
| `saving` | 主ボタン disabled |
| `save_ok` | 1 行成功（3 秒で消える） |
| `save_error` | 理由付きバナー · 再試行 |

## 3. 取引前 PII 導線（06 連動）

```text
06 申込/Stage1 ──PII 未設定──▶ SET-PII（誘導バナー）
06 counterparty ──成立後──▶ GET /api/trades/{id}/counterparty-pii（詳細は 12 詳細設計 §3）
SET-PII ──保存成功──▶ 06 へ戻る（return_to query）
```

| from → to | トリガ | ガード |
|-----------|--------|--------|
| 06 → SET-PII | 「取引前設定へ」 | `completeness` 任意（推奨バッジ） |
| SET-PII → 06 | 保存成功 + 戻る | `return_to` 有効 URL のみ |
| ボード/チャット | PII 入力禁止 | エラー導線 → SET-PII へ |
