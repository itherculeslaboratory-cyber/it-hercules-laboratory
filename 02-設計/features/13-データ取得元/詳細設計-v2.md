## 2. データ契約（4 概念 · Tier A/B · INSERT ONLY）

### 2.1 Placement（Tier A · INSERT ONLY）

| フィールド | 規約 |
|------------|------|
| `placement_id` | `pl_{hex32}` 安定 ID |
| `label` / `meta` | 人間可読 · 任意部屋名（幾何マップは optional） |
| 書込 | **新規 INSERT のみ** — UPDATE/DELETE 禁止（FR-ENV-01） |

### 2.2 DeviceBinding（Tier A · 設計正本 · 段階実装）

| フィールド | 規約 |
|------------|------|
| `device_id` | registry 公開 ID |
| `startedAt` | 必須 |
| `endedAt` | **終了イベント追記**で閉じる |
| 重複 | 同一 deviceId の未終了 binding 重複は **409**（FR-ENV-02） |

- **現行実装**: `openBinding: null` stub（shelf API）— **gap**（§7 P3）。

### 2.3 Occupancy（Tier A）

| フィールド | 規約 |
|------------|------|
| `subject_ref` | 固体 sessionId / specimen 参照可（FR-ENV-03） |
| 重複 | 未終了 occupancy の再 start は **409**（実装済） |

### 2.4 TelemetryIngest（Tier B · ADR-H-19/20）

| 項目 | 規約 |
|------|------|
| natural_key | `(device_id, bucket_start_unix)` · `floor(ts, 300)` |
| 許可列 | 測定値 + placement/annotation 文脈のみ（FR-ENV-08） |
| 未解決 placement | 許容 · 事後紐づけは **新イベント**（FR-ENV-04 · gap） |
| UPSERT | 値不変 + 当バケット既存 → `env_telemetry_skip_unchanged` |

### 2.5 QR トークン（Tier A INSERT）

- 再発行 = **新 INSERT**（FR-ENV-05）
- 期限切れ / 不明 token → **404**

## 3. API / route 契約（IHL 実エンドポイント · retrofit）

### 3.1 `POST /api/env/ingest`

| 項目 | 契約 |
|------|------|
| 認証 | `X-IHL-Collector-Id` · `Timestamp` · `Signature`（Ed25519） |
| 成功 | `201` + `{ sampleIds: [...] }` · 全スキップ時 `skipped: true` |
| 失敗 | `401` 署名 · `400` schema · `503` 公開鍵未設定 |

### 3.2 Placement / Shelf / Occupancy

| メソッド | ルート | 備考 |
|----------|--------|------|
| `POST` | `/api/env/placements` | 201 INSERT |
| `GET` | `/api/env/placements` | 一覧 |
| `POST` | `/api/env/placements/{id}/qr` | TTL QR |
| `GET` | `/api/env/qr/{token}` | Bearer 不要 |
| `GET` | `/api/env/placements/{id}/shelf` | open Occupancy · Binding（partial） |
| `POST` | `.../occupancy/start|end` | 409 重複 / 未 open |

### 3.3 Telemetry / History

| メソッド | ルート | 備考 |
|----------|--------|------|
| `GET` | `/api/env/placements/{id}/telemetry` | binding 済み device 想定（設計） |
| `GET` | `/api/env/history` | 直近既定 · max 400（FR-ENV-09 · partial dev） |

### 3.4 固体接続（#05 xref · FR-ENV-06）

- 環境の **二重 POST 禁止** — 棚は #13 Binding/Occupancy · 固体は `POST /api/captures` のみ。
- `include_env_measurements: true` → `measurement_method=iot_switchbot` · `value_origin=environment_derived`。
