## 1. 主タスク

**「登録済みの取得元（SwitchBot 等）を一覧で確認し、追加・バインド・解除する」**

> 秘密値（TOKEN/SECRET）は **UI から入力させない**（collector `.env` に隔離・専門班 C B-3）。本画面は **登録の可視化と Placement へのバインド管理**に限る。

## 4. チャンク・状態・規範

- **3 チャンク**（collector 状態 / 機器一覧 / 追加）。主要導線 3 クリック以内。
- **状態必須**: loading / 空（機器 0: 「collector を起動して取得元を登録してください」+ 手順リンク）/ エラー / 409 競合（同一 device の未終了 binding 重複 = FR-ENV-02）を **理由付き**表示。
- バインド/解除は **append-only**（解除 = 終了イベント追記・行は消えない）。UI は「解除済み」を履歴で見せる。
- 非 SwitchBot（BLE scale / Wi-Fi sensor）は **driver 種別 chip** で区別（[`ADR-device-driver-registry.md`](./ADR-device-driver-registry.md)）。driver 未対応は「対応予定の種別」を出さず、対応済みのみ表示。
- 色は意味のみ（稼働=success / 待機=neutral / エラー=fail）。
