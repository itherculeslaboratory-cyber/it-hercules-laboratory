## 0. この文書の位置づけ

本書は **要件 #14（貢献度）の `FR-CONTRIB-*` を「実装と検証で追える設計」に厚くした v2** である。

- **指標の意味**: Kernel 定義イベント（いいね・コピー・組み込み・観測投稿等）を換算し、**ノード/ユーザー単位で非負累積**する活動量指標（PlatinumCoinRules §5.1）。カルマ（#08）・マーケット評価（#06）と **統合しない**（ADR-H-08）。
- **研究スコア内訳**（§4.1）: `contribution_event.reason_code` / `event_type` を集計し、プロフィール「貢献度」カードに **数値 + 内訳** を併記。Coin は累計取得枚数のみ公開 · PT は非公開。
- **IHL 現状**: `contribution_event`（append-only）· `contribution_total`（340 ベース + Σdelta）· `/api/v1/contribution`（total · badges · platinum_balance stub）· `/api/v1/profile/metrics`（contribution.value + breakdown stub）· `/contribution` UI（合計 + バッジ）。legacy `contributionEconomy.ts` の **上流 10% 分配・100 単位ミント・依存グラフ** は **salvage 参照のみ**（IHL 未配線 · §7）。
- v2 の追加価値は **§2 contribution_event データ契約**・**§3 IHL route 契約**・**§4 非負・ミント状態機械（設計境界）**・**§6 NFR**・**§7 retrofit 差分（legacy 経済 ↔ IHL stub）**の厳密化にある。

## 2. データ契約（contribution_event · 非負 · INSERT ONLY）

### 2.1 contribution_event（`economy/contribution_event` · append-only）

| フィールド | 規約 |
|------------|------|
| `contribution_event_id` | 一意 ID（`contrib_*`） |
| `actor_id` | 対象ユーザー（`@handle`） |
| `delta` | 加算値（**非負** · FR-CONTRIB-01） |
| `reason_code` | `observation` / `paper` / `like` / `copy` / `embed` 等（FR-CONTRIB-02） |
| `source_type` | `human` / `kernel` / `manual` |
| `target_ref` | 任意 · ノード/投稿参照 |
| `created_at` | UTC ISO8601 |
| `schema_version` | 1 |

- **INSERT ONLY**（NFR-CONTRIB-01）: UPDATE/DELETE 禁止。監査は event 列の順序で再現。
- **研究スコア内訳**（§4.1）: `reason_code` を軸に `breakdown.by_paper` · `by_observation` 等へ投影（集計 API は §7 P2 gap）。

### 2.2 contribution_total 投影

```
contribution_total(actor_id) = BASE(340.0) + Σ(delta) where actor_id matches
```

- BASE は IHL stub 既定（デモ表示用）。本番は Σdelta のみへ移行可（CR）。
- **減算なし**（FR-CONTRIB-01）: delta < 0 のイベントは ingest 拒否（設計）· 現状は加算のみ記録。

### 2.3 NodeGraph / 上流分配（legacy 契約 · IHL 未配線）

| 概念 | legacy 正本 | IHL |
|------|------------|-----|
| `NodeGraphEntry.ancestors` | `{ nodeId, weight }[]` | **gap**（§7 P1） |
| upstreamPercent（既定 10%） | `applyContributionDelta` | **gap** |
| 祖先なし時子に残す | FR-CONTRIB-04 | **gap** |

## 4. 状態機械（加算 · ミント · 上流 · 非負）

```
[Kernel イベント] → write_contribution_event(delta>=0) → contribution_event append
[Σ score] → contribution_total 投影（非負 · 減算なし）
[score 100 到達ごと] → issueCoin 1 枚 · 端数繰越（FR-CONTRIB-05 · legacy · IHL gap）
[子ノード Δ] → upstream 10% 祖先へ重み配分（FR-CONTRIB-03/06 · legacy · IHL gap）
[祖先なし] → upstream 分は子に残す（FR-CONTRIB-04 · legacy）
[ユーザー総貢献] → applyUserContributionDelta（FR-CONTRIB-09 · profile 連携 · partial）
```

- C-USB: FeatureNode `economy` / Kernel 貢献度 — イベント ingest → Transform（換算・分配）→ OUT（state + coin）。
