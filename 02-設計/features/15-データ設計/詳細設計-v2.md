## 0. この文書の位置づけ

本書は **要件 #15（データ設計）の `FR-DATA-*` / `NFR-DATA-*` を「実装と検証で追える設計」に厚くした v2** である。#15 は **ユーザー画面を持たない横断憲法層** — 全 FeatureNode が schema を **消費** する。

- **#00 土台との関係**: #00 が `SchemaRegistry` · R2 I/O · catalog の**実装正本**。#15 は **文明 OS + IHL のデータ思想・schema-pack 索引・manifest 不変条件**の要件トレース層。
- **IHL rebuild が実装した部分**: schema-pack v1（45 schema · 24 dictionaries）· `validate_instance` · append-only R2/manifest · pipeline IN/OUT manifest · `run_info` / `errors.jsonl` · ULID prefix · tag event JSONL。
- **v2 の追加価値**: **§2 データ契約（文明 + IHL）** · **§3 検証・manifest 契約** · **§6 NFR** · **§7 legacy↔IHL parity（civ-os bridge gap 透明化）**。

## 2. データ契約（文明 OS · IHL ファイル契約）

### 2.1 CivilizationEntityBase（FR-DATA-01/02/08）

| フィールド層 | 規約 | FR |
|------------|------|-----|
| `id` + `kind` + `layer?` | 全エンティティ必須 | FR-DATA-01 |

### 2.2 R2 文明史（FR-DATA-03/04/07）

| 項目 | 契約 |
|------|------|
| イベント | `prev_hash` · `event_hash` 必須 · INSERT ONLY |

| 4 媒体同期 | spec / post / commit / R2（REQ-001） |

### 2.3 IHL 永続・manifest（FR-DATA-09..15）

| schema / 成果物 | 規約 | FR |
|----------------|------|-----|
| 保存先 | Cloudflare R2 のみ · DB 禁止 | FR-DATA-09 |
| pipeline IO | input manifest 読取 · output manifest + run_info + errors.jsonl 必須 | FR-DATA-10 |

| ID prefix | ULID 系 `ind_` / `cap_` / `run_` / `snap_` | FR-DATA-14 |
| タグ | append-only tag event JSONL + aggregate ビュー | FR-DATA-15 |

## 3. 検証・component 契約（retrofit）

### 3.1 `SchemaRegistry` / `validate_instance`

| 操作 | 契約 |
|------|------|
| `validate_instance(ref, doc)` | Draft 2020-12 · `enum_ref` → dictionaries |
| `validate_or_raise` | 失敗時 `ValidationError`（pipeline 前段） |

| 不変条件 | 内容 |
|----------|------|
| exists 後 write | 衝突時例外（NFR-DATA-03） |
| manifest のみ参照 | 列の複製禁止（ADR-Phase2） |
| embedding 大規模 | `.npy` + manifest offset（NFR-DATA-04） |

### 3.3 キー名前空間（NFR-DATA-05）

- civ-os `world/` vs IHL `raw/` / `manifests/` — **混在禁止**。
