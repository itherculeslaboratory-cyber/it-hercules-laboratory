## 1. 主タスク（1 つ）

**「既存部品を選んで画面に配置し、見た目を整えて保存する」**。機能を作るのではなく **組み立てる**（H-01 = REFRAME · 配置 + デザイン + catalog 紐づけ のみ）。

---

## 2. イメージ画面

![UIビルダー](./mockups/ihl-16-uibuilder-canvas.png)

| チャンク | 内容 |
|----------|------|
| **パレット（左）** | 配置（Section/List/Card/Button）+ catalog 部品（フィルタ・層不足はグレー） |
| **キャンバス（中央）** | ScreenDef ブロック配置・D&D・選択強調・🔗 binding 表示 |
| **プロパティ（右）** | 配置（列/順序/余白）/ デザイン（トークン/強調）/ 紐づけ（〔部品を選ぶ〕） |
| **保存バー（上）** | 編集⇄プレビュー切替・層選択・**1 主ボタン〔保存〕**（INSERT ONLY） |

> RAG / fork / Pilot / route-matrix タブは **去**（IDE 必須にしない · FR-16-REFRAME-08）。fork は L4 の Apply ボタンに縮小。
> ヘッダに **コンテキストパンくず「編集中: 観測 › 検索グリッド」** と **〔元の画面に戻る〕**（今見ている画面をそのまま編集・ホーム固定禁止）→ [`16-UIbuilder-コンテキスト編集.md`](./16-UIbuilder-コンテキスト編集.md)。

---

## 3. catalog 紐づけ（3 クリック）

ブロック選択 → 〔部品を選ぶ〕 → 〔選択〕（NFR-16-04）。catalog 外 binding はグレーアウト + 「この部品は利用できません」+ **代替導線**（repo で component 追加・ヘルプ）。

---

## 4. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | lint 中スピナ・保存中はボタン無効 |
| empty | 部品ゼロ → 「repo で追加（ヘルプ）」/ キャンバス空 → 「ブロックを追加して始めましょう」 |
| error | lint 失敗バナー（平易文言）+ 該当ブロック強調。**raw JSON/stack は L1 に出さない** |
| saved | 保存トースト（新 screen_def_id・版） |
| 権限なし | 層不足アクションはグレー + ツールチップ「L3 以上で利用可」 |

---

## 5. デザイン規範

- 配置・デザインに集中させる UI（機能開発を Builder に持ち込まない · §3 /dev 外導線）。
- primary は **保存 1 つ**。エラー=赤・情報=青・無効=グレー。
- 「新機能が欲しい」は **開発パイプラインへ誘導**（未実装文言を出さない）。
- キーボードでブロック選択・並べ替え・保存（Tab/矢印/Enter）。

---

## 6. コンテキスト編集（この画面を編集）

編集可能な ScreenDef ページに **〔この画面を編集〕**（**L3+ のみ**・右下フローティング）を置き、押すと **その画面（`?screenId=current`）をプリロードして** Builder を開く（**ホーム固定禁止**）。編集後は **元のルートへ戻す**。
