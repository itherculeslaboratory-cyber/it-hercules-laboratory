# 16 UIビルダー — コンテキスト編集（この画面を編集）

> **ステータス**: **草案 · 人間レビュー反映 2026-06-07**
> **目的**: 「**今見ている画面から、その画面をそのまま編集**」できるようにする。Builder 起動でホームに固定されない。

## 1. 背景（ユーザーの痛み）

これまでの Builder は **起動するとホームのダッシュボードに戻され**、「今いる画面」を編集できなかった（迷子・編集対象の取り違え）。本書は **コンテキスト編集**（viewing → そのまま編集）を固定する。

---

## 2. アンチパターン ADR（禁止）

| # | 決定 |
|---|------|
| **A1（禁止）** | **「Builder 起動 = ホーム固定」を禁止**。Builder を開いたとき、編集対象を毎回ホーム/空ダッシュボードにしない。 |
| A2 | 編集は **「今見ている ScreenDef」を初期対象**にする。起点画面のルートを Builder に引き継ぐ。 |
| A3 | 編集後は **元のルートへ戻す**（`return_to_same_route`）。別画面に飛ばさない。 |
| A4 | 編集対象は **ヘッダのコンテキストパンくず**で常時明示し、対象が固定（ロック）されていることを示す。 |

> 例外（ホームから始めてよい場合）: ユーザーが **明示的に「新しい画面を作る」** を選んだとき。その場合のみ空キャンバスで開始する。

---

## 3. 入口 —「この画面を編集」

編集可能な ScreenDef ページ（ScreenDef レンダラ画面）に、**フローティングの〔この画面を編集〕** を置く。

![この画面を編集（入口）](./mockups/ihl-16-edit-this-screen-entry.png)

| 項目 | 仕様 |
|------|------|
| 配置 | 画面 **右下のフローティングボタン**（ピル型・鉛筆アイコン）。本文の上にオーバーレイ。 |
| 権限 | **管理者 / L3 以上のみ表示**（L1/L2 には出さない）。ツールチップ「管理者（L3 以上）のみ」。 |
| 動作 | クリックで Builder を **`?screenId=<現在の screen_def_id>` プリロード**で開く（**ホームダッシュボードではない**）。 |
| 対象 | 起点画面の ScreenDef を **編集対象としてロック**（キャンバスに復元・パンくずに表示）。 |

---

## 4. 編集キャンバス（対象ロック）

![UIビルダー（編集中: 観測 › 検索グリッド）](./mockups/ihl-16-uibuilder-canvas.png)

| 要素 | 内容 |
|------|------|
| **コンテキストパンくず** | ヘッダに **「編集中: 観測 › 検索グリッド」**（鉛筆アイコン + chip）。何を編集しているか常時明示。 |
| **〔元の画面に戻る〕** | ヘッダ左の戻るボタン。保存せず起点ルートへ復帰（編集破棄は確認）。 |
| **対象固定表示** | キャンバス上部「対象: /observation（検索グリッド）固定」。誤って別画面を編集しない。 |
| 主ボタン | **〔保存〕1 つ**（INSERT ONLY）。保存後は起点ルートへ戻れる。 |

> Builder のスコープは ADR-H-01 のまま **配置 + デザイン + catalog 紐づけのみ**（機能開発は持ち込まない）。本書が足すのは **「対象の決まり方（コンテキスト）と戻り先」** だけ。

---

## 5. 状態遷移（コンテキスト編集）

```text
[viewing （ScreenDef ページ閲覧・L3+）]
      │ 〔この画面を編集〕クリック（screenId プリロード）
      ▼
[edit_this_screen （対象 = 起点 screen_def_id をロード）]
      │ 配置/デザイン/紐づけ
      ▼
[builder_canvas （対象ロック・パンくず表示）] ──プレビュー──▶ [preview] ──戻る──▶ [builder_canvas]
      │ 保存（lint → INSERT ONLY）
      ▼
[save] ──成功──▶ [saved（新 screen_def_id）]
      │ 〔元の画面に戻る〕 / 保存後復帰
      ▼
[return_to_same_route （起点ルートへ・新版を反映）]
```

### 5.1 状態定義

| 状態 | 意味 | 主アクション | 戻り先 |
|------|------|--------------|--------|
| `viewing` | ScreenDef ページ閲覧（L3+ に編集入口表示） | 〔この画面を編集〕 | — |
| `edit_this_screen` | 起点 screenId を Builder にロード中 | 自動（プリロード） | viewing（失敗時） |
| `builder_canvas` | 対象ロックで編集 | 配置/デザイン/紐づけ | viewing（元の画面に戻る） |
| `preview` | ライブプレビュー（read-only） | 配置確認 | builder_canvas |
| `save` | lint → INSERT ONLY 保存 | 保存 | builder_canvas（lint 失敗） |
| `saved` | 新 `screen_def_id` 確定 | トースト | return_to_same_route |
| `return_to_same_route` | **起点ルートへ復帰**（新版反映） | — | viewing（同ルート） |

### 5.2 許可辺・ガード

| from → to | トリガ | 許可層 | ガード |
|-----------|--------|:------:|--------|
| `viewing → edit_this_screen` | 〔この画面を編集〕 | **L3+** | 入口は L3+ のみ表示 |
| `edit_this_screen → builder_canvas` | screenId ロード成功 | L3+ | 対象 ScreenDef が存在 |
| `edit_this_screen → viewing` | ロード失敗 | L3+ | 失敗理由 + 再試行（raw 非表示） |
| `builder_canvas → viewing` | 〔元の画面に戻る〕 | L3+ | 未保存変更は破棄確認 |
| `builder_canvas → save` | 保存 | L3+ | lint pass · INSERT ONLY |
| `saved → return_to_same_route` | 復帰 | L3+ | **起点ルートに戻す（ホーム固定禁止 A1）** |

> **不正遷移**: 保存後にホームへ強制遷移しない（A1）。別画面の screenId を勝手に対象にしない（A2）。

---

## 6. 状態（UX 必須）

| 状態 | 表示 |
|------|------|
| loading | screenId ロード中スピナ・lint 中スピナ |
| empty | 対象が空 ScreenDef → 「ブロックを追加して始めましょう」 |
| error | ロード/lint 失敗バナー（平易文言）+ 再試行。raw JSON/stack は L1 に出さない |
| saved | 保存トースト（新 screen_def_id・版）+ 〔元の画面に戻る〕 |
| 権限なし | L1/L2 は **入口を表示しない**（押せるのに弾く UI にしない） |

---

## 7. レビュー観点

- [ ] 編集可能な ScreenDef ページに **〔この画面を編集〕**（L3+・右下フローティング）があるか。
- [ ] Builder 起動が **ホーム固定でなく**、起点 `screenId` をプリロードしているか（A1）。
- [ ] ヘッダに **コンテキストパンくず（編集中: …）** と **〔元の画面に戻る〕** があるか。
- [ ] 保存後 **同じルートへ戻る**（return_to_same_route）か。
- [ ] L1/L2 に編集入口を出していないか（権限）。
