## 1. 編集セッション状態機械

```text
        ┌─────────────────────────────────────────────┐
        ▼                                             │
   [draft] ──edit/place/bind──▶ [draft]               │ (戻る/編集継続)
        │                                             │
        │ preview                                     │
        ▼                                             │
   [preview] ──back──▶ [draft] ───────────────────────┘
        │
        │ run lint
        ▼
   [lint] ──fail──▶ [lint_error] ──fix──▶ [draft]
        │ pass
        ▼
   [save] ──INSERT ONLY（新 screen_def_id）──▶ [saved]
        │
        │ (任意・L4 のみ)
        ▼
   [fork_apply] ──公開トリガ──▶ [applied]
```

### 1.1 状態定義

| 状態 | 意味 | 主アクション | UI |
|------|------|--------------|----|
| `draft` | 配置・デザイン・紐づけ編集中 | ブロック追加/並べ替え/binding/トークン編集 | キャンバス + パレット |
| `preview` | ライブプレビュー | 実データ read-only 表示・配置確認 | プレビュー pane |
| `lint` | schema/binding/props 検証中 | 自動（保存前） | スピナ（ローディング） |
| `lint_error` | 検証失敗 | 該当ブロック修正へ誘導 | エラーバナー（平易文言） |
| `saved` | R2 INSERT ONLY 保存済 | 新 `screen_def_id` 確定 | 保存トースト |
| `fork_apply` / `applied` | 既存テンプレ派生の **公開トリガ**（L4） | Apply ボタン | 確認ダイアログ |

### 1.2 許可辺表

| from → to | トリガ | 許可層 | ガード |
|-----------|--------|:------:|--------|
| `draft → draft` | 編集 | L1–L4 | 編集対象が catalog 内（INV-CAT-01） |
| `draft → preview` | プレビュー | L1–L4 | — |
| `preview → draft` | 戻る | L1–L4 | — |
| `draft → lint` | 保存試行 | L1–L4 | — |
| `lint → lint_error` | 検証失敗 | — | schema/binding/props 不適合 |
| `lint_error → draft` | 修正 | L1–L4 | — |
| `lint → saved` | 検証成功 | L1–L4 | INV-SD-01（INSERT ONLY） |
| `saved → fork_apply` | 公開 | **L4 のみ** | fork は既存テンプレ派生のみ |
| `fork_apply → applied` | 確定 | L4 | backend 正本 API（プロトコルは Builder 外） |

> **不正遷移**: catalog 外 binding 試行 → `draft` に留まりグレーアウト + 「この部品は利用できません」。保存衝突（同 id 上書き）→ **409 相当**、新 id 発番で回避（INSERT ONLY）。

## 1.3 コンテキスト編集（この画面を編集 / ホーム固定禁止）

> **背景**: Builder 起動でホームに戻され、今見ている画面を編集できない問題への対処。**UI 設計正本**: [`../../02-設計/features/16-UIbuilder/ui/コンテキスト編集.md`](../../02-設計/features/16-UIbuilder/ui/コンテキスト編集.md)。

### アンチパターン ADR

| # | 決定 |
|---|------|
| **A1（禁止）** | **「Builder 起動 = ホーム固定」を禁止**。開いたとき編集対象を毎回ホーム/空にしない。 |
| A2 | 編集は **「今見ている ScreenDef」を初期対象**（起点ルートを Builder に引き継ぐ）。 |
| A3 | 編集後は **元のルートへ戻す**（`return_to_same_route`）。 |
| A4 | 編集対象は **ヘッダのコンテキストパンくず**で常時明示（対象ロック）。 |

> 例外: ユーザーが明示的に「新しい画面を作る」を選んだときのみ空キャンバスで開始。

### 状態機械（コンテキスト編集）

```text
[viewing（ScreenDef ページ・L3+）]
   │ 〔この画面を編集〕(?screenId=current プリロード)
   ▼
[edit_this_screen（起点 screen_def_id をロード）]
   │ ロード成功
   ▼
[builder_canvas（対象ロック・パンくず「編集中: …」）]
   │  ├─ preview ⇄ builder_canvas
   │  ├─ 〔元の画面に戻る〕 ──▶ viewing（未保存は破棄確認）
   │  保存（既存 §1 の lint → saved を再利用）
   ▼
[saved（新 screen_def_id）] ──▶ [return_to_same_route（起点ルートへ・新版反映）]
```

### 許可辺（コンテキスト）

| from → to | トリガ | 許可層 | ガード |
|-----------|--------|:------:|--------|
| `viewing → edit_this_screen` | 〔この画面を編集〕 | **L3+** | 入口は L3+ のみ表示 |
| `edit_this_screen → builder_canvas` | screenId ロード成功 | L3+ | 対象 ScreenDef 存在 |
| `edit_this_screen → viewing` | ロード失敗 | L3+ | 失敗理由 + 再試行（raw 非表示） |
| `builder_canvas → viewing` | 〔元の画面に戻る〕 | L3+ | 未保存変更は破棄確認 |
| `saved → return_to_same_route` | 復帰 | L3+ | **起点ルートに戻す（A1: ホーム固定禁止）** |

> `builder_canvas` は §1 の `draft` と等価（対象がロックされた draft）。lint/save/fork_apply は §1 をそのまま使う。

## 1.4 テーマ編集 state machine（ADR-H-17）

> **UI 正本**: [`../../02-設計/features/16-UIbuilder/ui/テーマ.md`](../../02-設計/features/16-UIbuilder/ui/テーマ.md) · **契約**: [ADR-H-17](./ADR-H-17-DesignTheme-テーマパック.md)

```text
[theme_idle]
   │ テーマタブ / 設定から入口
   ▼
[theme_browse] ──プリセット選択──▶ [theme_preview]
   │                                    │
   │ Phase 2: トークン編集              │ 全画面プレビュー
   ▼                                    ▼
[theme_edit_tokens] ──lint──▶ [theme_lint] ──fail──▶ [theme_edit_tokens]
                                    │ pass
                                    ▼
                              [theme_save] ──INSERT ONLY──▶ [theme_saved]
                                    │
                                    │ (L4 · Phase 2 · 任意)
                                    ▼
                              [theme_fork] ──ThemeForkEvent──▶ [theme_saved]
```

| 状態 | 意味 | 許可層 | ガード |
|------|------|:------:|--------|
| `theme_browse` | 同梱 / fork 済み pack 一覧 | L1+ | Phase 1 は選択のみ |
| `theme_preview` | 5 primitive サンプル + 任意 ScreenDef | L2+ | read-only データ |
| `theme_edit_tokens` | `design_token.yaml` 許可トークンの値編集 | L3+（Phase 2） | INV-TP-01 |
| `theme_lint` | トークン型 · contrast 検証 | — | NFR-DTH-09 警告可 |
| `theme_save` | 新 `theme_pack_id` で R2 INSERT | L1+（適用）/ L3+（編集） | INV-TP-02 |
| `theme_fork` | 親 pack から派生 | **L4** | `ThemeForkEvent` |

> **境界**: `theme_*` は OBS-TPL（計測テンプレ）・ScreenDef `content[]` 配置には遷移しない。観測テンプレ入口が必要なら **別 FeatureNode 導線**（§3）。

## 2. catalog pick フロー（紐づけ）

```text
[block 選択] ──「機能を紐づける」──▶ [catalog_browse]
   [catalog_browse] ──filter(kind/layer/status)──▶ [catalog_browse]
   [catalog_browse] ──部品選択──▶ [binding_config]
        [binding_config] ──props 入力──▶ [binding_config]
        [binding_config] ──確定──▶ lint(props_schema) ──pass──▶ [block bound]
                                                       └─fail──▶ [binding_config]（該当 prop ハイライト）
   [catalog_browse] ──キャンセル──▶ [block 選択]（none のまま）
```

| 状態 | 内容 | 空/エラー |
|------|------|-----------|
| `catalog_browse` | catalog 候補一覧（min_layer 以下はグレー） | **空状態**: 「利用可能な部品がありません（repo で component 追加が必要）」 |
| `binding_config` | props_schema に沿った宣言的値入力 | **エラー**: prop 不適合をハイライト・raw エラー非表示 |
| `block bound` | binding 確定（catalog_id セット） | — |

> **3 クリック導線**: ブロック選択 → 部品選択 → 確定（NFR-16-04）。

## 3. /dev 外導線（機能開発の分岐）

Builder 内で「新機能が欲しい」場合、**Builder では作らない**。導線は以下に分岐（read-only リンク）。

| 求めるもの | 導線先 | 補足 |
|------------|--------|------|
| 新 Component / Kernel | repo `components/` 追加手順（[フォルダ構成 ADR](./ADR-Phase1-IHL-repoフォルダ構成.md) §3） | フォルダ+manifest+ADR+catalog 行 |
| 新 API / ルート | backend + `docs/REQUIREMENTS.md` | 憲法更新必須 |
| 観測テンプレ/scale | 観測 FeatureNode / pipeline | REQ-027 |
| route 分類・カバレッジ | platform docs / CI（route-matrix） | **表示のみ**（FR-16-REFRAME-08） |

> UI 上は「この機能はまだありません → 開発パイプラインで追加（ヘルプ）」という **代替導線**を必ず示す（user-facing 未実装文言の禁止ルールと整合）。

## 4. エラー・例外導線

| ケース | 遷移 | UX |
|--------|------|----|
| lint 失敗 | `lint → lint_error → draft` | バナー + 該当ブロック強調・再試行 |
| catalog 外 binding | 留まる | グレーアウト + 代替（repo 手順リンク） |
| 保存衝突 | 新 id 発番 | INSERT ONLY のため上書きしない |
| 権限不足（層） | アクション無効 | ツールチップで必要層を表示 |
| ネットワーク失敗 | `lint`/`save` で再試行 | トースト + 自動リトライ（raw エラー非表示） |
