## 2. データ契約（ComponentBbsAnchor · board thread · R2）

### 2.1 GitHub BOARD 索引（読取モデル）

| フィールド | 型 | 規約 | FR |
|------------|-----|------|-----|
| `component_id` | string | `docs/components/{id}/` ディレクトリ名 | FR-19-01 |
| `github_board_url` | url | blob/{branch}/docs/components/.../BOARD.md | FR-19-01 |
| `github_discussion_url` | url | discussions?q={component_id} | FR-19-05 |
| `board_path` | string | repo 相対パス | NFR-19-02 |
| `board_exists` | bool | ローカル BOARD.md 有無 | FR-19-06 |

### 2.2 component-board API 応答 item

| フィールド | 説明 |
|------------|------|
| `thread_id` | #07 board_store スレ（任意） |
| `title` · `post_count` · `last_activity` | BBS メタ |
| `component_id` | C-USB / コンポーネント ID |
| `github_board_url` · `github_discussion_url` | 必須ディープリンク |

### 2.3 board_event（#07 共有 · append-only）

- thread/post 作成は **追記のみ**（NFR-19-01）。
- 削除 API **禁止** — 修正は追記。

### 2.4 legacy file_board_registry（salvage · gap）

| 列 | civ-os | IHL |
|----|--------|-----|
| repo_path | 975 行 CSV | **索引しない** |
| board_thread_rel | boards/P/p15... | GitHub URL |

## 3. API / route 契約（IHL 実エンドポイント · retrofit）

### 3.1 `GET /api/v1/component-board`

| 項目 | 契約 |
|------|------|
| 200 | `{ source: "github", items: [...] }` |
| items | component カテゴリ thread があればマージ · 無ければ BOARD.md のみ列挙 |

| route | 用途 |
|-------|------|
| `GET /api/v1/board/component/threads` | スレ一覧 |
| `POST .../threads` | スレ作成（append） |

|------|--------|-----|
| `--apply` linkage 生成 | `generate-file-board-linkage.ts` | **不使用** |
| registry-sync Vitest | CI 必須（legacy） | **対象外** |
