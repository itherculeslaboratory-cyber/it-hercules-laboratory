## 1. スコープ（IHL 最小）

本 v1 は **免罪符購入（count -1）** と **プロフィール経由の PT 残高表示** に限定。レイヤー 0–3 への一般投票 UI は **legacy 参照のみ**（IHL 初版スコープ外）。

## 2. データ契約（草案）

### 2.1 KarmaCountEvent（既存 08 連動）

| フィールド | 型 | 説明 |
|------------|-----|------|
| `user_id` | string | 対象ユーザー |
| `delta_count` | int | 通常 -1（免罪符） |
| `source` | enum | `amnesty_purchase` \| `monthly_decay` \| `dispute` |
| `pt_spent` | int | Fib(n) PT（免罪符時） |
| `created_at` | ISO8601 | R2 INSERT ONLY |

### 2.2 AmnestyPurchase（免罪符）

| フィールド | 型 | 説明 |
|------------|-----|------|
| `purchase_id` | uuid | |
| `user_id` | string | |
| `pt_price` | int | Fib(購入回数+1) — §22 連動 |
| `karma_count_before` | int | |
| `karma_count_after` | int | max(0, before-1) |

**副作用**: count 減少のみ · **カルマ値は直接変更しない**（08 §2.2）。
