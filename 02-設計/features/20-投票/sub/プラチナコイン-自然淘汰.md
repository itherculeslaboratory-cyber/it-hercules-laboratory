## ①機能概要（平易に3-5文）

投票・プラチナコイン・自然淘汰は、文明 OS の経済と統治を通じてレイヤー 0–3 の改善対象に評価を集め、使用率・投票・いいね・フォーク等の総合指標で優れた実装が残り不要なものが自然に消える仕組みである。プラチナコインは投票通貨として機能し、貢献度・依存グラフに基づく上流 10% 還元と月間発行上限等のルールが `PlatinumCoinRules.md` で定義される。レイヤー 4（固定資産）は投票・フォーク不可。実装は `/economy/vote`・admin platinum・contribution 連動まで **implemented** とインベントリ分類される。

## ④機能要件（番号付き）

### FR-20-01 投票対象範囲

- 投票可能: レイヤー 0（コード）〜 3（機能・OS 構成）（PlatinumCoinRules §2、Governance §5）。
- 投票不可: レイヤー 4（固定資産）。
- 対象の正当性判断は Governance に基づく。

### FR-20-02 プラチナ投票 UI

- `EconomyVotePage`: 認証必須。`useEconomyVote` で送信・バリデーション。
- `routeId` / `canonicalPath` / `returnUrl` クエリで画面レール文脈を表示（`data-testid="economy-vote-route-context"`）。
- 理由欄に「どの改善に対する投票か」を促す文言。

### FR-20-03 いいね・お気に入り

- いいね: レイヤー 0–3。貢献度入力となりうる（§3）。
- お気に入り: レイヤー 1–3 のみ。コード（L0）は収集対象外。

### FR-20-04 貢献度とプラチナ換算

- 貢献度 100 到達ごとにプラチナ 1 枚（端数繰越）（§5.3）。
- 子ノードへの評価の 10% を依存グラフの祖先へ分配（貢献度・プラチナ双方）（§5.2）。
- 集計・記録は **R2 append-only**（legacy: C-Sync → R2）。

### FR-20-05 自然淘汰

- 進化指標: 使用率、投票数、いいね、お気に入り、フォーク数、改善案数（Governance §10）。
- 総合評価により優れた機能は残り、不要な機能は自然消滅。
- Governance は基準定義のみ。記録は **R2 append-only**（legacy C-Sync は civ-os のみ）。

### FR-20-06 商用・成長インセンティブ

- IT.Hercules.Laboratory マーク使用時: 売上 3% を文明 OS へ（§6）。会計ログ R2。
- 月間定性付与・売上連動付与は §7・§16（政策数値は market_governance.csv 参照）。

### FR-20-07 フォークとの接続

- レイヤー 0–3 はフォーク可能。**IHL**: GitHub PR + R2 イベント（legacy: C-Sync 経由）。
- Builder fork v2・市場テンプレフォークは別 API だが経済評価は同一グラフに載る想定。

### FR-20-08 改善案テンプレ自動選択（legacy civ-os）

- legacy C-Sync はレイヤーに応じテンプレ選択（L3→FeatureNode 等）— **IHL では GitHub ADR/PR テンプレ**。

### FR-20-09 免罪符（カルマカウント減）

- プラチナマーケットで **黄金ヘラクレス教の免罪符** を販売する（商品名 · ユーザー確定 2026-06-07）。
- **1 購入 = カルマカウント -1**（0 未満にならない）。カルマ値の Fib 逆操作は **行わない**（§08 §4）。
- 正本: [`08-カルマシステム.md`](./08-カルマシステム.md) v2.0 · [`22-プラチナコインマーケット.md`](./22-プラチナコインマーケット.md)（購入 UI）· 裁判 §11 §14.3 · legacy `POST /api/economy/karma/indulgence` は salvage 参考。
- **価格**: **Fibonacci 段階** — 初回 **1 PT** · 購入ごと段階 +1 · 暦月ごと段階 -1（下限 1）· **購入上限なし**（[`22-プラチナコインマーケット.md`](./22-プラチナコインマーケット.md) §5 · §08 U-KRM-05 **解決**）。
- **TBD**: 永久 BAN 中の購入可否（§08 U-KRM-02）。

### FR-20-10 指摘課金（裁判連動）

- 掲示板・マーケット **指摘** は **30 回ごとにプラチナ 1 枚** 消費（クールダウンなし · §11 §4.3）。
- カルマ **Δcount とは別経路** — プラチナ台帳消費のみ。

## ⑥MiniKernel/C-USB上の位置づけ

```
FeatureNode(economy) → Kernel(view/process) → Component(投票フォーム) → 経済イベント（R2）
```

| 要素 | 位置づけ |
|------|----------|
| **レイヤー 0–3** | 投票・いいね・フォークの評価対象。WorldSystem 物理定義 + Governance 運用分類。 |
| **レイヤー 4** | 統治対象外。投票・フォーク不可。 |
| **PlatinumCoin** | 経済レイヤー（法律優先度 5）。Kernel 実装は `contributionEconomy.ts` 等。 |
| **legacy C-Sync** | civ-os 経済イベントの 4 媒体同期 — **IHL 不採用**（R2 + GitHub） |

**ルート**: `/economy/vote` — builder-coverage-matrix 上 **native**（Pilot-B13 対象）。`/economy` hub は hybrid 化済み。
