## 0. この文書の位置づけ

本書は **横断機能 #21（翻訳・言語）の `FR-I18N-*` を「実装と検証で追える設計」に厚くした v2** である。

## 2. データ契約（locale · messages · INSERT ONLY 方針）

### 2.1 ユーザー `locale` / `language`

| フィールド | 規約 |
|------------|------|
| 保存先 | `users/{id}.json` または preferences 投影（#12） |
| 形式 | BCP 47 想定 · 初期セット `ja` · `en` |
| 更新 | `PATCH /api/v1/me/preferences` · `language` キー（FR-I18N-SET-02） |
| 監査 | locale 変更 event（任意 · INSERT ONLY · 詳細設計 TBD） |

### 2.2 UI メッセージカタログ

| 項目 | 契約 |
|------|------|
| キー | ドット区切り（例: `nav.home` · `state.empty`） |
| フォールバック | `resolve_locale`: `en-*` → `en` · 未知 → `ja` ベースマージ（FR-I18N-UI-02） |
| 空白禁止 | マージ後もキー欠落時は ja 値を使用 |

### 2.3 UGC 原文（設計 · 未実装）

| 項目 | ルール |
|------|--------|
| R2 保存 | **原文のみ** · 翻訳結果を正本にしない |
| 表示 | 閲覧者 `locale` ≠ 原文言語 → **クライアント Transform**（FR-I18N-UGC-02） |
| 失敗 | 原文 + 1 行理由（FR-I18N-UGC-03） |

## 3. API / route 契約（IHL 実エンドポイント）

### 3.1 `GET /api/v1/i18n/messages`

| 項目 | 契約 |
|------|------|
| query | `locale`（任意 · 既定 `ja`） |
| 200 | `{ locale: string, messages: Record<string, string> }` |
| 解決 | `resolve_locale(locale)` 適用 |

### 3.2 `GET /api/v1/i18n/locales`

| 項目 | 契約 |
|------|------|
| 200 | `{ locales: ["ja", "en"] }` |
| 拡張 | ADR で言語追加 · オンボーディング/設定と **同一集合**（FR-I18N-SET-01） |

### 3.3 locale 変更（#12 委譲 · 横断正経路）

| 項目 | 契約 |
|------|------|
| route | `PATCH /api/v1/me/preferences` · body `{ language: "en" }` |
| legacy | `POST /api/auth/locale` → **設定変更専用に位置づけ直し**（salvage · IHL は PATCH 正） |
