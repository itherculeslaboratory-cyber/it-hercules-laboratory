# 21 翻訳・言語 — 遷移設計 v1（横断 i18n · locale フロー）

> **横断接続**: #03 新規登録 · #12 設定 · #07 掲示板 · #11 裁判

---

## 1. 主要 state（横断）

| state | 入口 | 出口 | エラー導線 |
|-------|------|------|------------|
| **locale 未設定** | 新規登録完了直前（#03） | オンボ locale 選択 → preferences 保存 | 必須未選択 → インライン理由 |
| **locale 既定（ja）** | 初回 API 応答 · 未 PATCH | 設定画面で変更 | — |
| **locale 変更中** | `/settings` · `PATCH /me/preferences` | 200 → 全 UI 再描画 | 400 未知 locale → 許可リスト表示 |
| **i18n カタログ取得** | アプリ起動 · locale 変更後 | `GET /i18n/messages?locale=` 200 | 404/500 → フォールバック `ja` + 短い理由 |
| **UGC 翻訳表示（gap）** | BBS/二人部屋閲覧 | クライアント Transform（設計） | 翻訳不可 → 原文のみ + 注記 |

---

## 2. 遷移図（製品 UI · 3 クリック以内）

```
[登録 #03] → locale 必須選択 → [preferences.language 保存]
[任意画面] → 設定 (#12) → 言語セレクタ → PATCH → [同一画面再描画]
[起動] → GET /i18n/messages → [ヘッダ/空状態/エラーが locale 反映]
```

---

## 3. API 遷移（retrofit 実装済）

| 操作 | 遷移 | 永続化 |
|------|------|--------|
| 初回 locale | #03 onboarding → #12 preferences | `language` フィールド |
| 変更 | `PATCH /api/v1/me/preferences` | event_store 投影 |
| カタログ | `GET /api/v1/i18n/messages` | 静的 catalog（コード内） |

---

## 4. エラー・空状態

| 条件 | UI 応答 |
|------|---------|
| 未知 locale | 400 + 許可 `ja`/`en` リスト |
| カタログ欠落キー | キー文字列フォールバック（開発時 WARN） |
| UGC 翻訳未実装 | 原文表示 · 「翻訳は端末側（予定）」注記（gap） |
