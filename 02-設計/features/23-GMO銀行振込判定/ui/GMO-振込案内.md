# 23 GMO 振込案内 — UI 設計（イメージ画面）

> **ステータス**: **草案 · 人間目視レビュー待ち**
> **正本**: [`../01-要件/23-GMO銀行振込判定.md`](../01-要件/23-GMO銀行振込判定.md) §2（振込コード）· §3（8% フロー）· [`06-マーケット.md`](../01-要件/06-マーケット.md) §11.7

---

## 1. 主タスク（1 つ）

**「振込コードを名義に付けて 8% 貢献費を振り込む」**。**取引成立後**（配送完了 + 評価確定 · §06 §11.0.1）にのみ表示。マッチング確定では出さない。

---

## 2. イメージ画面

![GMO 振込案内](./mockups/ihl-23-gmo-transfer.png)

| チャンク | 内容 |
|----------|------|
| **情報バナー（青）** | 「取引成立しました。下記の内容でお振込ください」 |
| **振込先 / 金額** | GMOあおぞらネット銀行 / 口座 / 金額（¥960 を強調）— 各行 **コピー** |
| **振込コード（強調）** | `U-XXXX`（monospace 大・**コピー** ボタン強調 · §2.2 `deriveTransferCode(userId)`） |
| **名義追記の注意（黄）** | 「振込依頼人名の末尾にコードを追記」例: `ﾔﾏﾀﾞ U-4F9A`（GMO 48 半角文字以内 · §2.3） |
| **主操作** | 〔振込を済ませた〕 |

> 銀行 API は userId を返さない → **名義 + 金額 + 入金日時** で照合（§2.5 日時 FIFO）。ユーザーごと **1 コード**（8% も PT も同一 · §2.4）。

---

## 3. 状態（必須）

| 状態 | 表示 |
|------|------|
| loading | 金額・コードはスケルトン |
| 照合待ち | 「入金確認中」（GMO Webhook 待ち） |
| 照合済 | 緑「入金を確認しました」+ fee_unpaid 月次停止（§3） |
| error | 「名義にコードを含めてください」等の短い理由（NFR-GMO-04） |

---

## 4. デザイン規範

- **コピー UX 最優先** — 振込コード・金額・口座は 1 タップでコピー。
- 振込コードは氏名より優先（照合失敗より名義省略の方がマシ · §2.3）。
- 秘密値（署名鍵）は画面・ログに出さない（NFR-GMO-01）。
- 本番実入金・署名形式は **人間ゲート**（`P0-NEXT-GMO-LIVE-EXEC` · FR-GMO-07）。
