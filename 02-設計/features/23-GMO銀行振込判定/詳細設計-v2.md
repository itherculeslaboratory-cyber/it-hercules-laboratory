## 2. 振込コード（deriveTransferCode · FR-GMO-05）

| ステップ | 処理 |
|----------|------|
| S1 | `SHA-256(UTF-8(userId))` |
| S2 | 先頭 3 バイト uint24 BE |
| S3 | Base36 大文字 |
| S4 | 4 文字 pad / 6 文字 truncate |
| S5 | `"U-" + body` |

- **保存**: `users.transfer_code` · **再生成禁止**
- **名義**: 振込依頼人名末尾追記 · GMO **48 半角字**（§2.3）
- **登録時衝突 alternate-slice**（要件 §2.2 · digest +3 バイトオフセット再試行）: **IHL 未実装** — `derive_transfer_code` は決定的 1 パスのみ。衝突検知・DB 一意制約・alternate 導出は **gap**（P4 以降 · RTM 追跡）。

## 4. 照合アルゴリズム

### 4.1 現行 IHL（retrofit）

```
C = pending ∧ amount 一致 ∧ remittance_reference ⊆ normalize(remitter+remarks)
→ created_at 最古 1 件のみ matched（FIFO · FR-GMO-02 単一決定）
```

**Webhook idempotency（NFR-GMO-02）**: `raw_body_digest`（SHA-256 先頭 32 hex）で重複検知。同一 body 再送時は **初回の matched_expected_ids を返却**し、追加 matched・webhook_log 重複 append を行わない（`receive_webhook_and_match` · `test_webhook_idempotency_same_body_no_double_match`）。

### 4.2 要件 v1.5（設計正本 · gap）

§2.5.3: `|C|≥2` → **remittance_datetime** + **`created_at` 下限** → **最古義務 FIFO 1 件**

| 区分 | 状態 |
|------|------|
| 単一 pending マッチ | **existing** |
| 日時 FIFO | **gap** |
| 部分入金残債減算（FR-GMO-08） | **gap** |
| 過入金クレジット（FR-GMO-09） | **gap** |
