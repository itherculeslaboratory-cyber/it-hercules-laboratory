## 1. 目的・範囲

土台が**システム全体として一貫**しているか（raw 取込→派生→検索投影の通し · 構造規範の静的整合）を担保する。#00 は UI を持たないため、システム層は **(a) パイプライン E2E** と **(b) 構造・契約の静的整合検査** の 2 系統で構成する。

## 4. カバレッジ方針

- E2E（ST-00-01/02）で **R2 SSOT + append-only の通し**（FOUND-D02/D05/D06）。
- 静的検査（ST-00-03/04/LINT-05）で **構造規範**（コードで強制されない FOUND-S01/S02/S04/S08）と **schema 整合**を担保。
- 規範系 TC は手動レビュー + grep を併用し、結果を RTM `status` に記録。
