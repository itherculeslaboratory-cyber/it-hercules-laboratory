## 1. 目的・範囲

土台ライブラリ（schema 検証 · R2 append-only I/O · component catalog）の**関数内部の振る舞い**を pytest で担保する。HTTP / E2E は対象外（結合・システム層）。

| 範囲 | 対象 |
|------|------|
| In | `SchemaRegistry` / `validate_instance` · `R2Client` / `LocalFilesystemBackend` · `catalog` |
| Out | API route（→ 結合）· 横断フロー（→ システム）· FR 受入（→ UAT） |

## 4. カバレッジ方針

- **FOUND-S03 / N01 / N02**（C-USB · RAG · OSS 薄ラップ契約）: schema 検証 UT で担保。
- **FOUND-D01 / D02 / D04 / D06**（R2 SSOT · INSERT ONLY · core+rag · 上書き禁止）: R2 UT で担保。
- **FOUND-N05**（構造整合性 / path 安全）: traversal · placeholder UT。
- **構造規範 FOUND-S01/S02/S04/S05/S07/S08**（階層・画面廃止・Kernel 種・IHL 分離）は**コードで強制されない規範**のため、単体では検証せず**受入/システム層の設計レビュー TC**（UAT-00-RV / ST-00-LINT）で担保する旨を RTM に明記。
