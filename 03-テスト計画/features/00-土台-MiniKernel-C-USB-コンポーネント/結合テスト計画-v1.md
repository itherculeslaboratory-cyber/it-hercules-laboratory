## 1. 目的・範囲

土台契約が**複数 component / パイプライン段をまたいで整合**することを担保する。1 関数内部（単体）と E2E（システム）の中間層。

| 範囲 | 対象 |
|------|------|
| In | パイプライン段間の manifest 受け渡し · schema↔実体データ整合 · R2 経由の段間連携 · catalog 駆動の実行順 |
| Out | 関数内部分岐（単体）· ブラウザ / UI フロー（システム）|

| TC ID | シナリオ | 期待 | req_id | 既存/差分 |

---

## 4. カバレッジ方針

- パイプライン段間の **manifest 契約 + append-only**（FOUND-S09 / D06 / D07 / N07）を中心に担保。
- route × schema は**結合層の契約 TC**（IT-00-06）で確認し、ステータスコードだけでなく schema 適合を assert（C3 準拠）。
- 構造規範（FOUND-S01/S02 等）は結合では検証せず、システム/受入の設計レビューへ委譲（RTM 明記）。
