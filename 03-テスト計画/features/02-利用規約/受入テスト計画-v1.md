## 1. 目的・範囲

利用規約の **FR-* がユーザー視点で充足**するかを、操作シナリオ（要件 §② U1〜U6）と設計レビューで判定する。技術同意（草案）と binding 法務受入を分ける。

| 種別 | 判定者 |
|------|--------|
| 操作受入（U1〜U3） | 設計 AI 監査役 + 人間 spot check（API/dev フロー） |
| 法務 binding 受入 | **HUMAN-02-LEGAL**（運営者・準拠法・管轄 · 条文確定） |

## 3. 人間ゲート / ギャップ（AI が `[x]` 不可）

| TC | 区分 | 理由 |
|----|------|------|
| UAT-02-10（FR-04/05/FR-LEGAL-03 binding） | **HUMAN** | HUMAN-02-LEGAL — 法務条文確定・運営者情報・準拠法は人間サインオフ |
| UAT-02-11（二層表示） | **GAP/deferred** | IHL 未配線（v2 §7 P4）。設計契約のみ · binding 法務後に retrofit |
| UAT-02-12（動画/resolver） | **GAP** | media-map API 未実装 · YouTube URL はマッピング側集約（設計のみ） |
| UAT-02-13（scroll gate 等） | **GAP/xref** | scroll gate · 暗黙同意廃止は #03 主導。FR-UX-03 は API 層で充足 |

その他 UAT-02-01〜09 は dev フロー + 設計レビューで AI 監査役 + 機械検査により判定可能。

## 4. 合格条件

- UAT-02-01〜09 が PASS（API テスト緑 + レビュー記録）。
- UAT-02-10 は HUMAN ゲート · UAT-02-11〜13 は GAP/deferred（将来 retrofit）。
- RTM の全 `FR-*` / `NFR-*` が UAT 行に紐づく（gap/human/xref 含め明示）。
