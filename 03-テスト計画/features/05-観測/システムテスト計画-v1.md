## 1. 目的・範囲

固体観測の **入力 → commit（capture + env measurement）→ 検索/詳細 → 再現性メタ** をシステム観点で確認する。R2 / event store の **INSERT ONLY** を通しで担保する。

---

## 2. シナリオ（ST）

| TC ID | シナリオ | 手順 | 期待 | req_id |
|-------|----------|------|------|--------|
| ST-05-01 | 固体 commit 通し | 種/性別入力 → POST /api/captures | 201 committed · cap_* 永続 | OBS-SOL-01 |
| ST-05-02 | 環境同時記録 | device+telemetry → include_env | env measurement 2 行付き commit | OBS-ENV-01/02 |
| ST-05-03 | INSERT ONLY | 同一 capture を 2 回 | 別 capture_id（上書き無し） | OBS-R2-01/02 |
| ST-05-04 | 由来分離 | 手入力 + iot 混在 | value_origin が direct/environment で区別保持 | OBS-REP-IHL-02 |
| ST-05-05 | 検索安全 | 不正 filter | 400（whitelist 外拒否） | OBS-TAX query |
| ST-05-06 | 3 クリック | テンプレ→値→保存 | 観測開始 3 クリック以内 | OBS-NF-03 |
| ST-05-07 | 秘密非表示 | env commit 応答 | telemetry 値のみ · 鍵/秘密非露出 | OBS-NF-02 |
