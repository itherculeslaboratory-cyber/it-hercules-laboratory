## 0. ユーザー向けサマリー（先に読む）

**今どこまで終わったか**: アイディア §1–8 の **設計判断（ADR）は全節カバー済み**。§7→ADR-H-04、§8→ADR-H-05、§5–6→ADR-H-06＋H-08、§3–4→ADR-H-07＋H-09＋H-10、§1–2→ADR-H-11＋H-12。**設計網羅 Batch 2–7 草案**（[`00-設計網羅監査-統合.md`](./00-設計網羅監査-統合.md)）により schema YAML **草案**（[`02-設計/_横断/schema-yaml-draft-v1.md`](../02-設計/_横断/schema-yaml-draft-v1.md)）・辞書 7 本・UI モック 30 枚も揃った。**残りは「ADR の決定を各要件たたき台へ反映（本ファイル Batch 2 の 2-2〜2-7）」と「Truth Layer の本番 YAML 確定（IHL repo 待ち · Batch 3）」**。

**何が嬉しいか**: 

- プロフィールは **カルマ／貢献度（研究スコア内訳）／マーケット評価の 3 つを混ぜずに**並べる方針が確定（H-08）。
- 研究は **「観測→仮説→試す→記録→引用」を往復ゼロ・約 7〜9 クリック**で回す主導線が確定。Dashboard・Knowledge Graph は **作り込みすぎない**（H-09）。
- 掲示板は **4 入口のまま裏側（投稿 Truth 層）もきれいに**（append-only の ThreadEvent/PostEvent・H-10）。
- 血統・交配は **CrossParent / OffspringAssignment / Lineage=Snapshot** で丁寧に設計＋モック完成（H-11）。
- 類似検索の「0.5/0.2/0.2/0.1」を **普通の日本語で説明**＋ v0.2 採用（H-12）。

**人間がやること（最後だけ）**: 設計ゲート 4 点（要件・詳細・遷移・UI）の **確定レビュー**。それ以外は AI が完走する。
