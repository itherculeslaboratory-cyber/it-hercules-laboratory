## 1. 問題点（現状が直感的でない理由）

### 1.1 名前と中身の不一致

| フォルダ名 | 期待 | 実際 |
|------------|------|------|
| `01-要件/` | 凍結 REQ (#00–#23) | 150+ ファイル：詳細/遷移/UI v1、ADR、監査、Batch、たたき台が混在 |
| `02-設計/_ui-global/` | UI 全体 | ゲート4点の `*-UI設計-v1.md` と役割分担が README 依存で不明 |
| `05-運用/queues/` | 横断設計 | 機能別実装設計（例: `13-データ取得元-実装設計-v1.md`）も同居 |

### 1.2 V字モデルと物理配置が非対称

- 左腕（設計）成果物が `01-要件/`・`02-設計/_ui-global/`・`05-運用/queues/` に三分割  
- 右腕（テスト計画）の置き場が計画上 `05-運用/queues/features/` だが **未作成**  
- 「#05 の全部」を開くのに 3 ツリーを横断する必要がある

### 1.3 具体例（#05 観測）

```
01-要件/05-観測.md                    ← REQ 凍結
02-設計/features/05-観測/05-観測-詳細設計-v1.md
02-設計/features/05-観測/ui/UI設計-v1.md
02-設計/features/05-観測/ui/入力UI設計-v1.md     ← サブ画面が REQ フォルダ内
02-設計/features/05-観測/ui/入力.md                         ← 同名トピックの別正本？
02-設計/_ui-global/mockups/mockups/mockups/ihl-05-*.png
```

→ **「どれが正本か」がパスだけでは判別不能**

### 1.4 その他

- トップの `02-`〜`06-*.md` と `05-運用/queues/00-*` が索引役割を競合  
- `02-設計/_横断/schema/` が設計 repo とコード repo で同名  
- `指示/it-hercules-laboratory/99-アーカイブ/2026.06-06-legacy` `指示/it-hercules-laboratory/99-アーカイブ/2026.06-08-review` が IHL ツリー外に孤立

## 2. 設計原則

| # | 原則 | 意味 |
|---|------|------|
| P1 | **V字の左右をトップで分離** | 左=設計 / 右=テスト計画 / 中央=凍結要件 |
| P2 | **機能 #NN は `features/NN-*/` に集約** | 1機能の文脈を1フォルダで完結（直感優先） |
| P3 | **凍結 REQ は書き換え禁止** | CR のみ · パスは `01-要件/` |
| P4 | **横断 ADR・キュー・監査は種別フォルダ** | 機能番号を付けない |
| P5 | **Strategy B: 設計+実装を単一ツリー** | 正本=`指示/it-hercules-laboratory/`（`apps/` `libs/` `collector/` と `01-05/` 同階層）· civ-os ルート `it-hercules-laboratory/` は shim のみ |
| P6 | **移行は doc 先行** | Phase 0=README リダイレクト · Phase 1=新規のみ新パス · Phase 2=bulk move |
| P7 | **命名: 日本語フェーズ + 既存英語技術語** | `01-要件` `RTM` `ADR` `mockups` は維持（ツール・ルール整合） |

**命名の理由**

- **`01-`〜`05-` 数字プレフィックス**: V字の上から下へ読む順と一致（README の「読む順」と同型）  
- **`features/NN-機能名/`**: V-WAVE・RTM・parity の `--feature NN` と1:1  
- **`_横断/` アンダースコア**: ソート最上部 · 「機能番号なし」を明示  
- **`02-設計` vs `03-テスト計画` 分離**: ゲート5点の左右腕を物理分割（ihl-waterfall-v-model-gate と整合）

## 4. 比較：機能1フォルダ vs 種別フォルダ

| 観点 | 機能 #NN 1フォルダ（全体） | 種別フォルダ（01-要件/02-設計/…） | **推奨** |
|------|---------------------------|-----------------------------------|----------|
| 「#05 全部見たい」 | ◎ 1クリック | △ 3ツリー横断 | **ハイブリッド** |
| V字左右の分離 | △ 設計とテストが同階層 | ◎ 明確 | **種別トップ + features/NN** |
| 150ファイル一括 move | ◎ 24フォルダに分割 | ○ 種別ごとに move | 段階移行で両方 |
| CI/ADR/監査 | △ NN に紐づけにくい | ◎ `_横断/` | **_横断/** |
| 既存ルール整合 | △ パス全面変更 | ○ 最小変更 | **features/ は V-model 計画と一致** |
| AI エージェント | ◎ `--feature 05` と一致 | △ パス推測が必要 | **features/NN** |

**推奨**: **種別トップ（01–05）+ `features/NN-機能名/` サブフォルダ**  
→ ユーザーが言う「直感的」= **機能単位で開ける** + **V字のフェーズが見える** の両立。
