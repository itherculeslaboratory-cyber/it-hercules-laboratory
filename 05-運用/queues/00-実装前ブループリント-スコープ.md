## 1. 完成の定義（Definition of Done）

### 1.1 城代（設計パック）とは

実装担当（人間 or Codex）が **追加の設計判断なしに** 着手できる成果物セット:

| 層 | 含むもの | 正本パス |
|----|----------|----------|
| 要件 | FR/NFR・境界・採用 REQ 参照 | `01-要件/NN-*.md` |
| 詳細 | schema · API · state · データ契約 | `NN-*-詳細設計-v1.md` |
| 遷移 | 画面/状態遷移 · エラー導線 · 深さ ≤3 クリック | `NN-*-遷移設計-v1.md` |
| UI | ワイヤー · モック PNG · 空/エラー/ローディング | `NN-*-UI設計-v1.md` · `02-設計/_ui-global/` |
| 横断 | ADR · 辞書 YAML · OSS 監査行 | `ADR-H-*.md` · `02-設計/_横断/schema/` · `05-運用/queues/` |
| トレース | mock↔FR↔遷移の機械対応表 | `00-モック要件トレーサビリティ-v1.md` |
| 監査 | 委任 Go 証跡 | `00-監査役-実装前ゲート-v1.md` |

### 1.2 「要望通り」の判定基準

1. **機能網羅**: 01〜20 の FR が 4 点設計 + モックで **漏れなく** トレース可能  
2. **横断整合**: H-04/H-05/H-08 等の ADR と数値・用語が矛盾しない  
3. **同一完成度**: どの機能も「詳細だけ草案」「モックだけある」といった **偏りなし**（ギャップマトリクスで ✗ が 0）  
4. **OSS 安全**: Phase 1 + Phase 2 の依存がライセンス監査表に **全行記載**  
5. **人間タッチポイント済み**: モック目視 OK が監査ログに記録されている  
6. **実装 Go は別**: 上記 1〜5 は **設計完了**。コード着手はユーザー Sign-off

### 1.3 明示的にスコープ外（今回のブループリント）

| 項目 | 扱い |
|------|------|
| 世界観画像の **生成** | 後追い。今は [`00-世界観アセット一覧-v1.md`](../02-設計/_ui-global/00-世界観アセット一覧-v1.md) + `02-設計/_ui-global/assets/` 配置のみ |
| GMO 本番入金 | 人間ゲートのまま |
| 法務条文の **最終版** | 02 たたき台まで。確定は人間 |
| `frontend/` `backend/` 実装 | Implementation Sign-off 後の別フェーズ |

## 4. 人間タッチポイント（のみ）

| タイミング | 人間がすること | AI がすること |
|------------|----------------|---------------|
| **モックレビュー** | PNG / walkthrough を目視。違和感を短いフィードバックで返す | 差分を設計 doc + mock に反映 |
| **実装デモ** | 実装 Sign-off 後、動作デモで最終確認 | E2E・自己 QA まで準備 |
| **法務・GMO・鍵** | 条文確定・本番証跡・Tier D | たたき台・runbook のみ |

**人間がしないこと**: 数万文字の設計全文読解、機械トレースの手作業、索引の機械更新。
