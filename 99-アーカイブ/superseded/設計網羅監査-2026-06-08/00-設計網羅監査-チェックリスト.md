## E. 実装 Go 前の P0 設計ギャップ（優先順）

1. **P0-1 観測入力 UI（懸念 7・8）** — 項目/値/単位 dropdown+追加・計測方法（手入力/IoT）・機器管理リンク・雌雄別テンプレ作成。現状 UI-v1 は検索のみ。
2. **P0-2 端末/Docker 実行アーキ ADR（懸念 2・3）** — 端末ブラウザ / Docker component / R2 の責務境界、Web→ローカル HTTP ジョブ投入の有無を ADR 化。**現在ゼロ**。
3. **P0-3 UI 設計の全機能ゼロ（X1）** — 05/06/07/10/14/16/23 のワイヤー（モック PNG はあるが人間未確定）。
4. **P0-4 遷移設計が概念止まり（X2）** — Listing/dispute/run-snapshot の state machine 図・許可辺表。
5. **P0-5 schema YAML 昇格（X3）** — `searchable_capture_set` 等の列は転記済だが `02-設計/_横断/schema/*.yaml` 未確定（型/required/enum）。
6. **P0-6 AI 学習 fast-path 詳細（懸念 1）** — dataset 生成 component の入出力契約・value_origin フィルタ。

> 補足: [`00-要件完成度監査`](./00-要件完成度監査-05-07-10-14-16-23.md) の **X4「16 二版未決着」は ADR-H-01 で REFRAME(B) 採用済**のため実質解消。監査表が ADR 確定前のため古い。
