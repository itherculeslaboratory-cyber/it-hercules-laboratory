## 1. 雄/雌ユーザー追加観測項目

### 1.1 現状

- IHL `searchable_capture_set` に `sex` 列（`male / female / unknown / undetermined`）は存在する（`05-観測.md` §⑪.2 辞書 enum 済み）。
- IHL `measurement` schema（縦持ち）は `measurement_name`, `measurement_value`, `measurement_unit`, `measurement_method`, `value_origin` を持ち、原理上は雄/雌問わず任意の計測名を記録可能。
- civ-os `observationTemplates.ts` の `extraFields` は **サーバーコード静的定義のみ**（`mass_g`, `batch_note`, `pre_molt`）。ユーザーが動的に観測項目を追加・保存する機能は未実装・未設計。

### 1.2 欠落している FR（未定義）

| FR ID（案） | 要件 |
|-------------|------|
| **OBS-TPL-01** | ユーザーが観測セッションに「雄用項目」「雌用項目」を追加できる（性別 = `sex` 値に応じた項目セット切替） |
| **OBS-TPL-02** | `sex = male` 時は「角長 / 体長 / 胸幅」が優先表示。`sex = female` 時は「体長 / 胸幅」が優先表示（角長は非表示 or 任意）。切替 UI を提供する |
| **OBS-TPL-03** | ユーザーが項目名（`measurement_name`）を選択または自由入力で追加できる |
| **OBS-TPL-04** | 追加した項目は `measurement` 縦持ち schema の行として IHL R2 に保存される |

### 1.3 ギャップ

- `measurement_name` の辞書（`dictionaries/measurement_name.yaml`）は **パス予約のみで内容未定義**（要件定義1 §9 に存在確認、内容なし）。
- 「雄専用計測項目（horn_length_mm）」「雌専用（ovipositor_length_mm 等）」の分類が schema / UI 要件いずれにも記載なし。
- `value_type = numeric / categorical / boolean / text` の分類（詳細設計書 §2 Observation）は Truth Layer に存在するが、UI フォームへの割り当て（テキスト入力/数値入力/チェックボックス）の対応表が未定義。

## 3. UI — 項目 DD/追加・数値・単位 DD/追加・計測方法

### 3.1 現状

- **IHL Streamlit 5 pane**（`05-観測-Streamlit.md`）: `filter → grid → detail → similar → tag` の検索 UI モックは存在する。
- **civ-os 固体観測フロー**（`SolidObservationFlow.tsx` 実装済み）: `extraFields` による動的フォームが Phase6 として実装されているが、項目は static。
- 計測項目の UI に必要な「項目選択/追加」「単位選択/追加」「計測方法（手入力 / IoT / →機器管理遷移）」のコンポーネント仕様が **どこにも定義されていない**。

### 3.2 欠落 UI 要件（フォーム仕様）

設計が必要な UI チャンク（モック未作成）:

```
観測項目追加フォーム（計測テンプレ行追加）
┌──────────────────────────────────────────────────┐
│ 項目 [ドロップダウン ▼ または「＋ 自由入力」]   │
│ 数値 [____________________]                      │
│ 単位 [ドロップダウン ▼ または「＋ 自由入力」]   │
│ 計測方法                                          │
│   ○ 手入力                                       │
│   ○ IoT（SwitchBot / collector）                 │
│   → 機器管理へ（遷移リンク）                     │
└──────────────────────────────────────────────────┘
```

**未定義の UI 仕様**:

| UI 要素 | 現状 | 欠落内容 |
|---------|------|----------|
| 項目ドロップダウン | `measurement_name` 辞書は予約のみ | 辞書内容・ドロップダウン候補の定義なし |
| 項目自由入力 | なし | 「＋ カスタム項目」追加フロー未設計 |
| 数値入力 | `extraFields type: number` | 単位なし・バリデーション未定義 |
| 単位ドロップダウン | なし | mm / g / °C / % / ppm / 任意の候補・辞書なし |
| 単位自由入力 | なし | カスタム単位保存先スキーマなし |
| 計測方法（手入力） | `SolidEnvironmentSnapshot.manual` あり | 観測項目ごとの `measurement_method` 選択 UI なし |
| 計測方法（IoT） | SwitchBot poller 実装あり | セッション内でどの項目が IoT 由来か `value_origin` に接続する UI なし |
| →機器管理遷移 | `/settings` などに機器管理あり | 計測フォームから直接機器管理へのリンク要件が FR に存在しない |

### 3.3 計測方法と `value_origin` の接続

IHL `measurement` schema には `value_origin`（`direct_observed / image_derived / environment_derived / estimated` 等）と `measurement_method`（計測方法文字列）が存在する。しかし:

- **civ-os の観測フォームから IHL measurement schema への書き込みフローが未設計**（`commit` API の `templateExtras` に入れるだけで IHL 側に正規化する経路なし）。
- `measurement_method = "IoT_SwitchBot"` 等の定義がなく、辞書化されていない。

## 4. ADR-H-11 血統メトリクス接続（instar 体重・死亡率）

### 4.1 現状（設計済み部分）

ADR-H-11 §5 の R2 パスは定義済み:

```
ihl/lineage/snapshots/
├── cross_growth_summary/    ← 世代別 成長/成績 集計（instar 体重候補）
└── cross_death_summary/     ← 死亡実績集計（mortality 候補）
```

また `2026.06.08/アイディア` §1 に「世代ごとの成績・成長・死亡・繁殖実績を集計可能とする」が方針として確定。`larva_subtype` 辞書に `L1 / L2 / L3 / unknown_instar` が定義済み。

### 4.2 欠落（具体的メトリクス定義）

| 欠落 | 内容 | 影響 |
|------|------|------|
| **幼虫体重 measurement_name** | `weight_g` は searchable_capture_set に列としてあるが、`measurement_name = "larva_weight_g"` 等の instar ステージ別計測名を `dictionaries/measurement_name.yaml` に定義していない | instar 別体重の集計 SQL が書けない |
| **死亡率集計ルール** | `cross_death_summary` のスキーマ（`cross_id, stage_name, instar, count_alive, count_dead, mortality_rate` 等）が未定義 | manifest_builder が集計できない |
| **cross_growth_summary スキーマ** | `cross_id, larva_subtype, avg_weight_g, max_weight_g, sample_count, snapshot_id` 等の列定義なし | 系統ごとの成長比較不可 |
| **life_event との接続** | `event_type = death`（ADR-H-11 §5 `life_event.jsonl`）が記録されるが、死因・ステージ・instar 情報を death イベントにどう載せるか未定義 | 死亡率計算の分母・分子が曖昧 |
| **instar 体重 UI** | `/individual/:id` や `/cross/:id` 画面で instar 別体重推移グラフ / 表の UI 仕様なし | ADR-H-11 §3「交配画面モック」には instar 成長表示が含まれていない |
